Reporta problemas causados por un uso incorrecto del valor nil.

El IDE analiza el flujo de datos para determinar si las variables pueden ser nil o no nil. Basado en esto, el IDE reporta problemas potenciales en el código. Considere la siguiente lista de situaciones que podrían llevar a consecuencias imprevistas:

  • Las llamadas a métodos con el receptor nil pueden llevar a una 'desreferencia de puntero nil'.
  • La indexación de una rebanada nil puede causar pánicos.
  • Comparaciones como v == nil pueden no tener sentido si se sabe que v siempre es nil o no nil.
  • Las variables con errores correspondientes no se verifican para no ser nil. Un error corresponde a una variable cuando se definen o asignan juntas en declaraciones como v, err := foo().
