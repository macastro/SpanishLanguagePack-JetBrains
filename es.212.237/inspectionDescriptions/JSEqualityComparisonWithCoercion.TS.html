Informa un uso de operadores de igualdad que pueden causar conversiones inesperadas de tipo. Sugiere reemplazar los operadores de igualdad == o != con operadores seguros === o !==.

Dependiendo de la opción seleccionada, se informará uno de los siguientes casos:

  • Todos los usos de los operadores == y !=.
  • Todos los usos excepto la comparación con nulo. Algunos estilos de código permiten usar x == null como reemplazo de x === null || x === undefined.
  • Sólo expresiones sospechosas, tales como: comparaciones == o != con 0, '', null, true, false, o undefined.
