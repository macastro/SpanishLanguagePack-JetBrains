Se reportan los métodos getClass() que se invocan en una instancia java.lang.Class.

Esto suele ser un error, ya que el resultado es siempre equivalente a Class.class. Si es un error, sería mejor eliminar la llamada a getClass() y usar el calificador directamente. Si el comportamiento es intencionado, sería mejor escribir explícitamente Class.class para evitar confusiones.

Ejemplo:


  void test(Class<?> clazz) {
    String name = clazz.getClass().getName();
  }

Después de que se aplique una de las posibles soluciones:


  void test(Class<?> clazz) {
    String name = clazz.getName();
  }
