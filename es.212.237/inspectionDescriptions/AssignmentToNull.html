Informa sobre las variables que se asignan a null fuera de una declaración.

El propósito principal de null en Java es denotar variables de referencia no inicializadas. En casos excepcionales, asignar una variable explícitamente a null es útil para ayudar a la recolección de basura. Sin embargo, utilizar null para denotar un valor ausente, no especificado o no válido o un elemento no encontrado se considera una mala práctica y puede hacer que su código sea más propenso a generar NullPointerExceptions. En su lugar, considere definir un objeto centinela con la semántica prevista o utilizar tipos de biblioteca como Optional para denotar la ausencia de un valor.

Ejemplo:


  Integer convert(String s) {
    Integer value;
    try {
      value = Integer.parseInt(s);
    } catch (NumberFormatException e) {
      // Aviso: null se utiliza para denotar un 'valor no válido'
      value = null;
    }
    return value;
  }

Utilice la opción Ignorar asignaciones a campos para ignorar las asignaciones a campos.
