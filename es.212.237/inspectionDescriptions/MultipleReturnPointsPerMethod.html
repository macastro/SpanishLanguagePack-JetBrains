Informa de métodos cuyo número de puntos de return supera el máximo especificado. Los métodos con demasiados puntos de return pueden ser confusos y difíciles de refactorizar.

Un punto de return es una sentencia de return o una caída a través del fondo de un método o constructor void.

Ejemplo:

El siguiente método se notifica si solo se permiten dos sentencias de return:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user)) {
        user.setId(getId());
        return;
      } else if (cond2(user)) {
        if (cond3(user)) {
          user.setId(getId());
          return;
        }
      }
    }
  }

Considera reescribir el método para que sea más fácil de entender:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user) || cond2(user) && cond3(user)) {
        user.setId(getId());
        return;
      }
    }
  }

Configurar la inspección:

  • Usa el campo Límite de puntos de retorno para especificar el número máximo permitido de puntos de return para un método.
  • Usa la opción Ignorar cláusulas de protección para ignorar las cláusulas de protección. Una cláusula de protección es una sentencia if que solo contiene una sentencia return
  • Usa la opción Ignorar métodos 'equals()' para ignorar los puntos de return dentro de los métodos equals().
