Informa los siguientes problemas en descriptores que se utilizan para implementar tu módulo web en un servidor:
  • Referencias a clases no instanciables
  • Referencias a clases que no extienden la clase requerida
  • Referencias a clases con alcance inapropiado
  • Valores de etiquetas y atributos vacíos
  • Valores de etiquetas y atributos que no coinciden con el patrón requerido (por ejemplo, identificadores de Java)
  • Etiquetas que no incluyen etiquetas o atributos secundarios requeridos
  • Etiquetas que definen objetos con nombres duplicados

Ejemplo:



<web-app>
  <filter-mapping>
    <filter-name>CustomFilter</filter-name> // Error: se debe definir una etiqueta secundaria servlet-name o url-pattern
  </filter-mapping>
  <servlet-mapping>
    <servlet-name>CustomServletName</servlet-name>
    <url-pattern>/ baseUrl</url-pattern> // Error: URI-pattern no debe contener espacios en blanco
  </servlet-mapping>
</web-app>
