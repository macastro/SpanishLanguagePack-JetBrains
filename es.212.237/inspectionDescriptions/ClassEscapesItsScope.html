Reporta los usos de clases en una firma de campo o método donde la clase tiene menos visibilidad que el miembro que la usa. Aunque es Java legal, dichos miembros no se pueden usar fuera del alcance de visibilidad del tipo de clase al que hacen referencia.

Ejemplo:


  public class Padre {
    public Hijo obtenerHijo() {
      return new Hijo();
    }

    private class Hijo {}
  }

Además, en Java 9 y superior, un módulo puede ocultar algunas de sus clases a otros módulos al no exportar sus paquetes. Sin embargo, si un miembro que es parte de la API exportada hace referencia a una clase no exportada en su firma, ese miembro no se puede utilizar fuera del módulo.

Configurar la inspección:

  • Use la opción Reportar clases no exportadas expuestas en la API del módulo (Java 9+) para informar a los miembros de la API del módulo que exponen clases no exportadas.
    Tenga en cuenta que el nivel de lenguaje del proyecto o módulo debe ser 9 o superior para esta opción.
  • Utilice la opción Reportar clases inaccesibles expuestas en la API pública para informar sobre miembros públicos que exponen clases con un alcance de visibilidad menor.
  • Utilice la opción Reportar clases privadas expuestas en la API de paquete local para informar sobre miembros locales del paquete que exponen clases private.
