Informa sobre sentencias y expresiones switch con muy pocas etiquetas case, y sugiere reescribirlas como sentencias if y else if.

Ejemplo (mínimo de ramas == 3):


  switch (expresión) {
    case "foo" -> foo();
    case "bar" -> bar();
  }

Después de aplicar la corrección rápida:


  if ("foo".equals(expresión)) {
    foo();
  } else if ("bar".equals(expresión)) {
    bar();
  }

No se informa sobre expresiones switch exhaustivas (Java 14+) o sentencias switch de patrones (vista previa de Java 17) sin la rama 'default'. Esto se debe a que la comprobación de exhaustividad en tiempo de compilación se perderá cuando el switch se convierta en if, lo que podría no ser deseable.

Configurar la inspección:

Utiliza el campo Número mínimo de ramas para especificar el número mínimo esperado de etiquetas case.

Utiliza la opción No informar sobre sentencias de switch de patrones para evitar informar sobre sentencias y expresiones switch que tengan ramas de patrones. Por ejemplo:


  String resultado = switch(obj) {
    case String str -> str.trim();
    default -> "ninguno";
  };

Puede ser preferible mantener el switch incluso con una sola rama de patrón, en lugar de usar la sentencia instanceof.
