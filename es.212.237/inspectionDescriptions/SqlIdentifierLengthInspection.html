너무 긴 식별자를 보고합니다. 길이 제한은 데이터베이스에 따라 다릅니다.

예(MySQL):

CREATE TABLE ooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo
(
   ooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo int
);

MySQL에서 식별자는 64자를 초과할 수 없습니다.

각기 다른 데이터베이스에 대한 다음 제한을 사용해 보세요.

  • Sybase ASE:

    • 따옴표로 묶인 이름: 253자
    • 따옴표로 묶이지 않은 이름: 255자

  • Db2 LUW:

    • BUFFER POOL, INDEX EXTENSION, TABLESPACE, TRANSFORM, TYPE MAPPING: 18자
    • 기타 객체: 128자
  • Db2 iSeries: 128

  • Db2 z/OS:

    • DATABASE, TABLESPACE: 8자
    • WITH RETURN이 사용된 열 및 커서: 30자
    • 기타 객체: 128자
  • Derby: 128
  • Exasol: 128
  • HSQLDB: 128

  • MySQL:

    • 쿼리 내 별칭: 256자
    • USER, ROLE: 32자
    • 기타 객체: 64자

  • MariaDB:

    • USER: 80자
    • ROLE: 128자
    • 다른 객체: MySQL과 같음
  • Microsoft SQL Server: 128자
  • Vertica: 128자
  • SQL 92: 128자
  • Snowflake: 255자
  • Redshift: 127자
  • PostgreSQL/Greenplum/Cockroach: 63자

  • Oracle:

    • 긴 이름이 활성화됩니다(database.oracle.names.long이 지정되어 있으며, Oracle 버전이 12.2 이전 혹은 알 수 없는 경우).

      • DISKGROUP, ROLLBACK SEGMENT, TABLESPACE, PLUGGABLE DATABASE: 30자
      • DATABASE: 8자
      • 기타 객체: 128자
    • 긴 이름이 비활성화됩니다:
      • DATABASE: 8자
      • DATABASE LINK: 128자
      • 기타 객체: 30자
