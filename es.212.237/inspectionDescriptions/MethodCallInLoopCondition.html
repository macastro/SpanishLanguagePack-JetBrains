Informa de llamadas de método en la parte de condición de una instrucción de bucle. En entornos con muchos recursos limitados, dichas llamadas pueden tener implicaciones adversas para el rendimiento.

La aplicación sin consideración de los resultados de esta inspección podría tener efectos negativos en la claridad y el diseño del código. Esta inspección está pensada para Java ME y otros entornos con muchos recursos limitados.

Ejemplo:


  String s = "example";
  for (int i = 0; i < s.length(); i++) {
    System.out.println(s.charAt(i));
  }

Tras aplicar la corrección rápida:


  String s = "example";
  int length = s.length();
  for (int i = 0; i < length; i++) {
    System.out.println(s.charAt(i));
  }

Utilice esta opción para ignorar las llamadas a métodos de iteración comunes de Java, como Iterator.hasNext(), y los métodos conocidos con efectos secundarios, como Atomic*.compareAndSet.
