Informa sobre clases internas no estáticas que implementan java.io.Serializable, pero no definen un campo serialVersionUID.

Sin un campo serialVersionUID, cualquier cambio en la clase hará que las versiones serializadas anteriormente sean ilegibles. Se recomienda encarecidamente que las clases internas no estáticas Serializable tengan un campo serialVersionUID; de lo contrario, el algoritmo de serialización predeterminado puede provocar que las versiones serializadas sean incompatibles entre compiladores debido a diferencias en los métodos de acceso sintéticos.

Se sugiere una solución rápida para agregar el campo serialVersionUID faltante.

Ejemplo:


  class Outer {
      class Inner implements Serializable {}
  }

Después de que se aplica la solución rápida:


  class Outer {
      class Inner implements Serializable {
          private static final long serialVersionUID = -7004458730436243902L;
      }
  }

Utiliza las siguientes opciones para configurar la inspección:

  • Lista las clases cuyos herederos no deben ser reportados por esta inspección. Esto es para clases que heredan Serializable de una superclase pero que no están destinadas a serialización.
  • Si ignorar las clases anónimas Serializable.
