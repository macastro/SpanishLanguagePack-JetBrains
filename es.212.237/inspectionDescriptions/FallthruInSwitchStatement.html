Informa de 'caída' en una declaración switch.

La caída se produce cuando no se garantiza que una serie de declaraciones ejecutables después de una etiqueta case transfieran el control antes de la siguiente etiqueta case. Por ejemplo, esto puede suceder si a la bifurcación le falta una declaración break. En ese caso, el control cae a través de las declaraciones después de esa etiqueta switch, incluso si la expresión switch no es igual a el valor de la etiqueta caída. Aunque a veces sea intencionado, esta construcción es confusa y a menudo es el resultado de una errata.

Esta inspección ignora cualquier caída comentada con un texto que coincida con el patrón de regex (?i)falls?\s*thro?u.

Hay una corrección que agrega un break a la bifurcación que puede caer a la siguiente bifurcación.

Ejemplo:


    switch(x) {
      case (4):
          if (condition) {
              System.out.println("3");
              // no break here
          } else {
              break;
          }
      case (6):
          System.out.println("4");
    }

Después de aplicar la corrección rápida:


    switch(x) {
        case (4):
            if (condition) {
                System.out.println("3");
            } else {
                break;
            }
            break;
        case (6):
            System.out.println("4");
    }
