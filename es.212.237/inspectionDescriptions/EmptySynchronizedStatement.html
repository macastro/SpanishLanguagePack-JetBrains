Informa sobre sentencias synchronized con cuerpos vacíos.

Las sentencias synchronized vacías a veces se utilizan para esperar a que otros hilos liberen un recurso determinado. Sin embargo, no hay garantía de que el mismo recurso no se adquiera de nuevo justo tras finalizar la sentencia synchronized vacía. Para una sincronización adecuada, el recurso debe utilizarse dentro del bloque synchronized.

Además, puede aparecer un bloque synchronized vacío tras una refactorización cuando se eliminó código redundante. En este caso, el propio bloque synchronized será redundante y deberá eliminarse también.

Ejemplo:


  synchronized(lock) {}

Se sugiere una solución rápida para eliminar la sentencia synchronized vacía.
