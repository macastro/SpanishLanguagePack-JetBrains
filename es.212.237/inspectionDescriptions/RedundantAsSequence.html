Informa de una llamada redundante a asSequence() que nunca puede tener un efecto positivo en el rendimiento.

asSequence() acelera el procesamiento de una colección que incluye múltiples operaciones porque ejecuta las operaciones con pereza y no crea colecciones intermedias.

Sin embargo, si se utiliza una operación terminal (como toList()) justo después de asSequence(), no se obtiene ningún efecto positivo en el rendimiento.

Ejemplo:


  fun test(list: List<String>) {
      list.asSequence().last()
  }

Después de aplicar la solución rápida:


  fun test(list: List<String>) {
      list.last()
  }
