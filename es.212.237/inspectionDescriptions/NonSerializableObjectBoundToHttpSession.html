Informa sobre objetos de clases que no implementan java.io.Serializable usados como argumentos de javax.servlet.http.HttpSession.setAttribute() o javax.servlet.http.HttpSession.putValue().

Dichos objetos no se serializarán si HttpSession se pasiva o migra, y podrían provocar errores difíciles de diagnosticar.

Esta inspección asume que los objetos de los tipos java.util.Collection y java.util.Map son Serializable, a menos que los parámetros de tipo no sean Serializable.

Ejemplo:


  void foo(HttpSession session) {
      session.setAttribute("foo", new NonSerializable());
  }
  static class NonSerializable {}
