Se reportan declaraciones cuyos modificadores no están en el orden canónico preferido (como se indica en la especificación del lenguaje Java).

Ejemplo:


  class Foo {
    native public final void foo();
  }

Después de aplicar la solución rápida:


  class Foo {
    public final native void foo();
  }

Usa la configuración de inspección para:

  • alternar el informe de anotaciones fuera de lugar: (anotaciones con ElementType.TYPE_USE no directamente antes del tipo y después de las palabras clave del modificador, o otras anotaciones no antes de las palabras clave del modificador). Cuando esta opción está desactivada, cualquier anotación puede colocarse antes o después de las palabras clave del modificador. Las listas de modificadores con anotaciones entre las palabras clave del modificador siempre se informarán.

  • especifica si la anotación ElementType.TYPE_USE debe colocarse directamente antes del tipo, incluso cuando la anotación tiene otros destinos especificados.
