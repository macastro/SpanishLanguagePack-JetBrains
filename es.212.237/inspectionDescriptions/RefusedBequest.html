Reporta métodos que sobrescriben un super método sin llamarlo. Esto es también conocido como legado rechazado. Esos métodos pueden representar una falla de abstracción y causar bugs difícil de rastrear.

La inspección no reporta métodos sobrescritos de java.lang.Object, a excepción de clone(). El método clone() debería por convención llamar a su super método, que devolverá un objeto del tipo correcto.

Ejemplo 1:


  public class A {
    @Override
    public Object clone() {
      // no llama a 'super.clone()'
      return new A();
    }
  }

Ejemplo 2:


  public interface I {
    default void foo() {}
  }

  public class A implements I {
    // advertencia en el método cuándo
    // está deshabilitado 'Ignorar métodos super 'default''
    @Override
    public void foo(){}
  }

Configura la inspección:

  • Usa la opción Reportar solo cuándo el método super esté anotado por para ignorar métodos super marcados con las anotaciones de la lista provista. Puedes añadir anotaciones a la lista manualmente.
  • Usa la opción Ignorar métodos super vacíos para ignorar métodos super que están vacíos o solo lanzan una excepción.
  • Usa la opción Ignorar métodos super 'default' para ignorar métodos super default de interfaces.
