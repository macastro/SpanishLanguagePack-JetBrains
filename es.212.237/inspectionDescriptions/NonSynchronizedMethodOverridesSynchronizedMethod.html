Informa sobre métodos no synchronized que reemplazan a métodos synchronized.

El método reemplazado no se sincronizará automáticamente si el método de la superclase se declara como synchronized. Esto puede generar condiciones de carrera inesperadas al utilizar la subclase.

Ejemplo:


  class Super {
    synchronized void process() {}
  }
  class Sub extends Super {
    // El método no sincronizado 'process()' reemplaza al método sincronizado
    void process() {}
  }
