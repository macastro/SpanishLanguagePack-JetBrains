Informa el innecesario calificador this

El uso de this para desambiguación de una referencia de código es desaconsejado por muchos estilos de codificación y puede fácilmente volverse innecesario mediante refactorizaciones automáticas.

Ejemplo:


  class Foo {
    int x;
    void foo() {
      this.x = 2;
    }
  }

Después de aplicar el arreglo rápido:


  class Foo {
    int x;
    void foo() {
      x = 2;
    }
  }

Usa las configuraciones de inspección para ignorar asignaciones a campos. Por ejemplo, this.x = 2; no será reportado, pero int y = this.x; sí lo será.
