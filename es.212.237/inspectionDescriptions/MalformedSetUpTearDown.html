public으로 선언되어 있거나 void를 반환하거나 인수를 취하는 JUnit 3 setUp() 또는 tearDown() 메서드를 보고합니다. 그러한 setUp() 또는 tearDown() 메서드는 의도치 않게 생성된 것일 가능성이 높으며 JUnit 테스트 러너에 의해 실행되지 않습니다.

예:


  public class WeightyTest extends TestCase {
    private File file;
    private int setUp(String prefix) throws IOException {
      file = File.createTempFile(prefix, ".tmp");
      return 0;
    }

    // ... 테스트가 여기에 도달합니다
  }
