Informa de recursos JDBC que no están cerrados correctamente. Los recursos JDBC informados por esta inspección incluyen java.sql.Connection, java.sql.Statement, java.sql.PreparedStatement, java.sql.CallableStatement y java.sql.ResultSet.

De manera predeterminada, la inspección asume que los recursos se pueden cerrar mediante cualquier método con 'close' o 'cleanup' en su nombre.

Ejemplo:


  ResultSet findAllElements(Connection connection) throws SQLException {
    PreparedStatement statement = connection.prepareStatement("SELECT * FROM TABLE"); // La declaración no está cerrada
    statement.execute();
    return statement.getResultSet();
  }

Utiliza las siguientes opciones para configurar la inspección:

  • Si se permite abrir un recurso JDBC dentro de un bloque try. Este estilo es menos deseable porque es más detallado que abrir un recurso delante de un bloque try.
  • Si el recurso se puede cerrar mediante cualquier llamada a un método con el recurso pasado como argumento.
