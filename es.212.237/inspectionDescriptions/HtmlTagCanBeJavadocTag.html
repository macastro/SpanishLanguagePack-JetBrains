Reporta los usos de las etiquetas <code> en los comentarios Javadoc. Desde Java 5, estas etiquetas pueden ser remplazadas con construcciones {@code ...}. Esto permite usar los signos de menor que < y mayor que > dentro del comentario en lugar de entidades de carácter HTML.

Ejemplo:


  /**
   * @return empty <code>ArrayList&lt;Integer&gt;</code>
   */
  List<Integer> getList(){ ... }

Después de que la corrección rápida es aplicada:


  /**
   * @return empty {@code ArrayList<Integer>}
   */
  List<Integer> getList(){ ... }
