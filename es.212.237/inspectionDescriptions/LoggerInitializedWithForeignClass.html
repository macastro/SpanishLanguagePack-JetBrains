Informa acerca de instancias Logger que se inicializan con un literal de class de una clase distinta de aquella en la que se contiene Logger. Esto puede ocurrir fácilmente cuando se copia y pega algún código de otra clase y puede resultar en el registro de eventos bajo una categoría inesperada y provocar que los filtros se apliquen incorrectamente.

Se proporciona una solución rápida para reemplazar el literal de clase externo por uno de la clase circundante.

Ejemplo:


  public class Paramount {
    protected static final Logger LOG = Logger.getLogger(Critical.class);

    // ... otros campos y métodos
  }

Después de aplicar la solución rápida:


  public class Paramount {
    protected static final Logger LOG = Logger.getLogger(Paramount.class);

    // ... otros campos y métodos
  }

Configurar la inspección:

  • Utiliza la tabla para especificar las clases y métodos de fábrica de registradores reconocidos por esta inspección.
  • Utiliza la opción Ignorar registradores inicializados con una superclase para ignorar los registradores que se inicializan con una superclase de la clase que contiene el registrador.
  • Utiliza Ignorar registradores en clases no públicas para advertir solo sobre los registradores en clases public.
