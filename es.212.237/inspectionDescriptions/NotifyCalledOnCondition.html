Informa de las llamadas a notify() o notifyAll() que se realizan en un objeto java.util.concurrent.locks.Condition.

Esto probablemente sea un error de programación y es posible que se pretendiera alguna variante del método signal() o signalAll(), ya que, de lo contrario, se podría producir IllegalMonitorStateException.

Ejemplo:


  class C {
    final Lock l = new ReentrantLock();
    final Condition c = l.newCondition();

    void release() {
      l.lock();
      try {
        c.notifyAll(); // probablemente se pretendía 'signalAll()' aquí
      } finally {
        l.unlock();
      }
    }
  }
