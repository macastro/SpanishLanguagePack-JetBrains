Los informes reflejan el acceso a campos y métodos que no existen o no son visibles.

Ejemplo:


  Field stringHashField() throws NoSuchFieldException {
    return String.class.getField("hash");
  }

Después de que se aplica la corrección rápida:


  Field stringHashField() throws NoSuchFieldException {
    return String.class.getDeclaredField("hash");
  }

Con una clase final, está claro si hay un campo o método con el nombre especificado en la clase.

Con clases que no son final, es posible que una subclase tenga un campo o método con ese nombre, por lo que podría haber falsos positivos. Utiliza la configuración de la inspección para deshacerte de dichos falsos positivos en todas partes o con clases específicas.
