Informa una anotación de tipo que no coincide con el estilo de código actual para tipos explícitos.

Las declaraciones de tipo no son necesarias cuando el tipo que se infiere del contexto coincide exactamente con la anotación de tipo, por ejemplo: 


    var pi: number = 3.14

En algunos casos es preferible tener siempre tipos explícitos - esto evita cambios accidentales de tipo y hace que el código sea más explícito.
