Reporta punto y coma (;) redundantes que pueden eliminarse de forma segura.

Kotlin no requiere un punto y coma al final de cada sentencia o expresión. Se sugiere una corrección rápida para eliminar los puntos y coma redundantes.

Ejemplo:


  val myMap = mapOf("one" to 1, "two" to 2);
  myMap.forEach { (key, value) ->  print("$key -> $value")};

Después de aplicar la corrección rápida:


  val myMap = mapOf("one" to 1, "two" to 2)
  myMap.forEach { (key, value) ->  print("$key -> $value")}

Sin embargo, hay dos casos en los que se requiere un punto y coma:

  1. Varias sentencias colocadas en una sola línea deben separarse con puntos y coma: 
    
  map.forEach { val (key, value) = it; println("$key -> $value") }
  2. Las clases enum que también declaran propiedades o funciones requieren un punto y coma después de la lista de constantes de la enumeración: 
    
  enum class Mode {
      SILENT, VERBOSE;

      fun isSilent(): Boolean = this == SILENT
  }
