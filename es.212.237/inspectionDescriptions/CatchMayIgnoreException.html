Informa de bloques catch vacíos o que pueden ignorar una excepción.

Si bien en ocasiones es intencionado, los bloques catch pueden complicar la depuración. Además, ignorar un parámetro catch podría ser incorrecto. Finalmente, el analizador de código estático informa si detecta que un bloque catch puede ignorar silenciosamente excepciones importantes de VM como NullPointerException. Ignorar dicha excepción (sin registrarla o volver a lanzarla) puede ocultar un error.

La inspección no informará sobre ningún parámetro catch llamado ignore o ignored. Por el contrario, la inspección le advertirá sobre cualquier parámetro catch llamado ignore o ignored que esté realmente en uso. Además, la inspección no informará sobre los parámetros catch dentro de las fuentes de prueba denominadas expected u ok.

Puede utilizar una solución rápida para cambiar el nombre de la excepción a ignored. Para los bloques catch vacíos, se sugiere una corrección rápida adicional para generar el cuerpo catch. Puede modificar la plantilla "Cuerpo de la declaración Catch" en la pestaña Código en Configuración | Editor | Plantillas de código y archivo.

Ejemplo:


  try {
    throwingMethod();
  } catch (IOException ex) {

  }

Después de aplicar la solución rápida:


  try {
    System.out.println(System.in.read());
  } catch (IOException ignored) {

  }

Configure la inspección:

  • Utilice la opción No advertir cuando el bloque "catch" contenga un comentario para ignorar los bloques catch con comentarios.
  • Utilice la opción No advertir cuando el bloque "catch" no esté vacío para ignorar los bloques catch que contienen declaraciones o comentarios en su interior, mientras que la variable en sí no se utiliza.
  • Utilice la opción No advertir cuando la excepción llamada "ignore(d)" no se ignore realmente para ignorar las variables llamadas ignored si están en uso.
