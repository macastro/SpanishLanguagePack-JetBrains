Informa de excepciones que se lanzan desde dentro de bloques catch pero que no "envuelven" la excepción capturada.

Cuando se lanza una excepción en respuesta a una excepción, envolver la excepción inicial impide perder información valiosa del contexto, como marcos de pila y números de línea.

Ejemplo:


  ...
  catch (IOException e) {
    closeAllConnections();
    throw new ConnectException("Connection problem."); // advertencia: 'throw' dentro del bloque 'catch' ignora la excepción capturada 'e'
  }

Configuración de la inspección:

  • Utiliza la opción Ignorar si se utiliza el resultado de la llamada al método de excepción para indicar si la inspección debería ignorar las excepciones cuyo argumento es el resultado de una llamada al método en la excepción original, como getMessage().
  • Utiliza la opción Ignorar si la excepción lanzada no puede envolver una excepción para ignorar las sentencias throw que lanzan excepciones sin un constructor que acepte una causa Throwable.
