Reporta las sentencias return con valores de retorno null. Si bien ocasionalmente son útiles, esta construcción puede hacer que el código sea más propenso a fallar con una NullPointerException.

Si un método está diseñado para devolver null, se sugiere marcarlo con la anotación @Nullable, estos métodos serán ignorados por esta inspección.

Ejemplo:


  class Persona {
    public String getNombre () {
      return null;
    }
  }

Después de aplicar el arreglo rápido:


  class Persona {
    @Nullable
    public String getNombre () {
      return null;
    }
  }

Si el tipo de retorno es java.util.Optional, se sugiere un arreglo rápido adicional para convertir null a Optional.empty().

Usa las siguientes opciones para configurar la inspección:

  • Si se ignoran los métodos 'private'. Esto también ignorará el retorno de null desde clases anónimas y lambdas.
  • Si se deben reportar los valores null en retornos de arreglos, retornos de objetos de colección, retornos de objetos simples o una combinación de los tres. El retorno de null en métodos con tipo de retorno java.util.Optional siempre se reporta.
  • Haz clic en Configurar anotaciones para especificar qué anotaciones deben considerarse 'nullable'.
