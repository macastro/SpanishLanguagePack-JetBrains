Informa conversiones de tipos que son excesivamente fuertes. Por ejemplo, convertir un objeto a ArrayList cuando convertirlo a List funcionaría igual de bien.

Nota: de forma muy parecida a la inspección de Conversión de tipo redundante, aplicar la solución para esta inspección puede cambiar la semántica de su programa si está usando intencionalmente una conversión demasiado fuerte para provocar que se genere una ClassCastException.

Ejemplo:


  interface Super {
    void doSmth();
  }
  interface Sub extends Super { }

  void use(Object obj) {
    // Advertencia: ((Super)obj).doSmth() podría usarse
    ((Sub)obj).doSmth();
  }

Use la casilla de verificación a continuación para ignorar conversiones cuando hay una comprobación instanceof coincidente en el código.
