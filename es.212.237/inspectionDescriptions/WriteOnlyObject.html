Reporta objetos que se modifican pero nunca se consultan.

La inspección se basa en el contrato de mutación del método, que podría inferirse o anotarse previamente para algunos métodos de biblioteca. Esta inspección no informa sobre colecciones, mapas y creadores de cadenas, ya que estos tipos son informados por otras inspecciones más precisas.

Ejemplo:


  AtomicReference<String> ref = new AtomicReference<>();
  ref.set("hello"); // ref nunca se vuelve a utilizar

Utilice la opción **Ignorar constructores impuros** para controlar si se deben procesar objetos creados por el constructor o método cuya pureza no se conoce. Desmarcar la opción puede introducir algunos falsos positivos si la referencia del objeto se filtra intencionalmente durante la construcción.
