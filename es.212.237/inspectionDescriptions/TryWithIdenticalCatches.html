Informa secciones `catch` idénticas dentro de una única sentencia `try`.

Colapsar dichas secciones en un único bloque `multi-catch` puede reducir la duplicación de código y prevenir situaciones en las que una sección `catch` es actualizada pero otra no lo es.

**Ejemplo:**


    try {
        doSmth();
    }
    catch (IOException e) {
        LOG.error(e);
    }
    catch (URISyntaxException e) {
        LOG.error(e);
    }

Se puede utilizar un arreglo rápido para hacer que el código sea más conciso:


    try {
        doSmth();
    }
    catch (IOException | URISyntaxException e) {
        LOG.error(e);
    }
