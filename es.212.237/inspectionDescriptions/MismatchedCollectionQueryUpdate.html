Reporta colecciones cuyo contenido es consultado y no actualizado, o actualizado y no consultado.

Tales consultas y actualizaciones inconsistentes no tienen sentido y pueden indicar código muerto o un error tipográfico.

Utiliza las configuraciones de inspección para especificar patrones de nombre que correspondan a métodos de actualización/consulta. Los métodos de consulta que retornan un elemento son detectados automáticamente y solo aquellos que escriben datos en un parámetro de salida (por ejemplo, un OutputStream) necesitan ser especificados.

Ejemplo:

Supón que tienes tu clase FixedStack personalizada con método store(): 


  public class FixedStack<T> extends Collection<T> {
    public T store(T t) {
      // implementación
    }
  }

Puedes añadir store a la tabla de métodos de actualización para reportar consultas no coincidentes como: 


  void test(int i) {
    FixedStack<Integer> stack = new FixedStack<>();
    stack.store(i);
  }
