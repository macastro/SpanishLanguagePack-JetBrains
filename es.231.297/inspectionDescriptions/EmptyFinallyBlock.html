Reporta los bloques vacíos finally.

Los bloques vacíos finally en general indican errores de programación. También pueden permanecer tras la refactorización del código y pueden eliminarse sin peligro.

Esta inspección no reporta los bloques vacíos finally hallados en los archivos JSP.

Ejemplo:


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  } finally {

  }

Tras aplicar la corrección rápida: 


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  }
