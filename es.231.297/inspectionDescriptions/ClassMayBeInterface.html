Informa de clases abstract que pueden convertirse en interfaces.

Es preferible utilizar interfaces en lugar de clases puesto que Java no soporta la herencia múltiple de clases, mientras que una clase puede implementar múltiples interfaces.

Una clase puede convertirse en una interfaz si no tiene superclases (aparte de Object), sólo tiene campos public static final, métodos public abstract y clases internas public.

Ejemplo:


clase abstracta Ejemplo {
    public static final int MI_CONSTANTE = 42;
    public abstract void foo();
}

clase Heredera extends Ejemplo {
    @Override
    public void foo() {
        System.out.println(MI_CONSTANTE);
    }
}

Después de aplicar la corrección rápida:


interfaz Ejemplo {
    int MI_CONSTANTE = 42;
    void foo();
}

clase Heredera implementa Ejemplo {
    @Override
    public void foo() {
        System.out.println(MI_CONSTANTE);
    }
}

Configurar la inspección:

Usa la opción Informar de clases que contienen métodos no abstractos al usar Java 8 para informar sólo sobre las clases con métodos static y métodos no abstractos que se pueden convertir en métodos default (sólo aplicable a nivel de lenguaje 8 o superior).
