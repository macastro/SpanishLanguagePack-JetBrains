Reporta usos de instanceof o getClass() == SomeClass.class en los que se comprueba una expresión this.

Estas expresiones indican que el diseño orientado a objetos es deficiente y deben reemplazarse por construcciones polimórficas.

Ejemplo:


class Super {
  void process() {
    if (this instanceof Sub) { // warning
      doSomething();
    } else {
      doSomethingElse();
    }
  }
}
  
class Sub extends Super {}

Para arreglar el problema, usa un método sobreescrito:


class Super {
  void process() {
    doSomethingElse();
  }
}
  
class Sub extends Super {
  @Override
  void process() {
    doSomething();
  }
}
