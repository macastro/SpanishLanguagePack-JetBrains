Reporta comentarios que no comienzan con el nombre del elemento exportado.

De acuerdo con Comment Sentences en github.com/golang, esta es una convención para empezar un comentario con el nombre del elemento exportado.

Ejemplo:

// representa una solicitud para ejecutar un comando.
type Request struct {}

El comentario comienza con la descripción de la estructura, no con el nombre de la estructura. Para cumplir con las reglas de la convención, puedes aplicar el quick-fix Añadir prefijo al comentario. Después de aplicar el quick-fix, el comentario se ve como sigue:

// Request representa una solicitud para ejecutar un comando.
type Request struct {} // mejor
