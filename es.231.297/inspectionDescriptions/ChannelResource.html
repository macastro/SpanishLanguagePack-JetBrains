Reporta recursos Channel que no se cierran de forma segura, incluyendo cualquier instancia creada llamando getChannel() en un archivo o recurso de socket.

Por defecto, la inspección asume que los recursos pueden cerrarse por cualquier método con 'close' o 'cleanup' en su nombre.

Ejemplo:


  void send(Socket socket) throws IOException {
    SocketChannel channel = socket.getChannel(); //advertencia
    channel.write(ByteBuffer.wrap("message".getBytes()));
  }

Usa las siguientes opciones para configurar la inspección:

  • Si se permite que un recurso Channel se abra dentro de un bloque try Este estilo es menos deseable porque es más verboso que abrir un Channel en frente de un bloque try.
  • Si el recurso puede cerrarse por cualquier llamada de método con el recurso pasado como argumento.
