(Gradle 4.9+) Detecta el uso de API que interactúa con tasks de forma rápida (eagerly)

La interacción rápida con tasks implica algunos inconvenientes:

  • El usuario debe establecer manualmente todas las dependencias entre tasks;
  • En la fase de configuración, todos los tasks a los que se accede a través de la API rápida se configuran, incluso si no se ejecutan después. Esto deriva en una degradación del rendimiento.

Finalmente, la API rápida quedará obsoleta a favor de la lenta (lazy).

Para una guía de migración, consulta la documentación de Gradle.

Ejemplo:

task foo { // Informa de 'task', sugiere reemplazarlo con 'task.register'
  // ...
}
