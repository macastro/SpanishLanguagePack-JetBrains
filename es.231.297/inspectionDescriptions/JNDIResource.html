Informa sobre recursos JNDI que no están cerrados de forma segura. Los recursos JNDI notificados por esta inspección incluyen javax.naming.InitialContext y javax.naming.NamingEnumeration.

Por defecto, la inspección supone que los recursos pueden ser cerrados mediante cualquier método cuyo nombre tenga "close" o "cleanup".

Ejemplo:


  Object findObject(Properties properties, String name) throws NamingException {
    Context context = new InitialContext(properties); //el contexto no se cierra
    return context.lookup(name);
  }

Utilice las siguientes opciones para configurar la inspección:

  • Si un recurso JNDI puede abrirse dentro de un bloque try. Este estilo es menos deseable porque es más detallado que abrir un recurso delante de un bloque try.
  • Si el recurso puede ser cerrado mediante cualquier llamada a método con el recurso pasado como argumento.
