Esta inspección revisa las llamadas a API de Android con varias anotaciones auxiliares (como RequiresPermission o UiThread) y marca todas las llamadas que no usan correctamente la API según la especificación de la anotación.

Ejemplos de errores marcados por esta inspección:
  • Pasar un tipo incorrecto de entero de recurso (como R.string) a una API que requiere un tipo distinto (como R.dimen)
  • Olvidar llamar a un método anulado en el método que lo requiere (a través de super)
  • Llamar a un método que requiere un permiso sin declarar ese permiso en el manifiesto
  • Pasar una referencia de color de recurso a un método que requiere un valor de entero RGB.
...y muchos más.

Para obtener más información, consulta la documentación en http://developer.android.com/tools/debugging/annotations.html.
