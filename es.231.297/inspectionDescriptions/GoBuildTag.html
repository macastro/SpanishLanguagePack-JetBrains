Reporta etiquetas de compilación mal formadas y etiquetas de compilación en la ubicación incorrecta. La herramienta go espera que las etiquetas de compilación estén ubicadas en lugares particulares y que sigan una sintaxis especial. Si no se siguen estos requisitos, las etiquetas de compilación podrían ignorarse o los archivos podrían excluirse incorrectamente de la compilación.

Consulta Restricciones de compilación en go.dev.

Ejemplo:

package main

// +build ignore

func main() {}

La parte // +build ignore debe estar antes de la declaración del paquete. Para corregir eso, puedes aplicar el asistente rápido Colocar etiqueta de compilación antes del paquete. Después de aplicar el asistente rápido:

// +build ignore

package main

import "fmt"
