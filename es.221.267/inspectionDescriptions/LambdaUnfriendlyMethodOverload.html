Informa acerca de métodos sobrecargados que adoptan interfaces funcionales con firmas de método abstracto conflictivas.

Tales sobrecargas presentan ambigüedad y requieren que las personas que llaman conviertan lambdas a un tipo específico o especifiquen tipos de parámetros lambda explícitamente. Es preferible dar a los métodos sobrecargados nombres diferentes para eliminar la ambigüedad.

Ejemplo:


  interface MyExecutor {
    void execute(Supplier<?> supplier);
    void execute(Callable<?> callable);
  }

Aquí, Supplier y Callable son interfaces funcionales cuyos únicos métodos abstractos no adoptan parámetros y devuelven un valor no nulo. Como resultado, el tipo de lambda no se puede inferir en el sitio de llamada a menos que se use una conversión explícita.
