Informa de los métodos y constructores en los que una constante de String literal de juego de caracteres (por ejemplo, "UTF-8") puede reemplazarse por un código predeterminado de StandardCharsets.UTF_8.

El código después de la corrección puede funcionar más rápido, porque la búsqueda de juego de caracteres se vuelve innecesaria. De igual modo, es posible que dejar de capturar UnsupportedEncodingException sea innecesario también. En este caso, el bloque catch se eliminará automáticamente.

Ejemplo:


  try {
    byte[] bytes = "str".getBytes("UTF-8");
  } catch (UnsupportedEncodingException e) {
  }

Después de aplicar una corrección rápida:


  byte[] bytes = "str".getBytes(StandardCharsets.UTF_8);

La inspección está disponible en Java 7 y versiones posteriores.
