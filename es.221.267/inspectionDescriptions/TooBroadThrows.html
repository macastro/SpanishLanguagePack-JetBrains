Informa cláusulas throws con excepciones que son más genéricas que las excepciones que el método lanza realmente.

Ejemplo:


  public void createFile() throws Exception { // advertencia: 'throws Exception' es demasiado amplia y enmascara la excepción 'IOException'
    File file = new File("pathToFile");
    file.createNewFile();
  }

Después de aplicar la corrección rápida:


  public void createFile() throws IOException {
    File file = new File("pathToFile");
    file.createNewFile();
  }

Configura la inspección:

  • Utiliza el campo Número máximo de excepciones ocultas para advertir para ignorar excepciones que ocultan un número de excepciones más amplio que el especificado.
  • Utiliza la opción Solo advertir en RuntimeException, Exception, Error o Throwable para que esta inspección avise solo en las excepciones más genéricas.
  • Utiliza la opción Ignorar excepciones declaradas en métodos que sobrescriben un método de biblioteca para ignorar cláusulas throws demasiado amplias en métodos que sobrescriben un método de biblioteca.
  • Utiliza la opción Ignorar excepciones que ocultan otras pero se lanzan ellas mismas para ignorar excepciones que ocultan otras excepciones pero que todavía se pueden lanzar desde el cuerpo del método y, por lo tanto, técnicamente no son demasiado amplias.
