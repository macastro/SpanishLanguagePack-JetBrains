Informa sobre las llamadas a ciertas funciones y métodos que no manejan el resultado de la llamada.

Una API de dichas funciones implica que los usuarios deberían llamarlas principalmente para obtener un valor de retorna y procesarlo, no para efectos secundarios. Las llamadas que no manejan el resultado podrían ser un indicio de un mal uso de la API.

Ejemplo:

fmt.Errorf("error: %s", reason) // error construido es ignorado

Después de la aplicación de la corrección rápida Introducir variable local:

err := fmt.Errorf("error: %s", reason)
