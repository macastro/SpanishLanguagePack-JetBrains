Se informa del uso de sameElements y corresponds con colecciones sin ordenar.

Los métodos sameElements y corresponds utilizan iteradores de las colecciones proporcionadas para comparar elementos en función del orden de su posición en la colección. Por lo tanto, el resultado es impredecible si el orden de los elementos no está bien definido, como es el caso de mapas o conjuntos desordenados.

Para corregir esto, considera utilizar == o subsetOf, o bien ordenar los elementos en algún orden.

Ejemplo:


  Set(3,2,1) sameElements Set(1, 2, 3) // es falso

  // pero

  Set(1, 2, 3, 4, 5) sameElements Set(5, 4, 3, 2, 1) // es verdadero
