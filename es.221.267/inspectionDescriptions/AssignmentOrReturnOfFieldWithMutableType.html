Reporta el retorno o la asignación desde un parámetro de método a una matriz o a un tipo mutable como Collection, Date, Map, Calendar, etc.

Debido a que tales tipos son mutables, esta construcción puede resultar en modificaciones inesperadas del estado del objeto desde fuera de la clase propietaria. Aunque esta construcción puede ser útil por razones de rendimiento, es inherentemente propensa a errores.

Se informan los siguientes tipos mutables:

  • java.util.Date
  • java.util.Calendar
  • java.util.Collection
  • java.util.Map
  • com.google.common.collect.Multimap
  • com.google.common.collect.Table

La solución rápida agrega una llamada al método .clone() del campo.

Ejemplo:


  class Registro {
    String[] mensajes;
    ...

    String[] getMensajes() {
      return mensajes; // advertencia: Retorno del campo String[] 'mensajes'
    }
  }

Después de aplicar la solución rápida:


  class Registro {
    String[] mensajes;
    ...

    String[] getMensajes() {
      return mensajes.clone();
    }
  }

Utiliza la opción Ignorar asignaciones y retornos en métodos privados para ignorar las asignaciones y retornos en métodos private.
