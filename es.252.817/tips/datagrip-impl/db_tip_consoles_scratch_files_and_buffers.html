Consolas de bases de datos, archivos scratch y búferes scratch

Consolas de bases de datos son archivos SQL que ya están vinculados a una fuente de datos. Cuando se crea una fuente de datos, se crea automáticamente una consola de base de datos. Para abrir la consola, hacer clic con el botón derecho en la fuente de datos y seleccionar Navegación | Ir a la consola de consultas | Abrir consola predeterminada. Para crear una nueva consola, hacer clic con el botón derecho en la fuente de datos y seleccionar Nuevo | Consola de consultas.

Archivos scratch son archivos SQL que no están vinculados a una fuente de datos. Se utilizan como notas temporales o borradores de algunas ideas de código fuera del contexto del proyecto. Para crear un archivo scratch, navegar a Archivo | Nuevo | Archivo scratch (o pulsar &shortcut:NewScratchFile;).

Búferes scratch son archivos de texto simples sin ninguna función de asistencia de codificación. Los búferes scratch se pueden utilizar para listas de tareas simples y notas personales. No existe ningún elemento de menú específico para la acción de crear un nuevo búfer de scratch, pero se puede utilizar el popup Buscar acción (&shortcut:GotoAction;) y ejecutar la acción Nuevo búfer scratch.
