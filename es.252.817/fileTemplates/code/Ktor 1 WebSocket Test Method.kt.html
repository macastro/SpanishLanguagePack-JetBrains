Este es un template para crear métodos de prueba de WebSocket para Ktor 1.x.x en una clase de prueba de JUnit 4.
Las variables predefinidas aceptarán los siguientes valores:
${NAME} El nombre del método creado
${ROUTE} La ruta relativa al servicio actual que necesita ser probado
${CONFIGURE_SERVER} Un método de extensión de ruta para configurar el árbol de enrutamiento de la parte del servidor que necesita ser probada
${BODY} El cuerpo del método generado
