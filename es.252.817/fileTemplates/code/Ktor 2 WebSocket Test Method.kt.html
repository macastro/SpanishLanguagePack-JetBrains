Esta es una plantilla para crear métodos de prueba WebSocket en clases de prueba JUnit 4 para Ktor 2.x.x.
Las variables predefinidas aceptarán los siguientes valores:
${NAME}   Nombre del método creado
${ROUTE}   Ruta relativa al servicio actual que necesita ser probado
${CONFIGURE_SERVER}   Un método de extensión de ruta para configurar el árbol de rutas para la parte del servidor que necesita ser probada
${BODY}   Cuerpo del método generado
