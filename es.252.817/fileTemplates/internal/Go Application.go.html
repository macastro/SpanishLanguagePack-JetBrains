Go Application es una plantilla integrada que se utiliza cada vez que se crea un nuevo archivo Go. A diferencia de la plantilla Go File, Go Application tiene una función main predefinida. Para utilizar esta plantilla, vaya a Archivo | Nuevo, seleccione Go File en la categoría Go, y luego seleccione Simple Application.

Esta plantilla es editable. En la plantilla, puede utilizar las variables predefinidas que se enumeran a continuación. Estas variables se expandirán posteriormente a sus valores correspondientes en el editor.

También puede especificar variables personalizadas. Las variables personalizadas utilizan el siguiente formato: ${VARIABLE_NAME}, donde VARIABLE_NAME es el nombre de la variable (por ejemplo, ${MY_CUSTOM_FUNCTION_NAME}). Antes de que el IDE cree un nuevo archivo utilizando variables personalizadas, verá un cuadro de diálogo donde puede definir los valores de las variables personalizadas en la plantilla.

Mediante el uso de la directiva #parse, puede incluir plantillas desde la pestaña Include. Para incluir una plantilla, especifique el nombre completo de la plantilla como parámetro entre comillas (por ejemplo, #parse("File Header")).
Lista de variables predefinidas
${GO_PACKAGE_NAME_WITH_TEST_SUFFIX} Nombre del paquete donde se crea el nuevo archivo de prueba. El archivo debe tener el sufijo _test. El nombre utiliza el formato package_test.
${GO_PACKAGE_NAME} Nombre del paquete donde se crea el nuevo archivo.
${DATE} Fecha actual del sistema.
${DAY} Día del mes actual.
${DAY_NAME_SHORT} Las 3 primeras letras del nombre del día actual (por ejemplo, Mon, Tue, etc.).
${DAY_NAME_FULL} Nombre completo del día actual (por ejemplo, Monday, Tuesday, etc.).
${DIR_PATH} La ruta del directorio para el nuevo archivo. Esta ruta es relativa a la raíz del proyecto.
${DS} El signo de dólar ($). Esta variable se utiliza para escapar el carácter de dólar, de modo que no se trate como prefijo de una variable de plantilla.
${FILE_NAME} Nombre del nuevo archivo.
${HOUR} Hora actual.
${MINUTE} Minuto actual.
${SECOND} Segundo actual.
${MONTH} Mes actual.
${MONTH_NAME_SHORT} Las 3 primeras letras del nombre del mes actual (por ejemplo, Jan, Feb, etc.).
${MONTH_NAME_FULL} Nombre completo del mes actual (por ejemplo, January, February, etc.).
${NAME} Nombre de la nueva entidad: archivo, tipo, interfaz, etc.
${PRODUCT_NAME} Nombre del IDE.
${PROJECT_NAME} Nombre del proyecto actual.
${TIME} Hora actual del sistema.
${USER} Nombre de inicio de sesión del usuario actual en el sistema.
${YEAR} Año actual.
