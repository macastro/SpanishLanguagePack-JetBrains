Go Application es una plantilla incorporada que se utiliza cada vez que se crea un nuevo archivo Go. En comparación con la plantilla Go File, Go Application tiene una función main predefinida. Para usar esta plantilla, vaya a Archivo | Nuevo, seleccione Go File en la categoría Go y luego seleccione Aplicación Simple.

La plantilla es editable. En la plantilla, puede usar las variables predefinidas que se enumeran a continuación. Estas variables se expandirán más tarde en el editor a sus valores correspondientes.

También puede especificar variables personalizadas. Las variables personalizadas utilizan el siguiente formato: ${VARIABLE_NAME}, donde VARIABLE_NAME es el nombre de la variable (por ejemplo, ${MY_CUSTOM_FUNCTION_NAME}). Antes de que el IDE cree un nuevo archivo usando variables personalizadas, verá un diálogo donde puede definir los valores para las variables personalizadas en la plantilla.

Usando la directiva #parse, puede incluir plantillas de la pestaña Include. Para incluir una plantilla, especifique su nombre completo como un parámetro entre comillas (por ejemplo, #parse("File Header")).
Lista de variables predefinidas
${GO_PACKAGE_NAME_WITH_TEST_SUFFIX}   El nombre del paquete en el que se crea el nuevo archivo de prueba. El archivo debe tener el sufijo _test. El formato del nombre es: package_test.
${GO_PACKAGE_NAME}   El nombre del paquete en el que se crea el nuevo archivo.
${DATE}   La fecha actual del sistema.
${DAY}   El día del mes actual.
${DAY_NAME_SHORT}   Las primeras 3 letras del nombre del día actual (por ejemplo, Mon, Tue, etc.).
${DAY_NAME_FULL}   El nombre completo del día actual (por ejemplo, Monday, Tuesday, etc.).
${DIR_PATH}   La ruta del directorio del nuevo archivo. Esta ruta es relativa a la raíz del proyecto.
${DS}   El símbolo del dólar ($). Esta variable se usa para escapar el carácter dólar para que no se interprete como prefijo de una variable de plantilla.
${FILE_NAME}   El nombre del nuevo archivo.
${HOUR}   La hora actual.
${MINUTE}   El minuto actual.
${SECOND}   El segundo actual.
${MONTH}   El mes actual.
${MONTH_NAME_SHORT}   Las primeras 3 letras del nombre del mes actual (por ejemplo, Jan, Feb, etc.).
${MONTH_NAME_FULL}   El nombre completo del mes actual (por ejemplo, January, February, etc.).
${NAME}   El nombre de la nueva entidad: archivo, tipo, interfaz, etc.
${PRODUCT_NAME}   El nombre del IDE.
${PROJECT_NAME}   El nombre del proyecto actual.
${TIME}   La hora actual del sistema.
${USER}   El nombre de inicio de sesión del sistema del usuario actual.
${YEAR}   El año actual.
