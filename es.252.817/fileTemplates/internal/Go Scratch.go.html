Go Scratch es una plantilla integrada que se utiliza cada vez que crea un nuevo archivo temporal/borrador (scratch) de Go. Los archivos scratch son archivos totalmente funcionales, ejecutables y depurables que puede utilizar para esbozar sus ideas de codificación. Para crear un archivo scratch, haga clic en Archivo | Nuevo | Archivo scratch. En el cuadro de diálogo Nuevo archivo scratch, busque y seleccione Go.

Esta plantilla es editable. En la plantilla, puede utilizar las variables predefinidas que se enumeran a continuación. Estas variables se expandirán posteriormente a los valores correspondientes en el editor.

También puede especificar variables personalizadas. Las variables personalizadas utilizan el siguiente formato: ${VARIABLE_NAME}, donde VARIABLE_NAME es el nombre de la variable (por ejemplo, ${MY_CUSTOM_FUNCTION_NAME}). Antes de que el IDE cree un nuevo archivo utilizando variables personalizadas, verá un cuadro de diálogo donde podrá definir los valores de las variables personalizadas en la plantilla.

Mediante el uso de la directiva #parse, puede incluir plantillas desde la pestaña Incluir. Para incluir una plantilla, especifique el nombre completo de la plantilla como parámetro entre comillas (por ejemplo, #parse("File Header")).
Lista de variables predefinidas
${GO_PACKAGE_NAME_WITH_TEST_SUFFIX} Nombre del paquete donde se crea el nuevo archivo de prueba. El archivo debe tener el sufijo _test. El formato del nombre es package_test.
${GO_PACKAGE_NAME} Nombre del paquete donde se crea el nuevo archivo.
${DATE} Fecha actual del sistema.
${DAY} Día actual del mes.
${DAY_NAME_SHORT} Las 3 primeras letras del nombre del día actual (por ejemplo, Lun, Mar, etc.).
${DAY_NAME_FULL} Nombre completo del día actual (por ejemplo, Lunes, Martes, etc.).
${DIR_PATH} Ruta del directorio del nuevo archivo. Esta ruta es relativa a la raíz del proyecto.
${DS} Símbolo de dólar ($). Esta variable se utiliza para escapar el carácter de dólar, de modo que no se trate como un prefijo de variable de plantilla.
${FILE_NAME} Nombre del nuevo archivo.
${HOUR} Hora actual.
${MINUTE} Minuto actual.
${SECOND} Segundo actual.
${MONTH} Mes actual.
${MONTH_NAME_SHORT} Las 3 primeras letras del nombre del mes actual (por ejemplo, Ene, Feb, etc.).
${MONTH_NAME_FULL} Nombre completo del mes actual (por ejemplo, Enero, Febrero, etc.).
${NAME} Nombre de la nueva entidad: archivo, tipo, interfaz, etc.
${PRODUCT_NAME} Nombre del IDE.
${PROJECT_NAME} Nombre del proyecto actual.
${TIME} Hora actual del sistema.
${USER} Nombre de inicio de sesión del sistema del usuario actual.
${YEAR} Año actual.
