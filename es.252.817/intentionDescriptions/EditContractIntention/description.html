Esta intención cambia un contrato del método de la biblioteca. El contrato del método tiene la siguiente sintaxis:
contract ::= (clause ';')* clause
clause ::= args '->' effect
args ::= ((arg ',')* arg )?
arg ::= value-constraint
value-constraint ::= '_' | 'null' | '!null' | 'false' | 'true'
effect ::= value-constraint | 'fail' | 'new' | 'this' | 'param' number
number ::= [1-9] [0-9]*

Las restricciones denotan lo siguiente:

  • _ - qualquer valor
  • null - valor nulo
  • !null - un valor estáticamente probado que no es nulo
  • true - valor booleano verdadero
  • false - valor booleano falso
  • fail - el método lanza una excepción, si los argumentos satisfacen las restricciones de argumentos
  • new - cada vez que se ejecuta el método, devuelve un nuevo objeto no nulo que es distinto de otros objetos existentes en el montón antes de la ejecución del método. Si el método es puro, el nuevo objeto no se almacena en ningún campo o matriz y se perderá si no se utiliza el valor de retorno del método.
  • this - el método devuelve una referencia This no nula
  • param1 (param2, param3, etc.) - el método devuelve su primer (segundo, tercer, etc.) argumento
Ejemplos:

@Contract("_, null -> null") - el método devuelve nulo si su segundo argumento es nulo
@Contract("_, null -> null; _, !null -> !null") - el método devuelve nulo si su segundo argumento es nulo y no nulo de lo contrario
@Contract("true -> fail") - un método assertFalse típico que lanza una excepción si true se le pasa a él
