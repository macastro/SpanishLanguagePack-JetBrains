Informa sobre parámetros de método que tienen el mismo nombre que los campos de las clases circundantes. Debido a este tipo de nomenclatura, puede usar accidentalmente el parámetro cuando tiene la intención de usar el campo del mismo nombre.

Se recomienda renombrar el parámetro mediante una corrección rápida.

Ejemplo:


  class Main {
    private String value;
    public Main(String value) {
      value = value.toUpperCase();
    }
  }

Puede configurar las siguientes opciones para esta inspección:

  1. Ignorar para setters de propiedades - Ignora los parámetros de setters simples.
  2. Ignorar campos de superclase no visibles desde subclases - Ignora los campos private en la superclase que no son visibles para este método.
  3. Ignorar para constructores - Ignora los parámetros de constructores.
  4. Ignorar para métodos abstract - Ignora los parámetros de métodos abstract.
  5. Ignorar parámetros de métodos static que ocultan campos de instancia - Ignora los parámetros de métodos static que ocultan campos de instancia y los parámetros de métodos de instancia en clases internas static que ocultan campos de instancia de la clase externa. Aunque técnicamente no es ocultamiento, estos parámetros pueden resultar confusos.
