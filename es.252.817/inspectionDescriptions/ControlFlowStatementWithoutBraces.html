Esta inspección notifica sentencias if, while, do o for sin llaves. Algunos estilos de código, como la Guía de estilo de Java de Google, requieren llaves para todas las sentencias de control.

Es importante recordar añadir llaves al añadir más sentencias a una sentencia de control sin llaves. También es importante tener más cuidado al comentar una línea de código sin usar llaves para no crear inadvertidamente la siguiente parte de una sentencia de control de flujo. El uso constante de llaves hace que sea más seguro insertar o comentar una línea de código.

Es probable que la vulnerabilidad goto fail no ocurra si utilizas un estilo de código que siempre utiliza llaves. Las sentencias de control con llaves hacen que el flujo de control sea más fácil de ver y eliminan la necesidad de una sangría que podría ser incorrecta.

Ejemplo:


class Strange {
  void x(boolean one, boolean two) {
    if(one)
      if(two)
        foo();
    else
      bar();
  }

  void foo() {}
  void bar() {}
}

La corrección rápida envolverá el cuerpo de la sentencia con llaves.


class Strange {
  void x(boolean one, boolean two) {
    if(one) {
      if(two) {
        foo();
      } else {
        bar();
      }
    }
  }

  void foo() {}
  void bar() {}
}
