Informa expresiones condicionales cuyas condiciones son negadas.

Invertir el orden de las ramas de la expresión condicional usualmente incrementa la claridad de tales sentencias.

Usa las opciones Ignorar comparaciones '!= null' e Ignorar comparaciones '!= 0' para ignorar comparaciones de la forma obj != null o num != 0. Dado que obj != null significa efectivamente "obj existe", el significado de toda la expresión no implica ninguna negación y es por lo tanto fácil de entender.

El mismo razonamiento aplica a expresiones num != 0, especialmente cuando se usan máscaras de bits.

Estas formas tienen el beneficio agregado de mencionar el caso interesante primero. En la mayoría de los casos, el valor para la rama == null es null en sí, como en los siguientes ejemplos: 


    static String getName(Person p) {
        return p != null ? p.getName() : null;
    }

    static String getExecutableString(int fileMode) {
        return (fileMode & 0b001001001) != 0 ? "ejecutable" : "no ejecutable";
    }
