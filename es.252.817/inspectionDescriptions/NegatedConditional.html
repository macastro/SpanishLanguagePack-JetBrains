Reporta expresiones condicionales que utilizan negación.

Invertir el orden de las ramas en expresiones condicionales suele mejorar la claridad de estas declaraciones.

Use las opciones Ignorar comparación '!= null' y Ignorar comparación '!= 0' para ignorar las comparaciones de la forma obj != null o num != 0. Como obj != null significa efectivamente "obj existe", el significado de toda la expresión no implica ninguna negación, por lo que es fácil de entender.

El mismo razonamiento se aplica a las expresiones num != 0, especialmente cuando se utilizan máscaras de bits.

Estas formas tienen la ventaja adicional de mencionar primero el caso interesante. En la mayoría de los casos, la rama == null tiene el valor null en sí mismo, como se muestra en los siguientes ejemplos: 


    static String getName(Person p) {
        return p != null ? p.getName() : null;
    }
    static String getExecutableString(int fileMode) {
        return (fileMode & 0b001001001) != 0 ? "executable" : "non-executable";
    }
