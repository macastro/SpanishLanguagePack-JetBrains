Informa sobre métodos que exceden el número máximo especificado de puntos de return. Los métodos con demasiados puntos de return pueden ser confusos y difíciles de refactorizar.

Un punto de return puede ser una declaración return o una caída directa desde el final de un método void o constructor.

Ejemplo:

El siguiente método se informará si solo se permiten dos declaraciones return:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user)) {
        user.setId(getId());
        return;
      } else if (cond2(user)) {
        if (cond3(user)) {
          user.setId(getId());
          return;
        }
      }
    }
  }

Considere reescribir el método para hacerlo más comprensible:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user) || cond2(user) && cond3(user)) {
        user.setId(getId());
        return;
      }
    }
  }

Configuración de la inspección:

  • Use el campo Límite de puntos return para especificar el número máximo permitido de puntos return en un método.
  • Use la opción Ignorar cláusulas críticas para ignorar las cláusulas críticas. Una cláusula crítica es una declaración if que solo contiene una declaración return
  • Use la opción Ignorar para método 'equals()' para ignorar los puntos de return en el método equals().
