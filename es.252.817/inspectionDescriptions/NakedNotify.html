Informa de las llamadas a Object.notify() u Object.notifyAll() sin que se produzca ningún cambio de estado detectable.

Normalmente, Object.notify() y Object.notifyAll() se utilizan para informar a otros subprocesos de que se ha producido un cambio de estado. Ese cambio de estado debe producirse en un contexto sincronizado que contenga la llamada a Object.notify() o Object.notifyAll(), y antes de la llamada. Aunque no tener un cambio de estado de este tipo no es necesariamente incorrecto, ciertamente merece la pena examinarlo.

Ejemplo:


  synchronized (this) {
    notify();
  }
  // sin cambio de estado
  synchronized (this) {
    notify(); // esta notificación podría ser redundante
  }
