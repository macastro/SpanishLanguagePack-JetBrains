Reporta conversiones de expresiones como string(x), donde x es un entero pero no es un byte o rune.

Estas conversiones no son recomendadas ya que devuelven la representación UTF-8 del punto de código Unicode x, en lugar de devolver la representación decimal de x como cadena, que es lo que se podría esperar. Además, si x representa un punto de código inválido, la conversión no puede ser rechazada estáticamente.

Para conversiones donde se pretende usar puntos de código, considere reemplazarlas por string(rune(x)). En caso contrario, strconv.Itoa y sus funciones equivalentes devolverán la representación en cadena del valor según sea necesario.

Ejemplo:

func main() {
 a := 1
 _ = string(a)
}

Después de aplicar la corrección rápida convertir entero a rune:

func main() {
 a := 1
 _ = string(rune(a))
}
