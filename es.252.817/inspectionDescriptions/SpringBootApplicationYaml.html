Informa las claves de configuración obsoletas y no resueltas, así como los valores no válidos en los archivos de configuración .yaml de la aplicación Spring Boot, que pueden generar errores en tiempo de ejecución.

Ejemplo:


server:
  port: inválido #Informa 'No se puede convertir 'inválido' a java.lang.Integer'

Si una clave de configuración obsoleta tiene una clave de reemplazo, puedes aplicar la solución rápida 'Usar clave de reemplazo'.

Ejemplo:


logging:
  path: ${path} #Informa 'Propiedad de configuración obsoleta 'logging.path''

Después de aplicar la solución rápida:


logging:
  file:
    path: ${path}

Si una clave de configuración no está definida en spring-configuration-metadata.json, puedes aplicar la solución rápida 'Definir clave de configuración' que crea el archivo META-INF/spring-additional-configuration-metadata.json y define la clave necesaria.

Ejemplo:


nuevo:
  clave: valor #Informa 'No se puede resolver la propiedad de configuración 'nuevo.clave''

Después de aplicar la solución rápida, se agrega lo siguiente a META-INF/spring-additional-configuration-metadata.json:


{
  "properties": [
    {
      "name": "nuevo.clave",
      "type": "java.lang.String",
      "description": "Descripción para nuevo.clave."
  }
] }

Utiliza la opción Tokens de reemplazo para definir tokens utilizados para marcadores de posición de valor en los archivos de configuración. Estos tokens están especificados en forma beginToken*endToken. Sin el *, se supone que el token es el mismo para el inicio y el final.

Por ejemplo, el valor predeterminado es @ para el token de inicio y el token de finalización, lo que te permite definir marcadores de posición, como some.property=@otra.property@.

Los valores dentro del token de reemplazo (@property.key@) no se resaltarán.
