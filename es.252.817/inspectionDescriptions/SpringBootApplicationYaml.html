Informa sobre claves de configuración no resueltas y obsoletas, así como valores inválidos en archivos de configuración .yaml de aplicaciones Spring Boot, que podrían causar errores en tiempo de ejecución.

Ejemplo:


server:
  port: invalid # Informa 'No se puede convertir 'invalid' a java.lang.Integer'

Si una clave de configuración obsoleta tiene una clave de reemplazo, puede aplicar la corrección rápida "Usar clave de reemplazo".

Ejemplo:


logging:
  path: ${path} # Informa 'Propiedad de configuración obsoleta 'logging.path''

Después de aplicar la corrección rápida:


logging:
  file:
    path: ${path}

Si la clave de configuración no está definida en spring-configuration-metadata.json, puede aplicar la corrección rápida "Definir clave de configuración" para crear el archivo META-INF/spring-additional-configuration-metadata.json y definir las claves necesarias.

Ejemplo:


new:
  key: value # Informa 'No se puede resolver la propiedad de configuración 'new.key''

Después de aplicar la corrección rápida, se añadirá lo siguiente a META-INF/spring-additional-configuration-metadata.json:


{
  "properties": [
    {
      "name": "new.key",
      "type": "java.lang.String",
      "description": "Description for new.key."
  }
] }

La opción Tokens de reemplazo permite definir tokens para los marcadores de posición de valores en los archivos de configuración. Estos tokens se especifican en el formato beginToken*endToken. Si no hay *, se asume que los tokens de inicio y fin son iguales.

Por ejemplo, el valor predeterminado para los tokens de inicio y fin es @, lo que permite definir marcadores de posición como some.property=@another.property@.

Los valores dentro de los tokens de reemplazo (@property.key@) no se resaltarán.
