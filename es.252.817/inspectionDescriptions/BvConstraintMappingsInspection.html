Reporta elementos incorrectos en el archivo de mapeo de validación de Bean <constraint-mappings>:

  • Referencias a clases no instanciables
  • Referencias a clases que no extienden las clases requeridas
  • Referencias a clases con ámbito inadecuado
  • Etiquetas y valores de atributos vacíos
  • Etiquetas y valores de atributos que no coinciden con el patrón requerido (por ejemplo, identificadores Java)
  • Etiquetas que no contienen las subetiquetas o atributos requeridos
  • Etiquetas que definen objetos con nombres duplicados

Ejemplo:


<constraint-mappings version="1.1">
    <bean class="my.UserClass" ignore-annotations="false">
        <field name="name">
            <constraint annotation="java.lang.String"/>  <!--'java.lang.String' no es una anotación Java -->
        </field>
        <field name="name">
            <constraint annotation="javax.validation.constraints.Size">
                <element name="min">non-number</element>    <!--debe ser un número entre '-2,147,483,648' y '2,147,483,647' -->
            </constraint>
        </field>
        <getter name="zzz"> No se puede resolver el símbolo 'zzz'
            <valid/>
        </getter>
    </bean>
</constraint-mappings>
