this o el acceso a un campo estático no constante bloqueado en el campo de instancia this.

El bloqueo del campo estático en datos de instancia no impide que el campo sea modificado por otras instancias, por lo que pueden ocurrir condiciones de carrera inesperadas.

Ejemplo:


  static String test;
  public void foo() {
      synchronized (this) {
          System.out.println(test); // warning
      }
  }

Existe una solución rápida para omitir campos estáticos de tipos específicos. Los tipos omitidos se pueden administrar en las opciones de la inspección.

Utilice las opciones de la inspección para especificar clases que se deben omitir para el campo estático.
