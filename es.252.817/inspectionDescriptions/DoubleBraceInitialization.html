Informe sobre la inicialización con dobles llaves.

La inicialización con dobles llaves tiene un rendimiento inferior comparado con la inicialización regular, ya que requiere cargar una clase adicional.

Si el método equals() no acepta subclases como parámetros, también puede causar que la comparación equals() falle.

El uso de la inicialización con dobles llaves en un contexto no static puede causar fugas de memoria. Esto se debe a que define una clase anónima que referenciará al objeto circundante cuando se compile con versiones de javac anteriores a Java 18.

Además, en versiones anteriores a Java 9, la inicialización con dobles llaves no puede combinarse con el operador Diamond, ya que no es compatible con clases anónimas.

Ejemplo:


  List<Integer> list = new ArrayList<>() {{
    add(1);
    add(2);
  }};

Después de aplicar la corrección rápida:


  List<Integer> list = new ArrayList<>();
  list.add(1);
  list.add(2);
