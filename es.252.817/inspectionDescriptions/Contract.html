Informa sobre problemas en la anotación @Contract del método. Los tipos de problemas que se pueden informar son:
  • Errores de sintaxis en el contrato
  • Contratos que no coinciden con la firma del método (número incorrecto de parámetros)
  • Implementaciones de método que no cumplen con el contrato (por ejemplo, devolver true cuando el contrato declara false)

Ejemplo:


  // El método no tiene parámetros, pero el contrato espera 1
  @Contract("_ -> fail")
  void x() {
    throw new AssertionError();
  }
