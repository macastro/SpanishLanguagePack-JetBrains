Informa sobre asignaciones y llamadas a funciones donde el nombre de la variable de destino o del parámetro de función no coincide con el nombre del valor que se le asigna.

Ejemplo 1:


  int x = 0;
  int y = x; // x se usa como coordenada y

Ejemplo 2:


  int x = 0, y = 0;
  // x se usa como coordenada y, y se usa como coordenada x
  Rectangle rc = new Rectangle(y, x, 20, 20);

Configuración de la inspección:

Use la sección Grupos de nombres para especificar nombres que no deberían usarse juntos: se informará un error si el nombre del parámetro o el objetivo de la asignación contiene una palabra de un grupo, mientras que el nombre de la variable asignada o pasada contiene una palabra de otro grupo.

Use la sección Métodos a ignorar para especificar métodos que no deberían ser verificados pero cuyos nombres podrían ser cuestionables. Por ejemplo, los parámetros de Integer.compare() se nombran como x e y, pero no están relacionados con coordenadas.
