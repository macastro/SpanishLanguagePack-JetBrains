Se informa sobre las llamadas JUnit assertEquals() que pueden ser reemplazadas por llamadas equivalentes assertSame(). Esto puede ocurrir cuando los argumentos son instancias de clases final que no sobrescriben el método Object.equals() y comparan explícitamente la identidad del objeto.

Se sugiere reemplazar assertEquals() por assertSame().

Ejemplo:


  @Test
  public void testObjectType() {
    Object o = getObject();
    Assert.assertEquals(String.class, o.getClass());
  }

Después de aplicar la corrección rápida:


  @Test
  public void testSort() {
    Object o = getObject();
    Assert.assertSame(String.class, o.getClass());
  }
