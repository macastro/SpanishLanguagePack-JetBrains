Reporta llamadas que solo tienen sentido en receptores nullables cuando se realizan en receptores no-null.

Algunas funciones de la biblioteca estándar como orEmpty() o isNullOrEmpty solo tienen sentido cuando se llaman en receptores de tipo nullable. En otros casos, pueden omitirse ya que el resultado será el mismo.

Las correcciones rápidas Eliminar llamada redundante y Cambiar llamada a ... están disponibles para corregir el código automáticamente.

Ejemplo:


  fun test(s: String) {
      val x = s.orEmpty() // La corrección rápida simplificará a 's'
      val y = s.isNullOrEmpty() // La corrección rápida simplificará a 's.isEmpty()'
  }
