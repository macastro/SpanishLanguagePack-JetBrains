Reporta llamadas en un receptor no nulo que tienen sentido solo para un receptor nulo.

Varias funciones de la biblioteca estándar como orEmpty() o isNullOrEmpty solo tienen sentido cuando se llaman a receptores de tipos nulos. De lo contrario, pueden omitirse o simplificarse ya que el resultado será el mismo.

Se pueden usar las correcciones rápidas "Eliminar llamada redundante" y "Cambiar llamada a ..." para enmendar el código automáticamente.

Ejemplos:


  fun test(s: String) {
      val x = s.orEmpty() // la corrección rápida simplifica a 's'
      val y = s.isNullOrEmpty() // la corrección rápida simplifica a 's.isEmpty()'
  }
