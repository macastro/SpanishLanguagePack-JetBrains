Informa sobre recursos de socket que no se han cerrado de manera segura. Esta comprobación reporta recursos de socket que incluyen java.net.Socket, java.net.DatagramSocket y java.net.ServerSocket.

Por defecto, esta comprobación asume que los recursos pueden ser cerrados por cualquier método que contenga 'close' o 'cleanup' en su nombre.

Ejemplo:


  byte[] getMessage(ServerSocket socket) throws IOException {
    Socket client = socket.accept(); //socket no cerrado
    return client.getInputStream().readAllBytes();
  }

Configure la comprobación con las siguientes opciones:

  • Si se permite abrir sockets dentro de un bloque try. Este estilo es menos preferible ya que es más verboso que abrir el recurso antes del bloque try.
  • Si el recurso puede ser cerrado mediante cualquier llamada a método que tome el recurso como argumento.
