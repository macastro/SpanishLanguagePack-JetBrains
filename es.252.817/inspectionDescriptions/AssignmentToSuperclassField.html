Informa sobre asignaciones o modificaciones de campos declarados en la superclase desde constructores de subclases.

Es mejor inicializar los campos de la superclase en su propio constructor y delegar a ese constructor en la subclase. Esto también permitirá declarar un campo como final si no cambia después de la construcción.

Ejemplo:


  class Super {
    int x;
  }
  class Sub extends Super {
    Sub(int _x) {
      // Warning: x is declared in a superclass
      x = _x;
    }
  }

Para evitar este problema, declare un constructor en la superclase:


  class Super {
    final int x;
    Super(int _x) {
      x = _x;
    }
  }
  class Sub extends Super {
    Sub(int _x) {
      super(_x);
    }
  }
