Informa de métodos cuyos nombres son demasiado cortos, demasiado largos o no siguen el patrón de la expresión regular especificada.

Los métodos de instancias que sobrescriben métodos y constructores de biblioteca se ignoran por esta inspección.

Ejemplo: si la inspección está habilitada para métodos estáticos y la longitud mínima del nombre del método es 4 (el valor predeterminado), el siguiente método estático produce una advertencia, porque su nombre mide 3, lo cual es menor que 4: public static int max(int a, int b).

Existe una corrección rápida para renombrar dichos métodos, y solo está disponible en el editor.

Configura la inspección:

Utiliza la lista en la sección Opciones para especificar qué métodos se deben verificar. Anula la selección de las casillas de verificación para los tipos de método que se van a omitir de la verificación. Especifica 0 en los campos de longitud para omitir las verificaciones correspondientes.

Las expresiones regulares se deben especificar con el formato estándar java.util.regex.
