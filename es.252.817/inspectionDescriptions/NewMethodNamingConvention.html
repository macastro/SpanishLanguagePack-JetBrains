Reporta métodos cuyos nombres son demasiado cortos, demasiado largos o no siguen el patrón de expresión regular especificado.

Esta inspección ignora los métodos de instancia que sobrescriben métodos de biblioteca y constructores.

Ejemplo: Si la inspección está habilitada para métodos static y la longitud mínima especificada para nombres de métodos es 4 (valor predeterminado), el siguiente método static generará una advertencia porque su nombre tiene una longitud de 3, que es menor que 4: public static int max(int a, int b).

La corrección rápida para renombrar estos métodos solo está disponible en el editor.

Configuración de la inspección:

Utilice la lista en la sección de Opciones para especificar qué métodos deben ser inspeccionados. Desmarque las casillas de los tipos de métodos que desea excluir de la inspección. Especifique 0 en los campos de longitud para omitir la inspección correspondiente.

Las expresiones regulares deben especificarse en el formato estándar de java.util.regex.
