Reportar problemas del contexto basado en XML de Spring Security:

  • Referencias de Bean sin resolver
  • Etiquetas o atributos requeridos faltantes
  • Tipo de propiedad incorrecto
  • Propiedad enum inconsistente
  • Tipo incorrecto de Bean referenciado

También se reportan los siguientes problemas:

  • Errores de configuración <https>
    • El <http> con patrón '/**' debe estar al final
    • No se permiten múltiples elementos <http> sin 'pattern'
  • Errores de configuración <http>
    • Etiqueta vacía si security='none'
  • <user-service>
    • Debe tener el atributo 'properties' o una lista de elementos <user>
  • <ldap-server>
    • Se requiere 'manager-password' cuando se usa 'manager-dn'
    • Debe tener 'user-context-mapper-ref' o 'user-details-class'
  • <authentication-provider>
    • Solo se permite un tipo de servicio de usuario
  • <global-method-security>
    • El valor de 'pre-post-annotations' debe ser 'enabled' para usar <pre-post-annotation-handling>
    • No se puede usar 'mode' con valor 'aspectj' para <protect-pointcut> y <intercept-url>
    • El atributo 'requires-channel' solo está permitido en el <http> padre
  • <protect>
    • No se recomienda mezclar <protect> con <global-method-security>
