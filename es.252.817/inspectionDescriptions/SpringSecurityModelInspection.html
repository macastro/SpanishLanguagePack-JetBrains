Informa problemas con el contexto de Spring Security basado en XML:

  • Referencias de bean sin resolver
  • Etiquetas o atributos obligatorios faltantes
  • Tipos de propiedad incorrectos
  • Propiedades enum inconsistentes
  • Tipos incorrectos de beans referenciados

También informa los siguientes problemas:

  • Errores de configuración <https>
    • <http> con el patrón '/**' debe ser el último
    • No se permiten varios elementos <http> sin 'patrón'
  • Errores de configuración <http>
    • Etiqueta vacía si security='none'
  • <user-service>
    • Debe tener el atributo 'properties' o una lista de elementos <user>
  • <ldap-server>
    • 'manager-password' es obligatorio cuando se utiliza 'manager-dn'
    • Debe tener 'user-context-mapper-ref' o 'user-details-class'
  • <authentication-provider>
    • Solo se permite un tipo de user-service
  • <global-method-security>
    • 'pre-post-annotations' debe tener el valor 'enabled' para utilizar <pre-post-annotation-handling>'
    • No se puede utilizar 'mode' con valor 'aspectj' para <protect-pointcut> y <intercept-url>
    • El atributo 'requires-channel' solo está permitido para el <http> principal
  • <protect>
    • No se recomienda mezclar <protect> con <global-method-security>
