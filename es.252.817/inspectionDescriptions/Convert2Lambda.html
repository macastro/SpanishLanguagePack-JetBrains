Se informa de clases anónimas que pueden ser reemplazadas por expresiones lambda.

Ejemplo:


  new Thread(new Runnable() {
    @Override
    public void run() {
      // Ejecutar hilo
    }
  });

Después de aplicar la corrección rápida:


  new Thread(() -> {
    // Ejecutar hilo
  });

Tenga en cuenta que si una clase anónima se convierte en un lambda sin estado, el tiempo de ejecución de Java puede reutilizar el mismo objeto lambda durante las llamadas posteriores. Por otro lado, cuando se utilizan clases anónimas, se crea un objeto separado cada vez. Por lo tanto, en casos muy raros, la aplicación de esta corrección rápida podría resultar en un cambio semántico, por ejemplo, cuando una instancia de clase anónima se utiliza como clave en un HashMap.

Utilice la opción Informar cuando la interfaz no está anotada con @FunctionalInterface para ignorar los casos donde las clases anónimas implementan interfaces que no tienen la anotación @FunctionalInterface.
