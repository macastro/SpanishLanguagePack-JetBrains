Informa sobre métodos que pueden establecerse de forma segura como static. Establecer métodos como static cuando sea posible reduce el consumo de memoria y mejora la calidad del código.

Un método puede ser static si:

  • No es synchronized, native o abstract,
  • No hace referencia a ningún método o campo no static en la clase contenedora,
  • No es una sobrescritura ni será sobrescrito en subclases.

Configure la inspección con las siguientes opciones:

  • Si solo se deben informar métodos private y final, mejorando así el rendimiento de esta inspección.
  • Si se deben ignorar los métodos vacíos.
  • Si se deben ignorar los métodos por defecto en interfaces cuando se usa Java 8 o superior.
  • Si la corrección rápida debe reemplazar los calificadores de instancia con referencias de clase al llamar a métodos convertidos a static, es decir, la llamada myClass.m() se reemplazará por MyClass.m().
