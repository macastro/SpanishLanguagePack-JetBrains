El informe indica declaraciones inválidas de métodos del ciclo de vida transaccional que utilizan las anotaciones @BeforeTransaction y @AfterTransaction en una clase de prueba anotada con @Transactional.

Los métodos anotados no deben tener argumentos ni tipo de retorno.

Ejemplo:


  @ContextConfiguration
  @Transactional
  public class AbstractShowcaseTest {
    @BeforeTransaction // El tipo de retorno del método debe ser 'void'
    public boolean setupData() {...}
    @AfterTransaction // Número incorrecto de argumentos
    public void disposeData(boolean a) throws Exception {...}
  }
