Reporta usos inválidos de la sintaxis de parámetros de tipo de PEP 695

Busca los siguientes problemas en definiciones de funciones y clases, y en declaraciones de alias de tipo de nuevo estilo:

  • Extensión de typing.Generic en clases genéricas de nuevo estilo
  • Extensión de typing.Protocol parametrizado en clases genéricas de nuevo estilo
  • Uso de límites y restricciones genéricas con parámetros de tipo en ParamSpec y TypeVarTuple
  • Mezcla de variables de tipo tradicionales y de nuevo estilo
  • Uso de variables de tipo tradicionales en alias de tipo de nuevo estilo

Ejemplos: 


  from typing import Generic
  class Example[T](Generic[T]): ... # Las clases con lista de parámetros de tipo no deberían extender 'Generic'


  class Example[T: (list[S], str)]: ... # No se permiten tipos genéricos en las restricciones y límites de los parámetros de tipo


  from typing import TypeVar
  K = TypeVar("K")
  class ClassC[V]:
      def method2[M](self, a: M, b: K) -> M | K: ... # No se permite mezclar TypeVars tradicionales y de nuevo estilo
