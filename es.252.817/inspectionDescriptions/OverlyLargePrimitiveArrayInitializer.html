Informa sobre expresiones de inicialización de arreglos para los arreglos primitivos que contienen demasiados elementos. Esos inicializadores pueden generar archivos de clase demasiado grandes, ya que se debe generar el código para inicializar cada elemento del arreglo. En entornos con restricciones de memoria o ancho de banda, puede ser más eficiente cargar arreglos grandes de primitivas desde archivos de recursos.

Esta inspección está diseñada para Java ME y otros entornos con muchos recursos restringidos. Aplicar los resultados de esta inspección sin consideración podría tener efectos negativos sobre la claridad y el diseño del código.

Utiliza la opción para especificar el número máximo de elementos que se permitirán en los inicializadores de arreglos primitivos.
