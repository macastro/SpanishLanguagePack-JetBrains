Informa sobre llamadas incondicionales a wait() en un contexto sincronizado.

wait() se utiliza normalmente para bloquear un hilo hasta que se cumpla una condición. Si se llama a wait() sin condición, esto suele indicar que la condición se verificó antes de obtener el bloqueo. En este caso, puede producirse una condición de carrera, donde la condición se vuelve verdadera entre la comprobación y la obtención del bloqueo.

Aunque los patrones detectados por esta inspección no son necesariamente incorrectos, definitivamente merecen ser revisados.

Ejemplo:


  class Bar {
      void foo() throws InterruptedException {
          synchronized (this) {
              wait(); // advertencia
          }
      }
  }
