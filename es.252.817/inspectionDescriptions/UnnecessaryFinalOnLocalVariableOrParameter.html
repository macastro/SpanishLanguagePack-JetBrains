Informa sobre variables locales o parámetros declarados innecesariamente como final.

Por razones de concisión, algunos estándares de codificación desaconsejan las variables declaradas como final.

Ejemplo:


  class Foo {
    Foo(Object o) {}
    void bar(final Object o) {
      new Foo(o);
    }
  }

Después de aplicar la corrección rápida:


  class Foo {
    Foo(Object o) {}
    void bar(Object o) {
      new Foo(o);
    }
  }

Las opciones de inspección permiten alternar los informes para:

  • Variables locales
  • Parámetros (incluyendo parámetros en bloques catch y sentencias for mejoradas)

Además, puede configurar la inspección para que solo informe sobre parámetros final en métodos abstract o de interfaz, que pueden considerarse redundantes ya que tales marcadores no afectan la implementación de estos métodos.
