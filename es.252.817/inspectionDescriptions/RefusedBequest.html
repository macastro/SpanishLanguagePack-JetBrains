Informa sobre métodos sobrescritos que no llaman al método super. Esto también se conoce como herencia rechazada. Estos métodos pueden indicar un fallo en la abstracción, lo que puede llevar a errores difíciles de rastrear.

La inspección no informa sobre métodos sobrescritos de java.lang.Object (excepto clone()). Por convención, clone() debe llamar a su método super, que devolverá un objeto del tipo correcto.

Ejemplo 1:


  class A {
    @Override
    public Object clone() {
      // No llama a 'super.clone()'
      return new A();
    }
  }

Ejemplo 2:


  interface I {
    default void foo() {}
  }
  class A implements I {
    // Advierte sobre el método cuando está desactivada
    // la opción "Ignorar métodos super 'default'"
    @Override
    public void foo(){}
  }

Configuración de la inspección:

  • Use la opción Informar solo cuando el método super está anotado con para ignorar los métodos super marcados con anotaciones de la lista proporcionada. Puede agregar anotaciones a esta lista manualmente.
  • Use la opción Ignorar métodos super vacíos para ignorar los métodos super que solo lanzan excepciones o están vacíos.
  • Use la opción Ignorar métodos super 'default' para ignorar los métodos super default en interfaces.
