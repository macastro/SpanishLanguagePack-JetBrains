Reporta inconsistencias en etiquetas y atributos en los puntos de inyección del contexto de aplicación XML.

  • <property> y <constructor-arg> deben especificar ref, value o elementos hijo
  • <property> y <constructor-arg> no pueden contener múltiples elementos hijo
  • <ref> puede contener el atributo bean, el atributo local o el atributo parent
  • <ref> debe especificar un Bean
  • <idref> puede contener el atributo bean o el atributo local
  • <idref> debe especificar un Bean
  • <map> debe especificar una clave
  • <idref> puede contener el atributo key, el atributo key-ref o el elemento hijo <key>

Ejemplo:


  <beans>
  <!-- 1. La propiedad solo puede contener el atributo "ref" o el atributo "value" o elementos hijo -->
  <bean class="MyClass">
    <property name="bean" ref="myBean" value="foo"/>
    <!-- La propiedad solo puede contener el atributo "ref" o el atributo "value" o elementos hijo -->
  </bean>
  <!-- 2. La propiedad no puede contener múltiples elementos hijo -->
  <bean class="MyFactory">
    <property name="bean">
    <ref>bean</ref> <!-- La propiedad no puede contener múltiples elementos hijo -->
     <value>foo</value> <!-- La propiedad no puede contener múltiples elementos hijo -->
     </property>
  </bean>
  <!-- 3. La entrada del mapa debe especificar una clave -->
  <bean class="MyBean">
     <property name="map">
        <map>
           <entry value="foo"/>  <!-- La entrada del mapa debe especificar una clave -->
        </map>
     </property>
  </bean>
  </beans>
