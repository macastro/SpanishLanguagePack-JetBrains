java.util.Properties objeto en las siguientes llamadas al método:
  • put()
  • putIfAbsent()
  • putAll()
  • get()

Por razones históricas, java.util.Properties hereda de java.util.Hashtable, pero no se recomienda utilizar estos métodos para evitar la contaminación de valores de propiedades que no sean de tipo String.

Si tanto los parámetros de clave como de valor en el mapa son de tipo String, no se notificará la llamada a java.util.Properties.putAll(). Estas llamadas son seguras y no hay una alternativa mejor.

Ejemplo:


  Object f(Properties props) {
    props.put("hello", "world");
    props.putIfAbsent("hello", "world");
    props.putAll(new HashMap<>());
    return props.get("Hello");
  }

Después de aplicar la solución rápida:


  Object f(Properties props) {
    props.setProperty("hello", "world");
    props.putIfAbsent("hello", "world");
    props.putAll(new HashMap<>());
    return props.getProperty("hello");
  }
