Informa sobre llamadas a Collection.addAll() y Map.putAll() inmediatamente después de instanciar una colección usando un constructor sin parámetros.

Estas estructuras pueden ser reemplazadas por una única llamada a un constructor parametrizado, simplificando así el código. Además, para algunas colecciones, el reemplazo puede ser más eficiente.

Ejemplo:


  Set<String> set = new HashSet<>();
  set.addAll(Arrays.asList("alpha", "beta", "gamma"));

Después de aplicar la corrección rápida:


  Set<String> set = new HashSet<>(Arrays.asList("alpha", "beta", "gamma"));

Las clases de colección JDK están soportadas por defecto. Además, se pueden especificar clases adicionales usando el panel Clases a verificar.
