Reporta las caídas directas en las declaraciones switch.

Una caída directa ocurre cuando una serie de declaraciones ejecutables después de una etiqueta case no garantiza la transferencia de control antes de la siguiente etiqueta case. Esto puede ocurrir, por ejemplo, cuando una rama carece de una declaración break. En tal caso, el control cae directamente a las declaraciones después de la etiqueta switch, incluso cuando la expresión del switch no es igual al valor de la etiqueta. Aunque a veces es intencional, esta estructura puede ser confusa y generalmente es el resultado de errores tipográficos.

Esta inspección ignorará cualquier caída directa que tenga un comentario de texto que coincida con el patrón de expresión regular (?i)falls?\s*thro?u.

Se puede corregir añadiendo una declaración break a las ramas que puedan caer directamente a la siguiente rama.

Ejemplo:


    switch(x) {
      case (4):
          if (condition) {
              System.out.println("3");
              // Sin break aquí
          } else {
              break;
          }
      case (6):
          System.out.println("4");
    }

Después de aplicar la corrección rápida:


    switch(x) {
        case (4):
            if (condition) {
                System.out.println("3");
            } else {
                break;
            }
            break;
        case (6):
            System.out.println("4");
    }
