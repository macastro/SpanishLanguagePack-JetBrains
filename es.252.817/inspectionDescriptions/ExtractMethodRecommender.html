Propone extraer un fragmento de código a un método diferente para hacer el código más legible. Esta inspección cuenta con numerosos heurísticos para seleccionar buenos candidatos para la extracción, incluidos:
  • El fragmento extraído no tiene flujo de control no local
  • El fragmento extraído tiene exactamente una variable de salida
  • No hay usos similares de la variable de salida dentro y fuera del fragmento extraído
  • El fragmento extraído solo tiene unos pocos parámetros de entrada (3 o menos de manera predeterminada; configurable en opciones de inspección)
  • La longitud del fragmento extraído no es menor que la longitud configurada (500 caracteres de manera predeterminada), pero no es más del 60% de lo que contiene el cuerpo del método
