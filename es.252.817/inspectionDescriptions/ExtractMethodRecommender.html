Se sugiere extraer el fragmento de código a un método separado para mejorar la claridad del código. Esta inspección tiene varias heurísticas para seleccionar buenos candidatos para la extracción, incluyendo las siguientes.
  • El fragmento extraído no tiene flujo de control no local
  • El fragmento extraído tiene exactamente una variable de salida
  • No hay variables de salida similares utilizadas dentro y fuera del fragmento extraído
  • El fragmento extraído tiene pocos parámetros de entrada (no más de tres por defecto; configurable en las opciones de inspección)
  • El fragmento extraído no es más pequeño que la longitud configurada (500 caracteres por defecto) pero no supera el 60% del cuerpo del método que lo contiene
