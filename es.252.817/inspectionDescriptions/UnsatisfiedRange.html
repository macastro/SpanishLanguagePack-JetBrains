Informa sobre los valores numéricos devueltos por métodos que no se ajustan al rango de devolución del método declarado. Puedes declarar el rango de devolución del método utilizando varias anotaciones:
  • org.jetbrains.annotations.Range del paquete de anotaciones JetBrains (especifica 'desde' y 'hasta')
  • org.checkerframework.common.value.qual.IntRange del paquete de anotaciones Checker Framework (especifica 'desde' y 'hasta')
  • org.checkerframework.checker.index.qual.GTENegativeOne del paquete de anotaciones Checker Framework (el rango es '>= -1')
  • org.checkerframework.checker.index.qual.NonNegative del paquete de anotaciones Checker Framework (el rango es '>= 0')
  • org.checkerframework.checker.index.qual.Positive del paquete de anotaciones Checker Framework (el rango es '> 0')
  • javax.annotation.Nonnegative del paquete de anotaciones JSR 305 (el rango es '>= 0')
  • javax.validation.constraints.Min (especifica el valor mínimo)
  • javax.validation.constraints.Max (especifica el valor máximo)

Ejemplo:


  @Range(from = 0, to = Integer.MAX_VALUE) int getValue() {
    // Aviso: -1 está fuera del rango declarado
    return -1;
  }
