Informa sobre bloques catch vacíos o que posiblemente ignoren excepciones.

Aunque a veces es intencional, los bloques catch vacíos pueden dificultar la depuración. Además, ignorar el parámetro catch podría ser un error. Finalmente, el analizador de código estático informará si detecta bloques catch que puedan ignorar silenciosamente excepciones importantes de la VM, como NullPointerException. Ignorar tales excepciones (sin registrarlas ni relanzarlas) puede ocultar errores.

Esta inspección no informa sobre parámetros catch nombrados como ignore o ignored. En cambio, emitirá una advertencia para los parámetros catch nombrados como ignore o ignored que realmente se estén utilizando. Además, la inspección no informa sobre parámetros catch nombrados como expected u ok en código fuente de pruebas.

Puede usar la corrección rápida para cambiar el nombre de la excepción a ignored. Para bloques catch vacíos, se sugieren otras correcciones rápidas para generar el cuerpo del catch. Puede modificar la plantilla "Catch Statement Body" en la pestaña "Código" en Configuración | Editor | Archivos y Plantillas de Código.

Ejemplo:


  try {
    throwingMethod();
  } catch (IOException ex) {
  }

Después de aplicar la corrección rápida:


  try {
    System.out.println(System.in.read());
  } catch (IOException ignored) {
  }

Configuración de la inspección:

  • Use la opción No informar cuando el bloque 'catch' contenga comentarios para ignorar bloques catch que contengan comentarios.
  • Use la opción No informar cuando el bloque 'catch' no esté vacío para ignorar bloques catch que contengan declaraciones o comentarios pero no usen la variable en sí.
  • Use la opción No informar cuando una excepción nombrada 'ignore(d)' no sea realmente ignorada para ignorar variables nombradas como ignored que estén en uso.
