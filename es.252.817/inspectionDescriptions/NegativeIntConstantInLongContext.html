Informa constantes hexadecimales int negativas en contexto long. Dichas constantes se amplían implícitamente a long, lo que significa que sus bits superiores se volverán 1 en lugar de 0 (por ejemplo, 0xFFFF_FFFF se convertirá en 0xFFFF_FFFF_FFFF_FFFF). Es poco probable que esto sea intencionado, e incluso si lo es, usar una constante long explícita sería menos confuso.

Ejemplo:


  // Advertencia: esta es una constante int -1 que se amplía a long
  // convirtiéndose en 0xFFFF_FFFF_FFFF_FFFF.
  long mask = 0xFFFF_FFFF;
