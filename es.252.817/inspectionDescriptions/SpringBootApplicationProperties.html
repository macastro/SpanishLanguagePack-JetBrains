Reporta claves de configuración sin resolver y obsoletas, y valores no válidos en archivos de configuración .properties de la aplicación Spring Boot, lo cual puede provocar errores de tiempo de ejecución.

Ejemplo:


server.port=invalid # Informa 'No se puede convertir 'invalid' a java.lang.Integer'

Si una clave de configuración obsoleta tiene una clave de reemplazo, puedes aplicar el arreglo rápido 'Usar clave de reemplazo'.

Ejemplo:


logging.path=${path} # Informa 'Propiedad de configuración obsoleta 'logging.path''

Después de aplicar el arreglo rápido:


logging.file.path=${path}

Si una clave de configuración no está definida en spring-configuration-metadata.json, puedes aplicar el arreglo rápido 'Definir clave de configuración' que crea el archivo META-INF/spring-additional-configuration-metadata.json y define la clave necesaria.

Ejemplo:


new.key=value # Informa 'No se puede resolver la propiedad de configuración 'new.key''

Después de aplicar el arreglo rápido, lo siguiente se agrega a META-INF/spring-additional-configuration-metadata.json:


{
  "properties": [
    {
      "name": "new.key",
      "type": "java.lang.String",
      "description": "Description for new.key."
  }
] }

La inspección también resalta errores de notación de índice en las claves de configuración de listas y mapas.

Ejemplo:


spring.datasource.schema[]=${schema} # Informa 'Valor de índice faltante'

Usa la opción Tokens de reemplazo para definir tokens usados para marcadores de posición de valor en archivos de configuración. Estos tokens se especifican en el formato beginToken*endToken. Sin el *, se asume que el token es el mismo para el inicio y el final.

Por ejemplo, el valor predeterminado es @ tanto para el token de inicio como para el de finalización, lo que te permite definir marcadores de posición como some.property=@another.property@.

Los valores dentro del token de reemplazo (@property.key@) no se resaltarán.
