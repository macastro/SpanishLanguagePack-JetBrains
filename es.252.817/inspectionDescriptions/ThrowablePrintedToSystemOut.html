Reporta llamadas a System.out.println() que utilizan excepciones como argumentos.

El uso de declaraciones print para registrar excepciones oculta el seguimiento de la pila, lo que puede complicar la investigación del problema. Se recomienda usar un logger en su lugar.

También se reportan las llamadas a System.out.print(), System.err.println() y System.err.print() con excepciones como argumentos. Se recomienda usar un logger para registrar las excepciones.

La corrección rápida proporcionada es compatible con SLF4J y Log4j 2. Reemplaza las llamadas a System.out.println() con llamadas al logger

Ejemplo:


  try {
      foo();
  } catch (Exception e) {
      System.out.println(e);
  }

Después de aplicar la corrección rápida:


  try {
      foo();
  } catch (Exception e) {
    log.error("e: ", e);
  }

La opción Método de logging corregido permite especificar el método utilizado para registrar los mensajes.
