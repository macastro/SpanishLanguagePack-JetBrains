Informa sobre clases que podrían ser clonables.

Una clase puede ser clonable si implementa la interfaz Cloneable y su método clone() no está definido para lanzar una excepción inmediatamente. Las clases clonables pueden ser peligrosas en código utilizado para propósitos de seguridad.

Ejemplo:

class SecureBean implements Cloneable {}

Después de aplicar la corrección rápida:

class SecureBean {}

Cuando la clase extiende una clase clonable existente o implementa la interfaz clonable, el código podría verse así después de aplicar la corrección rápida: 

class SecureBean extends ParentBean {
    @Override
    protected SecureBean clone() throws CloneNotSupportedException {
        throw new CloneNotSupportedException();
    }
}
