Informa sobre clases que implementan java.io.Serializable y cuyos métodos readResolve() o writeReplace() no están declarados como protected.

Declarar los métodos readResolve() y writeReplace() como private permite que las subclases los ignoren silenciosamente, mientras que declararlos como public permite que código no confiable los invoque.

Si la clase contenedora está declarada como final, estos métodos pueden declararse como private.

Ejemplo:


  class ClassWithSerialization implements Serializable {
    public Object writeReplace() { // Advertencia: 'writeReplace()' no está declarado como protected
        ...
    }
  }
