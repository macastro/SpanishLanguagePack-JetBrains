Reporta bloqueo doblemente verificado.

El bloqueo doblemente verificado intenta inicializar un campo a pedido y de manera segura para subprocesos, al tiempo que evita el costo de la sincronización. Desafortunadamente, no es seguro para subprocesos cuando se utiliza en un campo que no está declarado como volatile. Cuando se utiliza Java 1.4 o una versión anterior, el bloqueo doblemente verificado no funciona ni siquiera con un campo volatile. Lea el artículo vinculado anteriormente para obtener una explicación detallada del problema.

Ejemplo de bloqueo doblemente verificado incorrecto:


  clase Foo {
    private Helper helper = null;
    public Helper obtenerHelper() {
      si (helper == null)
        sincronizado(esto) {
          si (helper == null) helper = nuevo Helper();
        }
        devolver helper;
      }
    }
    // otras funciones y miembros...
  }
