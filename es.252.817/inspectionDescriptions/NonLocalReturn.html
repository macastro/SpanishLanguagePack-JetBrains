Informa cuando se utiliza una palabra clave return dentro de una función anónima para salir antes de tiempo. Los retornos no locales se implementan lanzando y capturando scala.runtime.NonLocalReturnException-s. Esto es raramente lo que pretende el programador. Puede ser problemático debido al coste de rendimiento oculto de lanzar y capturar excepciones.

Además, es una implementación con fugas: un manejador de excepciones catch-all puede interceptar un NonLocalReturnException.


def foo(números: Seq[Int]): Int =
  números.map { n =>
    si (n > 5)
      return 0 // resaltado
    n * 2
  }.sum
