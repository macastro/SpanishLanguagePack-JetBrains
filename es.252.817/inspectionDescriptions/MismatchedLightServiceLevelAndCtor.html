Informa sobre discrepancias entre el nivel del servicio ligero y sus constructores.

Se informará sobre los siguientes problemas:

  • Una clase de servicio ligero con un constructor que tiene dos parámetros de tipo com.intellij.openapi.project.Project y kotlinx.coroutines.CoroutineScope o un parámetro de tipo Project no está anotada como servicio a nivel de proyecto.
  • Una clase de servicio ligero anotada como servicio a nivel de aplicación no tiene un constructor sin parámetros ni un constructor con un parámetro de tipo CoroutineScope.

Ejemplo (Kotlin):


@Service                              // Se sugiere especificar el parámetro 'Service.Level.PROJECT' en la anotación '@Service'
class MyService(val project: Project) // Se sugiere eliminar el parámetro del constructor 'MyService'

Después de aplicar la corrección rápida para especificar el parámetro 'Service.Level.PROJECT' en la anotación '@Service':


@Service(Service.Level.PROJECT)
class MyService(val project: Project)

Después de aplicar la corrección rápida para eliminar el parámetro del constructor 'MyService':


@Service
class MyService()
