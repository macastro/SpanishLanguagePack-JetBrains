Informa sobre anotaciones que se utilizan en componentes de registros pero no tienen efecto.

Esto puede ocurrir en dos casos:

  • La anotación reportada tiene un objetivo METHOD, pero se ha definido explícitamente un accessor correspondiente.
  • La anotación reportada tiene un objetivo PARAMETER, pero se ha definido explícitamente un constructor canónico.

Ejemplo:


@Target(ElementType.METHOD)
@interface A { }
// Esta anotación no aparecerá en el bytecode,
// ya que debería propagarse al accessor, pero el accessor está definido explícitamente
record R(@A int x) {
  public int x() { return x; }
}
