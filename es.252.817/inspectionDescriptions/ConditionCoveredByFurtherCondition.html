Informa sobre condiciones redundantes donde las condiciones posteriores cubren completamente las anteriores.

Por ejemplo, en la condición value != -1 && value > 0, la primera parte es redundante: si es false, la segunda parte también será false. O en una condición como obj != null && obj instanceof String, la comprobación de null es redundante, ya que el operador instanceof implica que no es null.
