Informa condiciones que se vuelven redundantes porque están completamente cubiertas por una condición posterior.

Por ejemplo, en la condición value != -1 && value > 0 la primera parte es redundante: si es falsa, entonces la segunda parte también es falsa. O en una condición como obj != null && obj instanceof String, la verificación nula es redundante ya que el operador instanceof implica no nulidad.
