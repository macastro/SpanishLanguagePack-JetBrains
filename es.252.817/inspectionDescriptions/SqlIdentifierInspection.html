Informa sobre casos donde se utilizan palabras reservadas de SQL como nombres de identificadores en las consultas.

Ejemplo (Microsoft SQL Server):

CREATE TABLE select (identity INT IDENTITY NOT NULL, order INT NOT NULL);

Usamos select, identity y order como nombres de tabla y columnas. Sin embargo, estas son palabras reservadas en Microsoft SQL Server. Por lo tanto, para utilizarlas como nombres de objetos en consultas, debe entrecomillar estos identificadores. Para entrecomillarlos, puede usar la corrección rápida Entrecomillar identificadores.

Después de aplicar la corrección rápida:

CREATE TABLE [select] ([identity] INT IDENTITY NOT NULL, [order] INT NOT NULL);
