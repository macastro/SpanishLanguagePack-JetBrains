Informa de constructores primarios vacíos cuando están disponibles implícitamente de todos modos.

Un constructor primario es redundante y se puede omitir de forma segura cuando no tiene anotaciones ni modificadores de visibilidad. Utilice la corrección rápida "Eliminar constructor primario vacío" para limpiar el código.

Ejemplos:


  class MyClassA constructor() // redundante, se puede reemplazar con 'class MyClassA'

  annotation class MyAnnotation
  class MyClassB @MyAnnotation constructor() // requerido debido a la anotación

  class MyClassC private constructor() // requerido debido al modificador de visibilidad
