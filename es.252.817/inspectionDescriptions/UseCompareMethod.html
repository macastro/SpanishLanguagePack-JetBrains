Reporta expresiones que pueden ser reemplazadas por una llamada al método Integer.compare() o un método similar de las clases Long, Short, Byte, Double o Float, en lugar de construcciones más extensas o menos eficientes.

Si x e y son números enteros en caja, entonces se sugiere x.compareTo(y), si son primitivos se sugiere Integer.compare(x, y).

Ejemplo:


  public int compare(int x, int y) {
    return x > y ? 1 : x < y ? -1 : 0;
  }

Después de aplicar la solución rápida:


  public int compare(int x, int y) {
    return Integer.compare(x, y);
  }

Tenga en cuenta que Double.compare y Float.compare cambian ligeramente la semántica del código. En particular, hacen que -0.0 y 0.0 sean distinguibles (Double.compare(-0.0, 0.0) produce -1). Además, procesan consistentemente el valor NaN. En la mayoría de los casos, este cambio de semántica en realidad mejora el código. Utilice la casilla de verificación para deshabilitar esta inspección para números de punto flotante si el cambio de semántica es inaceptable en su caso.
