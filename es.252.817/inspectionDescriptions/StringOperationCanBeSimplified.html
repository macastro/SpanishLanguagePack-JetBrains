Informa sobre llamadas redundantes a constructores y métodos de String que pueden ser reemplazados por expresiones más simples, como toString() o substring().

Por ejemplo, en casos como "string".substring(0), "string".toString() o new StringBuilder().toString().substring(1,3), las llamadas a estos métodos pueden eliminarse de forma segura.

Ejemplo:


  System.out.println(new String("message"));

Después de aplicar la corrección rápida:


  System.out.println("message");

Tenga en cuenta que esta corrección rápida elimina las llamadas redundantes al constructor, lo que puede afectar a la igualdad de referencias de String. Si necesita mantenerlas, aunque se considere una mala práctica, deshabilite la advertencia o use la configuración de inspección para ignorar las llamadas redundantes al constructor de String.

Use la opción No informar sobre llamadas al constructor String para evitar informes sobre código como el mostrado arriba. Esto evitará que los resultados de las comparaciones de cadenas con == o != cambien después de aplicar la corrección rápida en código que usa llamadas a new String() para garantizar la desigualdad de objetos.

Use la opción No informar sobre llamadas substring() con un argumento para evitar informes sobre código como el siguiente. 


  stringBuilder.append(string.substring(5));

Puede reemplazarse por:


  stringBuilder.append(string, 5, string.length());
