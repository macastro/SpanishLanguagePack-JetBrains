Informe sobre el constructor runBlocking alcanzable desde corrutinas.

Las corrutinas comparten el hilo de ejecución. Cuando se llama al constructor runBlocking desde una corrutina, este bloquea el hilo subyacente e impide que otras corrutinas utilicen este recurso. Esto puede causar problemas de rendimiento y, en el peor de los casos, puede provocar la inanición del hilo.

Proporciona una pila de llamadas potencial desde primitivas de corrutinas (funciones suspendidas o constructores) hasta el constructor runBlocking.

Ejemplo:


  suspend fun main() {
      foo() 
   }
  fun foo() {
      runBlocking { suspendFunction() }
  }

Posible solución:


  suspend fun main() {
      foo() 
   }
  suspend fun foo() {
      suspendFunction()
  }

Use la opción Examinar funciones con sobreescrituras para configurar el análisis de funciones virtuales:

  • Use la opción No para omitir el análisis de funciones con sobreescrituras.
  • Use la opción Sí, excluir sobreescrituras para examinar solo las funciones en el tipo de receptor definido.
  • Use la opción Sí, incluir sobreescrituras para examinar las funciones y todas sus sobreescrituras.
