Informa sobre posibles llamadas al método wait() cuando el hilo actual mantiene dos bloqueos.

La llamada a wait() solo libera un bloqueo en el objeto objetivo, por lo que esperar mientras se mantienen dos bloqueos puede conducir fácilmente a un interbloqueo.

Ejemplo:


  synchronized (lockA) {
    synchronized (lockB) {
      lockB.wait(); //advertencia
      //El hilo A se bloquea aquí, manteniendo lockA
    }
  }
  synchronized (lockA) { //El hilo B no puede entrar en este bloque y liberar el hilo A
    lockB.notify();
  }
