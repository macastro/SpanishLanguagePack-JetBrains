Informa sobre los campos que se pueden establecer de forma segura como final. Todos los campos final tienen un valor que no cambia, por lo que es más fácil inferir el código.

Para evitar que el análisis sea demasiado costoso, esta inspección únicamente informa sobre si el campo tiene un modificador private o si se define en una clase local o anónima. Un campo puede ser final si cumple las siguientes condiciones:

  • Es static y se inicializa solo una vez en su declaración o en un inicializador static.
  • No es static y se inicializa solo una vez en su declaración, en un inicializador de instancia o en todos los constructores.
También, no se modifica en ningún otro lugar el campo.

Ejemplo:


  public class Person {
    private String name; // Puede ser final

    Person(String name) {
      this.name = name;
    }

    public String getName() {
      return name;
    }
  }

Después de aplicar una corrección rápida:


  public class Person {
    private final String name;

    Person(String name) {
      this.name = name;
    }

    public String getName() {
      return name;
    }
  }

Utiliza el botón "Anotaciones" para modificar la lista de anotaciones que pretende escritura de campos implícita.
