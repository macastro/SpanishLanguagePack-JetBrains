Informa sobre campos que pueden declararse como final de manera segura. Todos los campos final deberían tener un valor que no cambie, lo que hace que el código sea más fácil de razonar.

Para evitar una sobrecarga excesiva en el análisis, esta inspección solo informa sobre campos si tienen el modificador private o si están definidos en una clase local o anónima. Un campo puede ser final si:

  • El campo es static y se inicializa una vez en su declaración o en un inicializador static.
  • El campo es no static y se inicializa una vez en su declaración, en un inicializador de instancia o en cada constructor
y el campo no se modifica en ningún otro lugar.

Ejemplo:


  public class Person {
    private String name; // puede ser final
    Person(String name) {
      this.name = name;
    }
    public String getName() {
      return name;
    }
  }

Después de aplicar la corrección rápida:


  public class Person {
    private final String name;
    Person(String name) {
      this.name = name;
    }
    public String getName() {
      return name;
    }
  }
