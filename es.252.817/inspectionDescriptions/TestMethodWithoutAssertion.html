Informa sobre métodos de prueba que no contienen ninguna aserción. Tales métodos pueden indicar que los casos de prueba están incompletos o son débiles.

Ejemplo:


  public class ExtensiveTest {

    @Test
    public void testAlive() {
      System.out.println("nothing");
    }
  }

Configura la inspección:

  • Usa la tabla para especificar las combinaciones de nombre de clase completamente calificado y expresión regular de nombre de método que deberían calificar como aserciones. Los nombres de clase también coinciden con subclases.
  • Usa la opción La palabra clave 'assert' se considera una aserción para especificar si las declaraciones assert de Java que usan la palabra clave assert deberían considerarse una aserción.
  • Usa la opción Ignorar métodos de prueba que declaran excepciones para ignorar métodos de prueba que declaran excepciones. Esto puede resultar útil cuando tienes pruebas que arrojarán una excepción al producirse un error y, por lo tanto, no necesitan aserciones.
