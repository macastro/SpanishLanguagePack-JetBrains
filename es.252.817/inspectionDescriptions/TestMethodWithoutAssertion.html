Informa sobre métodos de prueba que no contienen ninguna aserción. Tales métodos pueden indicar casos de prueba incompletos o débiles.

Ejemplo:


  public class ExtensiveTest {
    @Test
    public void testAlive() {
      System.out.println("nothing");
    }
  }

Configuración de la inspección:

  • Use la tabla para especificar combinaciones de nombres de clase completamente calificados y expresiones regulares de nombres de métodos que deben considerarse como aserciones. Los nombres de clase también coinciden con las subclases.
  • Use la opción considerar la palabra clave 'assert' como aserción para especificar si las declaraciones Java assert que utilizan la palabra clave assert deben considerarse como aserciones.
  • Use la opción ignorar métodos de prueba que declaran excepciones para ignorar los métodos de prueba que declaran excepciones. Esto es útil cuando sus pruebas lanzan excepciones al fallar y por lo tanto no necesitan ninguna aserción.
