Reporta clases anónimas.

Ocasionalmente, reemplazar las clases anónimas con clases internas puede conducir a un código más legible y mantenible. Algunos estándares de código desaconsejan las clases anónimas.

Ejemplo:


  class Example {
    public static void main(String[] args) {
      new Thread() {
        public void run() {
          work()
        }

        private void work() {}
      }.start();
    }
  }

Después de aplicar el arreglo rápido:


  class Example {
    public static void main(String[] args) {
      new MyThread().start();
    }

    private static class MyThread extends Thread {
      public void run() {
        work();
      }

      private void work() {}
    }
  }
