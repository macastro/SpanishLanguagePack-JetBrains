El informe puede reemplazar las constantes de conjuntos de caracteres String literales (por ejemplo, "UTF-8") con el código predefinido StandardCharsets.UTF_8 en métodos y constructores.

La corrección puede acelerar el código ya que la búsqueda del conjunto de caracteres se vuelve innecesaria. Además, es posible que no necesite capturar UnsupportedEncodingException. En este caso, el bloque catch se eliminará automáticamente.

Ejemplo:


  try {
    byte[] bytes = "str".getBytes("UTF-8");
  } catch (UnsupportedEncodingException e) {
  }

Después de aplicar la corrección rápida:


  byte[] bytes = "str".getBytes(StandardCharsets.UTF_8);

Esta inspección está disponible en Java 7 y versiones posteriores.
