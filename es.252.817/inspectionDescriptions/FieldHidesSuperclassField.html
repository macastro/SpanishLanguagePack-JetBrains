Reporta los campos de una clase derivada que tienen el mismo nombre que un campo de una superclase. Los campos de Java no se pueden sobreescribir en clases derivadas, por lo que el campo de la clase derivada ocultará el campo de la superclase.

Como resultado de dicha denominación, se puede utilizar accidentalmente el campo de la clase derivada cuando se pretende utilizar el campo con el mismo nombre de una clase base.

Se sugiere una corrección rápida para cambiar el nombre del campo en la clase derivada.

Ejemplo:

class Padre {
  Padre padre;
}
class Hijo extiende Padre {
  Hijo padre;
}

Se pueden configurar las siguientes opciones para esta inspección:

  1. Ignorar los campos no accesibles: indica si esta inspección debe informar todas las coincidencias de nombres, o sólo las coincidencias con campos que son visibles desde la subclase.
  2. Ignorar los campos estáticos que ocultan campos estáticos - ignorar los campos estáticos que ocultan los campos estáticos en las clases base.
