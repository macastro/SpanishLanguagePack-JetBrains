Informa sobre campos transient que se inicializan durante la construcción normal del objeto pero cuya clase no tiene un método readObject.

Los campos transient no se serializan, por lo que durante la deserialización necesitan inicializarse específicamente en el método readObject().

Cualquier campo transient que no se inicialice durante la construcción normal del objeto se considera que usa la inicialización por defecto y no será reportado por esta inspección.

Ejemplo:


  class Person implements Serializable {
    transient String name = "Default"; //advertencia: podría ser null después de la deserialización
    transient String surname; //null se considera valor por defecto, no se reporta
  }
