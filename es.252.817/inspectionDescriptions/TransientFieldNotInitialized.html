Reporta campos transient que son inicializados durante la construcción normal de un objeto pero cuya clase no tiene un método readObject.

Como los campos transient no son serializados, deben inicializarse separadamente en un método readObject() durante la deserialización.

Cualquier campo transient que no se inicialice durante la construcción normal del objeto se considera que utiliza la inicialización predeterminada y este control no lo reportará.

Ejemplo:


  class Persona implements Serializable {
    transient String nombre = "Defecto"; //advertencia, realmente puede ser nulo tras la deserialización
    transient String apellido; //nulo se considera el valor predeterminado y no se reporta
  }
