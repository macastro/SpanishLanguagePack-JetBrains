Informa sobre anotaciones @OptIn innecesarias que pueden eliminarse de forma segura.

El código que utiliza APIs experimentales requiere la anotación @OptIn porque estas APIs pueden cambiar en cualquier momento. Si dichas APIs ya no se utilizan (por ejemplo, cuando una API experimental se estabiliza y ya no requiere opt-in), esta anotación se vuelve redundante y potencialmente engañosa.

La corrección rápida Eliminar anotación está disponible para eliminar anotaciones @OptIn innecesarias.

Ejemplo:


  @OptIn(ExperimentalApi::class)
  fun foo(x: Bar) {
      x.baz()
  }

Después de aplicar la corrección rápida:


  fun foo(x: Bar) {
      x.baz()
  }
