Informa sobre clases que implementan la interfaz Cloneable pero no sobrescriben el método clone().

Estas clases utilizan la implementación predeterminada de clone(), que no es public sino protected, y no copia el estado mutable de la clase.

Se puede generar un método clone() básico mediante una corrección rápida, que puede servir como base para un método clone() funcional como se espera en las clases Cloneable.

Ejemplo:


  public class Data implements Cloneable {
    private String[] names;
  }

Después de aplicar la corrección rápida:


  public class Data implements Cloneable {
    private String[] names;
    @Override
    public Data clone() {
      try {
        Data clone = (Data) super.clone();
        // TODO: Copiar el estado mutable aquí, para que el clon no pueda modificar los elementos internos del original
         return clone;
      } catch (CloneNotSupportedException e) {
        throw new AssertionError();
      }
    }
  }

Use la opción Ignorar clases clonables por herencia para ignorar las clases que son Cloneable por heredar de una clase Cloneable.

Use la opción Ignorar casos donde Cloneable es necesario para llamar al clone() de la superclase para ignorar las clases que necesitan implementar Cloneable debido a llamadas al método clone() desde la superclase.
