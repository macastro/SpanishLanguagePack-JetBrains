count() llama a map() o boxed() justo antes, reportando esto como un llamado redundante de la API de Stream.

Este tipo de llamados se pueden eliminar porque no modifican el conteo final. Es posible que el código dependa de los efectos secundarios de algún lambda dentro de este tipo de llamadas de map. Sin embargo, depender de los efectos secundarios de una cadena de stream es una muy mala práctica. No hay garantía de que este llamado no sea optimizado en versiones futuras de Java.

Ejemplo:


  // El llamado a `map()` es redundante
  long count = list.stream().filter(s -> !s.isEmpty()).map(s -> s.trim()).count();
