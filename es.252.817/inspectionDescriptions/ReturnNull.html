Informa sobre las declaraciones return que devuelven null. Aunque ocasionalmente útil, esta estructura puede hacer que el código falle debido a que es más propenso a NullPointerException.

Si un método está diseñado para devolver null, se recomienda marcarlo con la anotación @Nullable - los métodos así marcados serán ignorados por esta inspección.

Ejemplo:


  class Person {
    public String getName () {
      return null;
    }
  }

Después de aplicar la corrección rápida:


  class Person {
    public @Nullable String getName () {
      return null;
    }
  }

Si el tipo de retorno es java.util.Optional, se recomienda usar otra corrección rápida para convertir null en Optional.empty().

Configure la inspección con las siguientes opciones:

  • Si se deben ignorar los métodos private. Esto también ignorará los valores null devueltos desde clases anónimas y lambdas.
  • Si se deben informar los valores null devueltos para arrays, objetos de colección, objetos regulares o una combinación de los tres. Siempre se informarán los valores null devueltos en métodos con tipo de retorno java.util.Optional.
  • Haga clic en Configurar anotaciones para especificar qué anotaciones deben considerarse como 'nullable'.
