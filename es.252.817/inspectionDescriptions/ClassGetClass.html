Informa sobre el método getClass() cuando se llama en instancias de java.lang.Class.

Esto suele ser un error, ya que el resultado siempre es equivalente a Class.class. Si es un error, es mejor eliminar la llamada a getClass() y usar el calificador directamente. Si el comportamiento es intencional, es mejor escribir explícitamente Class.class para evitar confusiones.

Ejemplo:


  void test(Class<?> clazz) {
    String name = clazz.getClass().getName();
  }

Después de aplicar una de las posibles correcciones rápidas:


  void test(Class<?> clazz) {
    String name = clazz.getName();
  }
