Reporta expresiones lógicas ambiguas en ramas de when que causarán errores de compilación en Kotlin 1.8 y versiones posteriores.


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      this in (4..7) -> true // ambiguo
      else -> false
  }

Después de aplicar la corrección rápida:


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      (this in (4..7)) -> true // envuelto en paréntesis
      else -> false
  }

La verificación está disponible para el nivel de lenguaje Kotlin desde la versión 1.7.
