Informa accesos a campos private o protected de otro objeto. Java permite el acceso a dichos campos para objetos de la misma clase que el objeto actual, pero algunos estilos de codificación desaconsejan este uso. Además, dicho acceso directo a campos private puede fallar en arquitecturas orientadas a componentes, como Spring o Hibernate, que esperan que todos los accesos a otros objetos sean a través de llamadas a métodos para que el framework pueda mediar en el acceso utilizando proxies.

Ejemplo:


  public class Base {
    protected int bar;

    void increment(Base base) {
      bar++;
      base.bar++; // warning: direct access to another object's non-public field
    }
  }
Hay una corrección rápida disponible para encapsular el campo.

Configurar la inspección:

  • Utilice la opción Ignorar accesos desde la misma clase para ignorar el acceso desde la misma clase y solo informar el acceso desde clases internas o externas.

    Para ignorar también el acceso desde clases internas, utilice el anidado Ignorar accesos desde clases internas.

  • Utilice Ignorar accesos desde el método 'equals()' para ignorar el acceso desde un método equals().
