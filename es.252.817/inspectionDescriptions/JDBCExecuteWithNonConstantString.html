Informa sobre llamadas a java.sql.Statement.execute() o cualquiera de sus variantes que toman una cadena construida dinámicamente como la consulta a ejecutar.

Las sentencias SQL construidas son una fuente común de brechas de seguridad. Por defecto, esta inspección ignora las constantes de tiempo de compilación.

Ejemplo:


  ResultSet execute(Statement statement, String name) throws SQLException {
    return statement.executeQuery("select * from " + name); // informa de una advertencia
  }

Utiliza las opciones de inspección para considerar cualquier campo static final como constante. Ten cuidado, porque cadenas como la siguiente se ignorarán cuando la opción esté habilitada: 


  private static final String SQL = "SELECT * FROM user WHERE name='" + getUserInput() + "'";
