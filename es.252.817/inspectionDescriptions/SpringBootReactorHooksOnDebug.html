Reporta el uso de Hooks.onOperatorDebug() cuando se llama a ReactorDebugAgent.init().

Si ReactorDebugAgent está en el classpath, y spring.reactor.debug-agent.enabled es true (valor por defecto), ReactorDebugAgent.init() se llama automáticamente en el arranque. Usando tanto Hooks.onOperatorDebug() como ReactorDebugAgent.init() causa dobles frames de stack de depuración generados por Reactor. Además, Hooks.onOperatorDebug() puede causar gastos de performance.

El quick-fix remueve la invocación de Hooks.onOperatorDebug().

Ejemplo:


  public void hook() {
    Hooks.onOperatorDebug(); // Reporta 'Llama a Hooks.onOperatorDebug() mientras ReactorDebugAgent está inicializado'
  }

Después de aplicar el quick-fix:


  public void hook() {
  }

Esta inspección solo se dispara cuando la versión de Spring Boot para el proyecto o módulo es 2.2.0 o mayor.
