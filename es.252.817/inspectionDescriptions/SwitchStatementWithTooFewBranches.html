Informa sobre declaraciones y expresiones switch con muy pocas etiquetas case y sugiere reescribirlas como declaraciones if y else if.

Ejemplo (mínimo de ramas == 3):


  switch (expression) {
    case "foo" -> foo();
    case "bar" -> bar();
  }

Después de aplicar la corrección rápida:


  if ("foo".equals(expression)) {
    foo();
  } else if ("bar".equals(expression)) {
    bar();
  }

No se informan las expresiones switch exhaustivas (Java 14+) ni las declaraciones switch de patrones sin rama 'default' (versión preliminar de Java 17). Esto se debe a que cuando se convierte un switch a if, se pierde la verificación de exhaustividad en tiempo de compilación (lo cual puede ser un resultado no deseado).

Configuración de la inspección:

Utilice el campo número mínimo de ramas para especificar la cantidad mínima esperada de etiquetas case.

Utilice la opción no informar sobre declaraciones switch de patrones para evitar informes sobre declaraciones y expresiones switch que contengan ramas de patrones. Por ejemplo:


  String result = switch(obj) {
    case String str -> str.trim();
    default -> "none";
  };

El switch debe mantenerse incluso con una sola rama de patrón, en lugar de usar una declaración instanceof.
