Informa sobre anotaciones @GuardedBy donde el campo protegido no es final.

La protección en campos no finales puede llevar a condiciones de carrera inesperadas, ya que se bloquea el valor del campo (que puede cambiar) en lugar de bloquear el campo en sí.

Ejemplo:


    private ReadWriteLock lock = new ReentrantReadWriteLock(); //campo protegido no final
    private Object state;
    @GuardedBy("lock")
    public void bar() {
        state = new Object();
    }

Las anotaciones @GuardedBy soportadas son:

  • net.jcip.annotations.GuardedBy
  • javax.annotation.concurrent.GuardedBy
  • org.apache.http.annotation.GuardedBy
  • com.android.annotations.concurrency.GuardedBy
  • androidx.annotation.GuardedBy
  • com.google.errorprone.annotations.concurrent.GuardedBy
