Informa sobre las conversiones implícitas que ocurren entre los tipos XPath predefinidos STRING, NUMBER, BOOLEAN y NODESET. Ayuda a escribir scripts XSLT con un comportamiento de tipos más explícito y evita errores difíciles de detectar.

Ejemplo:


  <xsl:if test="foo" />
es diferente de 

  <xsl:if test="string(foo)" />

La primera prueba verifica si existe el elemento "foo" (count(foo) > 0)), mientras que la segunda prueba será "true" solo cuando el elemento contenga texto (string-length(foo) > 0). Se recomienda hacer las conversiones de tipo más explícitas.

Configure la inspección usando las siguientes opciones:

  • Habilitar o deshabilitar conversiones implícitas entre ciertos tipos
  • Informar siempre sobre conversiones explícitas que no resultan en el tipo esperado real, por ejemplo, <xsl:if test="number(foo)" />
  • Ignorar las conversiones de NODESET a BOOLEAN cuando se usa la función string() como un atajo para escribir string-length() > 0.
