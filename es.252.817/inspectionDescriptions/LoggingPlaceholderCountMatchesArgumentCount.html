Informa sobre llamadas de registro SLF4J, Log4j2 y akka.event.LoggingAdapter, como logger.info("{}: {}", key), donde el número de marcadores de posición {} en el mensaje del logger no coincide con el número de argumentos adicionales en la llamada de registro.

Esta opción de inspección permite especificar qué implementación usa SLF4J. Si se selecciona Detectar automáticamente, se busca org.apache.logging.slf4j.Log4jLogger en el classpath. Si se encuentra este archivo o se selecciona Sí, no se informarán los casos con marcadores de posición para el último parámetro con tipo de excepción para la API de SLF4J.
Por ejemplo: 


//Este caso no se informará si la opción "Sí" está seleccionada
log.error("For id {}: {}", "1", new RuntimeException());
En este caso, 'new RuntimeException()' se imprimirá usando 'toString()' (sin imprimir su traza de pila): 

For id 1: java.lang.RuntimeException
De lo contrario, se resaltará porque el último marcador de posición no se utiliza: 

For id 1: {}
java.lang.RuntimeException: null
La opción No se puede usar para resaltar siempre estos casos cuando se usa un marcador de posición para una excepción, incluso cuando org.apache.logging.slf4j.Log4jLogger se usa como backend.
Esta opción solo se aplica a SLF4J.
