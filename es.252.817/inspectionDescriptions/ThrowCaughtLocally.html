Informa sentencias throw cuyas excepciones siempre son atrapadas por las sentencias try que las contienen.

El uso de sentencias throw como "goto" para cambiar el flujo de control local genera confusión y da como resultado un mal rendimiento.

Ejemplo:


  try {
    if (!Files.isDirectory(PROJECTS)) {
        throw new IllegalStateException("Directory not found."); // advertencia: 'throw' capturado por la sentencia 'try' que lo contiene
    }
    ...
  } catch (Exception e) {
      LOG.error("run failed");
  }

Utiliza la opción Ignorar excepciones relanzadas para ignorar las excepciones que se relanzan.
