Reporta las llamadas a wait(), notify() y notifyAll() que no se realizaron dentro de una declaración synchronized correspondiente o un método synchronized.

Llamar a estos métodos en un objeto sin mantener un bloqueo en ese objeto provoca IllegalMonitorStateException. Tal construcción no es necesariamente un error, ya que el bloqueo necesario puede adquirirse antes de que se llame al método contenedor, pero vale la pena analizarlo.

Ejemplo:


  class Sync {
    private final Object lock = new Object();

    void test() throws InterruptedException {
      synchronized (this) {
        lock.wait(); // 'lock.wait()' no está sincronizado con 'lock'
      }
    }
  }
