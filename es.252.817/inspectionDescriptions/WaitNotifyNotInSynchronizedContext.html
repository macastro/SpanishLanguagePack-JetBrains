Reporta llamadas a wait(), notify() y notifyAll() que no se realizan dentro de una sentencia synchronized o método synchronized correspondiente.

Llamar a estos métodos en un objeto sin poseer su bloqueo provocará una IllegalMonitorStateException. Estas construcciones no son necesariamente erróneas, ya que el bloqueo necesario podría haberse adquirido antes de llamar al método contenedor, pero aun así merecen una revisión cuidadosa.

Ejemplo:


  class Sync {
    private final Object lock = new Object();
    void test() throws InterruptedException {
      synchronized (this) {
        lock.wait(); // 'lock.wait()' no está sincronizado en 'lock'
      }
    }
  }
