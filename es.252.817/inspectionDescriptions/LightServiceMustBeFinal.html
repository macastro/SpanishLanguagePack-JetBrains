Los servicios que no son finales pueden ser modificados y su estado puede no ser consistente.

Inspecciona el uso de clases con la anotación @com.intellij.openapi.components.Service que no son finales.

Si la clase es concreta, se sugiere hacerla final.

Ejemplo:


  // MyService.kt
  @Service(Service.Level.APP)
  open class MyService

Después de aplicar la reparación rápida:


  // MyService.kt
  @Service(Service.Level.APP)
  class MyService

Si es una clase abstracta o una interfaz, se sugiere quitar la anotación @Service.

Ejemplo:


  // MyService.java
  @Service(Service.Level.APP)
  abstract class MyService {}

Después de aplicar la reparación rápida:


  // MyService.java
  abstract class MyService {}
