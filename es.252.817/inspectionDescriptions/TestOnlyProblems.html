Informa sobre métodos y clases que utilizan las anotaciones @TestOnly y @VisibleForTesting en código de producción. También informa sobre los usos donde @TestOnly y @VisibleForTesting se aplican al mismo elemento.

No se informan problemas si los métodos o clases se referencian desde:

  • Código en la carpeta Test Sources
  • Clases de prueba (JUnit/TestNG)
  • Otro método anotado con @TestOnly

Ejemplo (en código de producción):


  @TestOnly
  fun foo() { ... }
  fun main () {
    foo()
  }
