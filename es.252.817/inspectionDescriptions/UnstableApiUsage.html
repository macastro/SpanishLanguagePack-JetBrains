Informa sobre el uso de APIs marcadas como inestables mediante anotaciones. Estas APIs pueden ser modificadas o eliminadas en versiones futuras, lo que podría romper el código que las utiliza.

Las anotaciones utilizadas para marcar APIs inestables se muestran en la siguiente lista.

Por defecto, esta inspección ignora el uso de APIs inestables cuando su declaración se encuentra en el mismo proyecto fuente. En estos casos, los usos pueden actualizarse cuando se modifica la API. Sin embargo, esto puede no ser conveniente en proyectos grandes, por lo que puede desactivar la opción Ignorar APIs declaradas en este proyecto para que se informen las APIs inestables declaradas tanto en el código fuente del proyecto como en las bibliotecas.
