Reportar setter de propiedades redundantes.

Un setter se considera redundante en uno de los siguientes casos:

  1. El setter no tiene cuerpo. La visibilidad del accesor no se modifica, la declaración no es external y no hay anotaciones. 
    
  var myPropWithRedundantSetter: Int = 0
      set // redundant
  var myPropA: Int = 0
      private set // permitido - la visibilidad de la propiedad se cambia a private
  var myPropB: Int = 0
      external set // permitido - no implementado en Kotlin (externo)
  var myPropC: Int = 0
      @Inject set // permitido - el accesor está anotado
  2. El cuerpo del setter es un bloque con una única sentencia que asigna el parámetro al campo de respaldo. 
    
  var prop: Int = 0
      set(value) { // redundante
          field = value
      }
