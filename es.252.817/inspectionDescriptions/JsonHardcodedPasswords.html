Inspección para detectar posibles tokens de seguridad o contraseñas que se encuentran en formato json mediante el análisis de la entropía y expresiones regulares.

Esta inspección examina el código fuente utilizando el análisis de la entropía y las expresiones regulares para escanear cadenas de caracteres similares a los tokens de seguridad o contraseñas. Resalta los resultados detectados para permitir que los desarrolladores identifiquen y protejan las vulnerabilidades potenciales. La efectividad de esta inspección depende de los patrones definidos en su configuración, lo que le permite adaptarse a diferentes entornos de codificación y requisitos. 


// Ejemplos de cadenas de caracteres que se detectarán como tokens potenciales:
'possibleToken': 'pa#sw@rd;'

// Ejemplos de patrones de expresiones regulares que se utilizan para la detección:
regexPattern = '/[0-9]+:AA[0-9A-Za-z\-_]{33}/'
