Informa sobre el uso innecesario de esquemas `grpc` o `http[s]` estándar en solicitudes de gRPC

Ejemplos de solicitudes: 


# El esquema `grpc` puede omitirse, ya que se asume una conexión insegura por defecto
GRPC grpc://localhost/TestService/testRpc


# El esquema `http` puede omitirse: se prefiere un esquema vacío para una conexión insegura y un esquema `grpcs` dedicado para una segura
GRPC http://localhost/TestService/testRpc


# El esquema `https` debe reemplazarse por `grpcs`: se prefiere un esquema `grpcs` dedicado para indicar que se debe habilitar la seguridad de la capa de transporte para ejecutar la solicitud
GRPC https://localhost/TestService/testRpc

Para evitar confusiones, se recomienda utilizar un esquema `grpcs` dedicado en una solicitud de gRPC, cuando la solicitud debe utilizar un canal seguro por debajo. De lo contrario, el esquema podría omitirse por completo
