Reporta el acceso a campos static que son de un tipo que no es seguro para la ejecución en múltiples subprocesos.

Cuando un campo static se accede desde un método de instancia o un bloque no sincronizado, múltiples subprocesos pueden acceder a ese campo. Esto puede provocar efectos secundarios no especificados, como excepciones y resultados incorrectos.

Ejemplo:


  class Sample {
    private static final SimpleDateFormat df = new SimpleDateFormat("yyyy-MM-dd");
    String method() {
      return df.format("");
    }
  }

Puedes especificar qué tipos deberían considerarse no seguros para la ejecución en múltiples subprocesos. Solo se reportan los campos con estos tipos exactos o inicializados con estos tipos exactos, porque puede haber subclases seguras para la ejecución en múltiples subprocesos de estos tipos.
