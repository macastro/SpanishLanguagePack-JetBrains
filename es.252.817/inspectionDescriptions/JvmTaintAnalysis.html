Esta inspección está diseñada para identificar fragmentos de código potencialmente vulnerables, en los que datos procedentes de fuentes que podrían no ser fiables se utilizan en operaciones que suponen un riesgo para la seguridad.

Seguimiento del flujo de datos:

La inspección realiza un seguimiento del flujo de datos, desde su origen hasta su uso en operaciones críticas. Esto incluye el análisis de la transferencia de datos a través de métodos, objetos y variables.

Considere un ejemplo en el que la entrada de un usuario se utiliza directamente en una consulta SQL sin la debida sanitización. La inspección emitirá una advertencia sobre una posible inyección SQL.

String userInput = getUserInput();
String sqlQuery = "SELECT * FROM users WHERE username = '" + userInput + "'";
