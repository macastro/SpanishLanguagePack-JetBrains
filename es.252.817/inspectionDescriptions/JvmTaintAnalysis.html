Esta inspección está diseñada para identificar segmentos de código potencialmente vulnerables donde los datos obtenidos de fuentes potencialmente no confiables podrían utilizarse en operaciones que constituyen un riesgo de seguridad. La inspección se ha optimizado para su ejecución dentro del IDE mediante el emparejamiento de herramientas locales con análisis de parámetros restringidos.

Seguimiento del flujo de datos:

Esta inspección rastrea el flujo de datos desde su origen hasta su uso en operaciones críticas. Esto incluye el análisis de la transferencia de datos a través de métodos, objetos y variables.

Considere un ejemplo donde la entrada del usuario se utiliza directamente en una consulta SQL sin el preprocesamiento adecuado. La inspección emitirá una advertencia sobre una posible inyección SQL.

String userInput = getUserInput();
String sqlQuery = "SELECT * FROM users WHERE username = '" + userInput + "'";
