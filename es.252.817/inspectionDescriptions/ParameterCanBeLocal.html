Informa sobre parámetros de método redundantes que pueden ser reemplazados por variables locales.

Un parámetro puede ser eliminado si todos sus usos locales están precedidos inmediatamente por una asignación a ese parámetro. No tiene sentido tener tales parámetros ya que el valor que se les pasa será sobrescrito. Este problema suele ser el resultado de una refactorización.

Ejemplo:


  void test(int p) {
    p = 1;
    System.out.print(p);
  }

Después de aplicar la corrección rápida:


  void test() {
    int p = 1;
    System.out.print(p);
  }
