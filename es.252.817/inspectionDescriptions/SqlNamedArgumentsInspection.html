Reporta argumentos que se utilizan sin nombres en llamadas de rutinas. Por defecto, esta inspección está desactivada.

Para más información sobre la diferencia entre parámetros nombrados y no nombrados, consulte Enlace de parámetros por nombre (named parameters) en docs.microsoft.com .

Ejemplo (Microsoft SQL Server):

CREATE FUNCTION foo(n INT, m INT) RETURNS INT AS
BEGIN
    RETURN n + m;
END;

CREATE PROCEDURE test AS
BEGIN
    foo n = 1, m = 2;

--- La siguiente llamada omite los nombres de los parámetros y se resaltará
    foo 1, 2;
END;

Los parámetros 1, 2 en la llamada foo 1, 2; están resaltados porque carecen de nombres.
