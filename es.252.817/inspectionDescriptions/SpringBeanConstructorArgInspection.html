Reporta el uso de inyección de dependencias basada en constructores de Spring <bean>.

La inspección se activa cuando no se puede encontrar un constructor o método de fábrica adecuado para el <bean> utilizando las etiquetas <constructor-arg> configuradas y la estrategia de autowire definida.

Ejemplo:


    public class MyComponent {
    // 构造函数
    public MyComponent(MyBean bean) {}
    // 工厂方法
    public static MyComponent getInstance(String name, int port) {
    }


    <beans>
      <bean class="beans.MyComponent" id="foo"> <!-- Reporta "No se encuentra un constructor coincidente en la clase 'MyComponent'" -->
          <constructor-arg ref="myBean"/>
         <constructor-arg value="123"/>
      </bean>
      <bean class="beans.MyComponent" factory-method="getInstance"> <!-- Reporta "No se encuentra un método de fábrica coincidente en la clase" -->
         <constructor-arg value="123"/>
      </bean>
    </beans>
