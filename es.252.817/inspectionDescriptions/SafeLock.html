Se informan los recursos de java.util.concurrent.locks.Lock que no se adquieren antes de un bloque try o que no se desbloquean en el bloque finally correspondiente. Dichos recursos pueden perderse sin darse cuenta si se lanza una excepción antes de que se cierre el recurso.

Ejemplo:


  lock.lock(); // se informará ya que falta el bloque 'finally'
  try {
    doSmthWithLock();
  } catch (IOException e) {
    throw new UncheckedIOException(e);
  }
  lock.unlock();
