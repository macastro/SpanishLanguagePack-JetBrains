Reporta cláusulas `throws` con más de una excepción en métodos de prueba JUnit. Dichas cláusulas son innecesariamente extensas. Los métodos de prueba no serán llamados desde otro código del proyecto, por lo que no es necesario manejar estas excepciones por separado.

Por ejemplo:


  @Test
  public void testReflection() throws NoSuchMethodException,
      InvocationTargetException, IllegalAccessException {
    String result = (String) String.class.getMethod("trim")
        .invoke(" hello ");
    assertEquals("hello", result);
  }

Se proporciona una solución rápida para reemplazar las declaraciones de excepciones con una única excepción:


  @Test
  public void testReflection() throws Exception {
    String result = (String) String.class.getMethod("trim")
        .invoke(" hello ");
    assertEquals("hello", result);
  }
