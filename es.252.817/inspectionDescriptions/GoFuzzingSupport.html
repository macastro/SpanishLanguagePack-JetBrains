Reportar pruebas de fuzzing cuando la versión del SDK de Go es inferior a 1.18

El fuzzing es un método de prueba automatizado que implica una búsqueda dirigida de datos de entrada que podrían causar fallos en el programa o exponer comportamientos inválidos. Go admite fuzzing a partir de la versión 1.18.

Ejemplo de prueba de fuzzing:


func Div(a, b int) int {
	return a / b
}
func FuzzDiv(f *testing.F) {
	f.Fuzz(func(t *testing.T, a, b int) {
		Div(a, b) // reporta error en tiempo de ejecución: división entre cero
	})
}

Consulte Go Fuzzing para más información.
