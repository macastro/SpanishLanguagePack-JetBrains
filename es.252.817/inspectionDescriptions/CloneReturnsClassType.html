Reporta métodos clone() cuyo tipo de retorno difiere de la clase en la que se encuentran.

El tipo de retorno del método clone() suele ser java.lang.Object, lo que dificulta su uso por parte del cliente. Effective Java (segunda y tercera edición) recomienda establecer el tipo de retorno del método clone() igual al tipo de la clase del objeto que devuelve.

Ejemplo:


  class Foo implements Cloneable {
    public Object clone() {
      try {
        return super.clone();
      } catch (CloneNotSupportedException e) {
        throw new AssertionError();
      }
    }
  }

Después de aplicar la corrección rápida:


  class Foo implements Cloneable {
    public Foo clone() {
      try {
        return (Foo)super.clone();
      } catch (CloneNotSupportedException e) {
        throw new AssertionError();
      }
    }
  }
