Informa sobre situaciones en las que el código en tiempo de ejecución impide que un framework (como Spring o Hibernate) realice la subclasificación de clases.

Ejemplos típicos donde la subclasificación es necesaria pero imposible:

  • Clases final marcadas con anotaciones específicas del framework (como Spring @Configuration)
  • Métodos final, static o private marcados con anotaciones específicas del framework (como Spring @Transactional)
  • Métodos marcados con anotaciones específicas del framework dentro de clases final

La lista de situaciones reportadas depende del framework utilizado.
