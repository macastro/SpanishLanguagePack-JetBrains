Informa argumentos de llamada con tipo Boolean sin nombres de parámetro explícitos especificados.

Cuando se transmiten múltiples literales booleanos secuencialmente, es fácil olvidar el orden de los parámetros, lo que podría llevar a errores. Los nombres de parámetros explícitos permiten una lectura y comprensión más fáciles del código.

Ejemplo:


  fun check(checkName: Boolean, checkAddress: Boolean, checkPhone: Boolean) {}

  fun usage() {
      check(true, false, true) // ¿Qué significa esto?
  }

La solución rápida agrega los nombres de parámetro faltantes:


  fun check(checkName: Boolean, checkAddress: Boolean, checkPhone: Boolean) {}

  fun usage() {
      check(checkName = true, checkAddress = false, checkPhone = true)
  }
