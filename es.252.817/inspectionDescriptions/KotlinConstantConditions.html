Informa sobre condiciones y valores no triviales que se sabe estáticamente que siempre son true, false, null o cero. Aunque a veces esto es intencional, generalmente es un indicio de un error lógico en el programa. Además, informa sobre ramas when que nunca son alcanzables y ciertas expresiones que se sabe estáticamente que siempre fallarán.

Ejemplo:


fun process(x: Int?) {
  val isNull = x == null
  if (!isNull) {
    if (x != null) {} // La condición siempre es true
    require(x!! < 0 && x > 10) // La condición siempre es false
  } else {
    println(x!!) // El operador !! siempre fallará
  }
}
fun process(v: Any) {
  when(v) {
    is CharSequence -> println(v as Int) // La conversión siempre fallará
    is String -> println(v) // La rama nunca será alcanzada
  }
}

Desmarca la opción "Emitir advertencias cuando las constantes se almacenan en variables" para evitar informes sobre variables con valores constantes que no están en condiciones.
