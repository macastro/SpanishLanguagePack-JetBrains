Informa sobre etiquetas de construcción (build tags) mal formateadas y mal ubicadas. La herramienta go requiere que las etiquetas de construcción se coloquen en ubicaciones específicas y sigan una sintaxis especial. Si no se cumplen estos requisitos, las etiquetas de construcción pueden ser ignoradas o los archivos pueden ser excluidos incorrectamente de la construcción.

Consulte Build Constraints en go.dev.

Ejemplo:

package main
// +build ignore
func main() {}

La sección // +build ignore debe estar ubicada antes de la declaración del paquete. Para corregir este problema, puede aplicar la corrección rápida colocar etiqueta de construcción antes del paquete. Después de aplicar la corrección rápida:

// +build ignore
package main
import "fmt"
