Informa sobre problemas relacionados con anotaciones de nulabilidad.

Ejemplos:

  • Método sobrescrito sin anotación: 
    
abstract class A {
  @NotNull abstract String m();
}
class B extends A {
    String m() { return "empty string"; }
}
  • Tipo primitivo anotado: @NotNull int myFoo;
  • @Nullable y @NotNull en el mismo miembro: @Nullable @NotNull String myFooString;
  • Colección de elementos nulables asignada a una colección de elementos no nulables: 
    
void testList(List<@Nullable String> nullableList) {
   List<@NotNull String> list2 = nullableList;
}

Utilice el botón Configurar anotaciones para especificar las anotaciones de nulabilidad y use las casillas de verificación para ajustar dónde la inspección proporciona advertencias.

Esta inspección solo informa cuando el nivel de lenguaje del proyecto o módulo es 5 o superior y las anotaciones de nulabilidad están disponibles en el classpath.
