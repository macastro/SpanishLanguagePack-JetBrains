Informa sobre problemas relacionados con las anotaciones de anulabilidad.

Ejemplos:

  • Los métodos que sobreescriben no están anotados: 
    
abstract class A {
  @NotNull abstract String m();
}
class B extends A {
    String m() { return "empty string"; }
}
  • Tipos primitivos anotados: @NotNull int myFoo;
  • @Nullable y @NotNull están presentes en el mismo miembro: @Nullable @NotNull String myFooString;
  • La colección de elementos anulables se asigna a una colección de elementos no anulables: 
    
void testList(List<@Nullable String> nullableList) {
   List<@NotNull String> list2 = nullableList;
}

Usa el botón Configurar anotaciones para especificar las anotaciones de anulabilidad y las casillas de verificación para ajustar dónde debe proporcionar advertencias la inspección.

Esta inspección solo informa si el nivel de lenguaje del proyecto o módulo es 5 o superior, y las anotaciones de anulabilidad están disponibles en la classpath.
