El informe mostrará expresiones que serán de tipo Int, lo que causará errores de compilación en Kotlin 1.9 y versiones posteriores.

Ejemplo:


  fun takeByte(x: Byte) {}
  fun foo() {
      takeByte(1 + 1) // Se resolverá como Int en la versión 1.9
  }

Después de aplicar la corrección rápida:


  fun takeByte(x: Byte) {}
  fun foo() {
      takeByte((1 + 1).toByte()) // Se resolverá como Int en la versión 1.9
  }

La verificación está disponible para el nivel de lenguaje Kotlin desde la versión 1.7.
