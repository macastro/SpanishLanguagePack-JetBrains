Reporta instancias de Logger inicializadas con un literal de clase distinto de la clase en la que reside Logger.

Esto puede ocurrir al copiar y pegar desde otra clase. Puede dar lugar a eventos de registro en una categoría inesperada y a un filtrado incorrecto.

Utilice las opciones de inspección para especificar las clases y métodos de fábrica de registradores reconocidos por esta inspección.

Ejemplo:


class AnotherService
class MyService {
    private val logger = LoggerFactory.getLogger(AnotherService::class.java)
}

Tras aplicar la corrección rápida:


class MyService {
    private val logger = LoggerFactory.getLogger(MyService::class.java)
}
