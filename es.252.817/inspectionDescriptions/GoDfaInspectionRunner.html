Realiza informes sobre uniones innecesarias de cadenas.

Las uniones innecesarias de cadenas aparecen cuando un programa contiene una secuencia de uniones de cadenas que se puede convertir en una sola unión de cadena. Este informe detecta uniones innecesarias de cadenas y ofrece una solución rápida para reemplazarlas con una sola unión.

Para abrir las opciones relacionadas con esta inspección directamente desde aquí, consulte: settings://inspections.java/ConstantExpressionCondition

Detecta las posibles referencias nulas.

Identifica las posibles referencias nulas y proporciona sugerencias de corrección rápida para agregar verificaciones de referencias nulas donde sea necesario. Esto ayuda a mejorar la confiabilidad del código y a evitar errores de tiempo de ejecución.

Para abrir las opciones relacionadas con esta inspección directamente desde aquí, consulte: settings://inspections.java/DereferenceOfNullable

Resalta la asignación potencialmente innecesaria de una variable con un valor de retorno.

Identifica las asignaciones de variables que tienen como valor asignado el valor de retorno de un método. En algunos casos, estas asignaciones pueden ser innecesarias y potencialmente pueden llevar a errores.

Resalta el uso de bucles for con un rango vacío.

Identifica los bucles for que se declaran con un rango vacío y proporciona sugerencias para eliminarlos o reemplazarlos con una construcción diferente según el caso de uso. Los bucles con rangos vacíos pueden no tener el comportamiento esperado, por lo que esta inspección ayuda a mejorar la exactitud y el rendimiento del código.

Marca el uso de cláusulas 'else' redundantes.

Detecta las cláusulas 'else' que siguen a una cláusula 'if' que siempre se evalúa como 'true' o 'false', lo que hace que la cláusula 'else' sea redundante. Esta inspección ayuda a eliminar el código innecesario y mejora la legibilidad.

Resalta las condiciones que siempre son falsas o verdaderas.

Identifica las condiciones que se evalúan de forma constante como 'verdadero' o 'falso' en el tiempo de compilación. Estas condiciones pueden indicar errores lógicos o código innecesario, y esta inspección ayuda a mejorar la exactitud y claridad del código.

Resalta los operadores de comparación invertidos.

Detecta los operadores de comparación invertidos, como '>' usado en lugar de '<', lo que puede indicar un error lógico. Esta inspección ayuda a identificar y corregir problemas potenciales, mejorando la exactitud del código.

Resalta las condiciones complejas.

Identifica las condiciones que tienen una complejidad ciclomática alta, lo que indica que pueden ser difíciles de entender y mantener. Esta inspección alienta la refactorización de condiciones complejas en formas más simples y fáciles de leer, mejorando la legibilidad y el mantenimiento del código.

Resalta los bloques try-catch vacíos.

Detecta los bloques try-catch vacíos que no contienen ningún código para manejar las excepciones. Estos bloques pueden ser innecesarios y pueden indicar un manejo incompleto de excepciones. Esta inspección ayuda a identificar y eliminar bloques vacíos, mejorando la exactitud y el manejo de excepciones del código.

Resalta los métodos que se pueden hacer privados.

Identifica los métodos que se declaran como protegidos o públicos pero que solo son llamados desde dentro de la misma clase. Estos métodos son candidatos para hacerse privados, lo que mejora la encapsulación y reduce el acoplamiento.

Resalta los campos que se pueden hacer privados.

Identifica los campos que se declaran como protegidos o públicos pero que solo se acceden desde dentro de la misma clase. Estos campos son candidatos para hacerse privados, lo que mejora la encapsulación y reduce el acoplamiento.

Resalta los campos que se pueden hacer finales.

Identifica los campos que nunca se reasignarn después de la inicialización. Estos campos son candidatos para hacerse finales, lo que mejora la inmutabilidad y la seguridad del tipo.

Resalta los métodos que se pueden hacer finales.

Identifica los métodos que nunca se sobrescriben en subclases. Estos métodos son candidatos para hacerse finales, lo que mejora el rendimiento y la corrección.

Resalta los bloques switch redundantes.

Identifica los bloques switch que tienen un único caso predeterminado y ningún otro caso. Estos bloques pueden reemplazarse con una instrucción if-else para mejorar la legibilidad y el rendimiento.

Resalta las condiciones redundantes.

Identifica las condiciones que se evalúan a un valor constante y luego se usa ese valor para una verificación adicional. Esta inspección ayuda a simplificar las condiciones y mejorar la legibilidad al eliminar las comprobaciones redundantes.

Resalta las llamadas a `toString` innecesarias.

Detecta los casos en los que se llama al método `toString` en un objeto y el resultado no se utiliza. Estas llamadas se pueden eliminar para mejorar el rendimiento y la legibilidad.

Resalta las dependencias circulares.

Identifica las dependencias circulares entre las clases. Estas dependencias pueden provocar problemas de carga de clases y dificultar el mantenimiento del código. Esta inspección ayuda a detectar y eliminar dependencias circulares, mejorando la estabilidad y la mantenibilidad del proyecto.

Resalta las excepciones que nunca se lanzan.

Identifica las clases de excepción que se declaran pero nunca se lanzan. Estas clases pueden ser innecesarias y pueden indicar una implementación incompleta de la gestión de excepciones. Esta inspección ayuda a identificar y eliminar excepciones no lanzadas, mejorando la exactitud y la corrección del código.

Resalta las variables y parámetros no utilizados.

Identifica las variables y parámetros que se declaran pero nunca se utilizan. Estas declaraciones no utilizadas pueden ser indicativas de código innecesario o errores en la lógica del programa. Esta inspección ayuda a detectar y eliminar declaraciones no utilizadas, mejorando la concisión y el mantenimiento del código.
