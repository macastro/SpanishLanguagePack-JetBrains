Reporta supresiones que usan nombres antiguos de diagnóstico, como @Suppress("HEADER_WITHOUT_IMPLEMENTATION").

Algunos diagnósticos de Kotlin 1.2 y versiones anteriores están ahora obsoletos, lo que hace que estas supresiones sean redundantes.

Ejemplo:


@Suppress("HEADER_DECLARATION_WITH_BODY")
expect fun connection() {
  // ...
}

Después de aplicar la corrección rápida:


@Suppress("EXPECTED_DECLARATION_WITH_BODY")
expect fun connection() {
  // ...
}
