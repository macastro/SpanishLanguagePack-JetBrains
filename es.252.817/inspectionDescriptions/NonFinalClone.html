Informa de métodos clone() sin el modificador final.

Dado que clone() se puede usar para instanciar objetos sin usar un constructor, permitir que el método clone() se reemplace puede ocasionar la corrupción de objetos e incluso en la explotación de seguridad. Esto se puede prevenir haciendo que el método clone() o la clase que lo encierra sea en sí misma final.

Ejemplo:


  class Main implements Cloneable {
    @Override
    protected Object clone() throws CloneNotSupportedException {
      return super.clone();
    }
  }
