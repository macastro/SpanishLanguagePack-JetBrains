Reporta campos cuyos nombres son demasiado cortos, demasiado largos o no siguen el patrón de expresión regular especificado.

Ejemplo: Si la inspección está habilitada para constantes y la longitud mínima especificada para nombres de campos es 5 (valor predeterminado), se generará una advertencia para la siguiente constante porque su nombre tiene una longitud de 3, que es menor que 5: public static final int MAX = 42;.

La corrección rápida para renombrar dichos campos solo está disponible en el editor.

Configuración de la inspección:

Utilice la lista en la sección Opciones para especificar qué campos deben ser inspeccionados. Desmarque las casillas correspondientes a los campos que desea excluir de la inspección.

Para cada tipo de campo, utilice los campos de entrada proporcionados para especificar la longitud mínima, máxima y la expresión regular para los nombres de campo. Especifique 0 en los campos de longitud para omitir la verificación correspondiente.

Las expresiones regulares deben especificarse en el formato estándar de java.util.regex.
