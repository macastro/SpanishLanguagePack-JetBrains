Informa sobre excepciones que se declaran en la firma del método pero que nunca son lanzadas por el método mismo o sus implementaciones y sobreescrituras.

Esta inspección ignora los métodos relacionados con la serialización, como los métodos readObject() y writeObject().

Ejemplo:


  void method() throws InterruptedException {
    System.out.println();
  }

La corrección rápida elimina las excepciones innecesarias de la declaración y normaliza las sentencias try-catch redundantes:


  void method() {
    System.out.println();
  }

Nota: Por razones de rendimiento, algunas excepciones pueden no ser reportadas durante el resaltado en el editor. Para ver todos los resultados, ejecute la inspección seleccionando Código | Inspeccionar código o Código | Analizar código | Ejecutar inspección por nombre desde el menú principal.

Use la opción Ignorar excepciones lanzadas por métodos de punto de entrada para no reportar excepciones lanzadas por métodos como main(). Los métodos de punto de entrada se pueden configurar en la configuración de la inspección Java | Declaraciones redundantes | Declaraciones no utilizadas.
