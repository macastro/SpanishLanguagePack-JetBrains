Informa sobre las comillas simples redundantes en las referencias.

Algunas de las palabras clave de Kotlin son identificadores válidos en Java, por ejemplo: in, object, is. Si una biblioteca de Java utiliza una palabra clave de Kotlin para un método, puedes seguir llamando al método escapándolo con la comilla simple (`), por ejemplo, foo.`is`(bar). A veces, este escape es redundante y puede omitirse de forma segura. La inspección descubre e informa de estos casos y está emparejada con la solución rápida "Eliminar comillas simples redundantes", que te permite modificar el código resaltado.

Ejemplos:


  fun `is`(x: String) {}
  fun foo() {
      `is`("bar") // 'is' es una palabra clave, se requieren comillas simples
  }

  fun `test that smth works as designed`() {} // OK, identificador complejo para mejorar la legibilidad

  val `a` = 1  // no es necesario usar comillas simples
