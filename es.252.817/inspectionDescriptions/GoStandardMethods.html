Informa sobre los métodos con determinados nombres en los siguientes casos:
  • el nombre del método coincide con el nombre de varios métodos de interfaz conocidos de la biblioteca estándar
  • la firma no coincide con la firma del método de interfaz correspondiente

Dichos métodos podrían indicar que el tipo de receptor está diseñado para cumplir una interfaz de la biblioteca estándar, pero falla al hacerlo debido a un error en la firma del método.

Ejemplo:

type MyReader []byte

func (r MyReader) ReadByte(data []byte) (byte, error) {
}

El uso es sospechoso porque parece un intento de implementar io.ByteReader pero la firma es incorrecta. Una versión más correcta sería la siguiente:

type MyReader []byte

func (r MyReader) ReadByte() (byte, error) {
}
