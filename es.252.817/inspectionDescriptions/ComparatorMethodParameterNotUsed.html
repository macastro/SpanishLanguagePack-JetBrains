Informa sobre problemas en las implementaciones de Comparator.compare() y Comparable.compareTo().

Se informará de los siguientes casos:

  • Parámetros no utilizados. Probablemente esto sea un error tipográfico, una comparación de un parámetro consigo mismo, o una implementación incorrecta del método.
  • El método evidentemente no devuelve 0 para elementos iguales. Este método de comparación viola el contrato y puede producir resultados impredecibles cuando se encuentran elementos iguales. En particular, algunos datos pueden causar excepciones y fallos en la ordenación.
  • El método de comparación nunca devuelve valores positivos o negativos. Para cumplir con el contrato, si el método de comparación devuelve un valor positivo, también debería devolver un valor negativo cuando los argumentos se proporcionan en orden inverso.
  • El método de comparación devuelve Integer.MIN_VALUE. Aunque está permitido por el contrato, puede ser propenso a errores, ya que algunos puntos de llamada podrían usar incorrectamente el operador de negación unaria para invertir el valor devuelto por el método de comparación. El valor negativo de Integer.MIN_VALUE es Integer.MIN_VALUE.

Ejemplo:


  Comparator<String> lambda =
    (a, b) -> a.length() > b.length()
              ? 0
              : Math.random() > 0.5 ? -1 : 1;
