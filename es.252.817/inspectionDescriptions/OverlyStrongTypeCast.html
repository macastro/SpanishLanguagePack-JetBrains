Informa sobre conversiones de tipo demasiado específicas. Por ejemplo, cuando un objeto se convierte a ArrayList cuando una conversión a List también funcionaría.

Nota: Similar a la comprobación de conversiones de tipo redundantes, si intencionalmente está utilizando una conversión más específica para generar un ClassCastException, aplicar esta corrección podría alterar la semántica del programa.

Ejemplo:


  interface Super {
    void doSmth();
  }
  interface Sub extends Super { }
  void use(Object obj) {
    // Advertencia: se puede usar ((Super)obj).doSmth()
    ((Sub)obj).doSmth();
  }

Utilice la casilla de verificación a continuación para ignorar las conversiones cuando haya una comprobación de instanceof correspondiente en el código.
