Reporta bucles for que iteran sobre colecciones o arrays y pueden ser reemplazados automáticamente por bucles for mejorados (sintaxis de iteración foreach).

Ejemplo:


  for (Iterator<String> iterator = list.iterator(); iterator.hasNext(); ) {
    String item = iterator.next();
    System.out.println(item);
  }

Después de aplicar la corrección rápida:


  for (String item : list) {
    System.out.println(item);
  }

La opción Reportar bucles 'java.util.List' que usan índices encuentra bucles que involucran llamadas a list.get(index). Estos bucles generalmente pueden ser reemplazados por bucles for mejorados, a menos que modifiquen la lista subyacente durante el proceso, por ejemplo, mediante llamadas a list.remove(index). En este último caso, el bucle for mejorado podría lanzar una ConcurrentModificationException. En algunos casos, los bucles con list.get(index) también pueden ser más rápidos.

La opción No reportar iteraciones sobre colecciones sin tipo ignora las colecciones sin parámetros de tipo. Esto evita la creación de variables de bucle for mejorado de tipo java.lang.Object y la inserción de conversiones donde se usa la variable del bucle.
