Informa sobre expresiones y condiciones que siempre producen el mismo resultado, como true, false, null o cero. Tales expresiones pueden ser reemplazadas por sus valores constantes correspondientes. Sin embargo, generalmente indican la presencia de un bug en el código.

Ejemplo:

  // Siempre es true
  // Causa raíz: se usa || en lugar de &&
  if (x > 0 || x < 10) {}
  System.out.println(str.trim());
  // Siempre es false
  // Causa raíz: la variable se desreferencia antes de la comprobación de null
  if (str == null) {}

El comportamiento de la inspección puede ser controlado por varias anotaciones, como las anotaciones de nulabilidad, la anotación @Contract, la anotación @Range, etc.

Configuración de la inspección:

  • Use la opción No informar sobre aserciones que se demuestre estáticamente que siempre son verdaderas para evitar informes sobre aserciones que se demuestre estáticamente que siempre son verdaderas. Esto incluye también condiciones como if (alwaysFalseCondition) throw new IllegalArgumentException();.
  • Use la opción Ignorar declaraciones assert para controlar cómo la inspección maneja las declaraciones assert. Por defecto, esta opción está desactivada, lo que significa que las aserciones se consideran ejecutadas (modo -ea). Si se activa esta opción, las aserciones se ignoran por completo (modo -da).
  • Use la opción Advertir cuando las constantes se almacenan en variables para mostrar advertencias cuando se utilizan variables cuyo valor se sabe que es constante.

Antes de IntelliJ IDEA 2022.3, esta inspección era parte de la inspección "Condiciones constantes y excepciones". Ahora está dividida en dos inspecciones: "Valores constantes" y "Problemas de nulabilidad y flujo de datos".
