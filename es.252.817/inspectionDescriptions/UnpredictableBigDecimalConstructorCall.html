Informa sobre llamadas al constructor de BigDecimal que acepta valores double. Estos constructores generan un BigDecimal que es exactamente igual al valor double proporcionado. Sin embargo, debido a que double está codificado en formato binario de punto flotante IEEE 754 de 64 bits de doble precisión, el valor exacto puede ser inesperado.

Por ejemplo, new BigDecimal(0.1) genera un objeto BigDecimal. Su valor es 0.1000000000000000055511151231257827021181583404541015625, que es el número más cercano a 0.1 que se puede representar como double. Para obtener un BigDecimal donde el valor almacenado sea igual al valor escrito en el código fuente, use new BigDecimal("0.1") o BigDecimal.valueOf(0.1).

Ejemplo:


  class Constructor {
    void foo() {
      new BigDecimal(0.1);
    }
  }

Después de aplicar la corrección rápida:


  class Constructor {
    void foo() {
      new BigDecimal("0.1");
    }
  }
