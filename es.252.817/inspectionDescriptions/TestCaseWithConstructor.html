Informa sobre los casos de prueba con lógica de inicialización en sus constructores. Si falla un constructor, no se llamará al método @After anotado o a tearDown(). Esto puede dejar el entorno de prueba parcialmente inicializado, lo que puede afectar negativamente a otras pruebas. En su lugar, la inicialización de los casos de prueba debe hacerse en un método setUp() o @Before anotado.

Mal ejemplo:


  public class ImportantTest {
    private File file;

    public ImportantTest() throws IOException {
      file = File.createTempFile("xyz", ".tmp");
    }

    // ... tests go here
  }
