Se informa de los siguientes errores que se producen en las consultas EJB QL:
  • Tipos de expresiones no coincidentes
  • Parámetros incorrectos
  • Condiciones vacías o condiciones constantes
  • Símbolos no resueltos

Ejemplo de consulta:


SELECT OBJECT(e) FROM Employee e WHERE SQRT(ABS(e.firstName - ?1)) > 50000; // Error: tipos no coincidentes, se necesita un tipo numérico

SELECT e.firstName FROM Employee e WHERE SUBSTRING(e.firstName, ?1, ?2) = ?4; // Error: solo hay 3 parámetros de consulta reales
