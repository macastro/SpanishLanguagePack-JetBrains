Informa sobre recursos de socket que no se cierran de forma segura. Los recursos de socket informados por esta inspección incluyen java.net.Socket, java.net.DatagramSocket, y java.net.ServerSocket.

De manera predeterminada, la inspección asume que los recursos pueden ser cerrados por cualquier método con 'close' o 'cleanup' en su nombre.

Ejemplo:


  byte[] getMessage(ServerSocket socket) throws IOException {
    Socket client = socket.accept(); //socket no está cerrado
    return client.getInputStream().readAllBytes();
  }

Utiliza las siguientes opciones para configurar la inspección:

  • Si se permite que se abra un socket dentro de un bloque try. Este estilo es menos deseable porque es más prolijo que abrir un recurso frente a un bloque try.
  • Si el recurso puede ser cerrado por cualquier llamada a método con el recurso pasado como argumento.
