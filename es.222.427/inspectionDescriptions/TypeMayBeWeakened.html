Reporta tipos de retorno de variables y métodos que se pueden cambiar a un tipo más abstracto (más débil). Esto permite hacer que el código sea más abstracto, por lo tanto, más reutilizable.

Ejemplo:


  // Type of parameter can be weakened to java.util.List
  void processList(ArrayList<String> list) {
    if (list.isEmpty()) return;
    System.out.println("Processing");
    for (String s : list) {
      System.out.println("String: " + s);
    }
  }

Habilita el checkbox Sólo debilitar a una interfaz debajo para sólo reportar un problema cuando el tipo puede ser debilitado a un tipo de interfaz.

Habilita el checkbox No sugerir debilitar variables declaradas como 'var' debajo para prevenir reportar variables locales declaradas usando la palabra clave 'var' (Java 10+)

Las Clases de parada pretenden prevenir debilitar a clases menores que las de parada, aún cuando sea posible. En algunos casos, esto puede mejorar la legibilidad.
