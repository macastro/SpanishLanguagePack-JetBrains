Informa de métodos con la misma firma que un método inaccesible de una superclase, por ejemplo, un método privado o un método package-private de una superclase en otro paquete.

Estos nombres de método pueden ser confusos porque el método en la subclase puede parecer una invalidación cuando de hecho oculta el método inaccesible de la superclase. Además, si la visibilidad del método en la superclase cambia más adelante, puede cambiar silenciosamente la semántica de la subclase o causar un error de compilación.

Se sugiere una corrección rápida para cambiar el nombre del método.

Ejemplo:


  public class Super {
    private void test() {
    }
  }

  public class Sub extends Super {
    void test() { // convertir 'Super.test()' en public provoca un error de compilación
                  // convertir 'Super.test()' en package-private convierte 'Sub.test()' en una invalidación
    }
  }
