Reporta bloques synchronized que se bloquean en una instancia de java.util.concurrent.locks.Lock. Dicha sincronización es casi ciertamente no intencionada, y en cambio deberían utilizarse versiones adecuadas de .lock() y .unlock().

Ejemplo:


  final ReentrantLock lock = new ReentrantLock();

  public void foo() {
    synchronized (lock) {}
  }
