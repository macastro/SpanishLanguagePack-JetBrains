Informa la anotación @GuardedBy donde el campo protegido no es final.

El hecho de proteger un campo no final puede provocar condiciones de carrera inesperadas, ya que el valor del campo se bloquea (y puede cambiar), en lugar del campo en sí.

Ejemplo:


    private ReadWriteLock lock = new ReentrantReadWriteLock(); // campo no final protegido
    private Object state;

    @GuardedBy("lock")
    public void bar() {
        state = new Object();
    }

Las anotaciones @GuardedBy admitidas son:

  • net.jcip.annotations.GuardedBy
  • javax.annotation.concurrent.GuardedBy
  • org.apache.http.annotation.GuardedBy
  • com.android.annotations.concurrency.GuardedBy
  • androidx.annotation.GuardedBy
  • com.google.errorprone.annotations.concurrent.GuardedBy
