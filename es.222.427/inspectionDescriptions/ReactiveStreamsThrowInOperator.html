Informa las expresiones throw en el código del operador Reactor/RxJava.

Lanzar excepciones desde un operador Reactor/RxJava indica un posible problema, porque puedes retornar un error "similar a Reactive": Mono.error() o Flowable.error() de flatMap(), o llamar a sink.error() desde el operador reactor handle().

Además, los métodos factory de Reactor permiten retornar excepciones marcadas sin ningún error, mientras que lanzar dichas excepciones sin la clase Exceptions lleva a un error de compilación.

Ejemplo:


Flux.just(1, 2, 3).flatMap(i -> {
            throw new RuntimeException();
        })

Después de aplicar el arreglo rápido:


Flux.just(1, 2, 3).flatMap(i -> {
            return Flux.error(new RuntimeException());
        })
