Reporta métodos con nombres tipo accesor que tienen un tipo de resultado Unit.

Se espera que los métodos que siguen el contrato de nomenclatura de JavaBean para accesores no tengan efectos secundarios. Sin embargo, los métodos con un tipo de resultado de Unit solo se ejecutan para sus efectos secundarios.

Esto indica que el método debería resultar en un tipo distinto de Unit o renombrarse para reflejar mejor el efecto secundario.

Ejemplo:


  trait Test {
    def getTarget(): Unit
  }

Esta inspección no reportará los métodos que anulan otros miembros.
