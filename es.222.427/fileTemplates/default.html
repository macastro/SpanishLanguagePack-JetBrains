En las plantillas de archivo, puedes utilizar texto, código, comentarios y variables predefinidas. A continuación se ofrece una lista de variables predefinidas. Cuando utilizas estas variables en las plantillas, más adelante se expanden automáticamente en los valores correspondientes en el editor.

También es posible especificar variables personalizadas. Las variables personalizadas utilizan el formato siguiente: ${NOMBRE_VARIABLE}, donde NOMBRE_VARIABLE es el nombre de tu variable (por ejemplo, ${NOMBRE_FUNCION_PERSONALIZADA}). Antes de que el IDE cree un archivo nuevo con variables personalizadas, verás un cuadro de diálogo donde puedes definir los valores de las variables personalizadas en la plantilla.

Mediante la directiva #parse puedes incluir plantillas desde la pestaña Incluye. Para incluir una plantilla, especifica el nombre completo de la plantilla como parámetro entre comillas (por ejemplo, #parse("File Header")).
Lista de variables predefinidas
${DATE} Fecha actual del sistema
${DAY} Día actual del mes
${DAY_NAME_SHORT} Primeras tres letras del nombre del día actual (Lun, Mar, etc.)
${DAY_NAME_FULL} Nombre completo del día actual (Lunes, Martes, etc.)
${DIR_PATH} Ruta al directorio del nuevo archivo (relativa a la raíz del proyecto)
${DS} Signo de dólar ($). Esta variable se utiliza para escapar el carácter de dólar, para que no se trate como un prefijo de una variable de plantilla.
${FILE_NAME} Nombre del nuevo archivo
${HOUR} Hora actual
${MINUTE} Minuto actual
${SECOND} Segundo actual
${MONTH} Mes actual
${MONTH_NAME_SHORT} Las primeras 3 letras del nombre del mes actual (Ene, Feb, etc.)
${MONTH_NAME_FULL} Nombre completo del mes actual (Enero, Febrero, etc.)
${NAME} Nombre de la nueva entidad (archivo, tipo, interfaz, etc.)
${ORGANIZATION_NAME} Nombre de tu organización especificado en la configuración del proyecto
${PRODUCT_NAME} Nombre del IDE
${PROJECT_NAME} Nombre del proyecto actual
${TIME} Hora actual del sistema
