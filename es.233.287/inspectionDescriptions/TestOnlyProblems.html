Inspecciona métodos y clases anotados con @TestOnly y @VisibleForTesting que se utilizan en código de producción. También informa del uso de la aplicación de @TestOnly @VisibleForTesting al mismo elemento.

Los problemas no se informan si se hace referencia a dicho método o clase desde:

  • Código bajo la carpeta Fuentes de prueba.
  • Una clase de prueba (JUnit/TestNG)
  • Otro método anotado con @TestOnly

Ejemplo (en código de producción):


  @TestOnly
  fun foo() { ... }

  fun main () {
    foo()
  }
