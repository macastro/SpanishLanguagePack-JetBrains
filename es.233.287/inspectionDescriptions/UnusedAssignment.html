Informa los valores de la asignación que no se utilizan después de la asignación. Si el valor de asignación no se utiliza, es mejor eliminar la asignación para acortar el código y evitar asignaciones redundantes.

Se informan los siguientes casos:

  • variables que nunca se leen después de la asignación
  • variables que siempre se sobrescriben con un nuevo valor antes de leerse
  • inicializadores de variables que son redundantes (por una de las dos razones anteriores)

Configurar la inspección:

Utiliza la opción Informar inicializadores redundantes para informar inicializadores redundantes: 


  int getI() {
    int i = 0; // inicialización redundante
    i = 2;
    return i;
  }

Utiliza la opción Informar expresiones de prefijo que pueden reemplazarse con expresiones binarias para informar casos donde una expresión ++i puede reemplazarse con i + 1: 


  int preInc(int value) {
    int res = value;
    return ++res;
  }

Utiliza la opción Informar expresiones de sufijo donde el valor cambiado no se utiliza para informar casos i++ donde el valor de i no se utiliza más adelante: 


  int postInc(int value) {
    int res = value;
    return res++;
  }

Utiliza la opción Informar variables de patrón cuyos valores nunca se utilizan para informar casos donde el valor de una variable de patrón se sobrescribe antes de leerse: 


  if (object instanceof String s) {
      s = "hola";
      System.out.println(s);
  }

Utiliza la opción Informar parámetros de iteración cuyos valores nunca se utilizan para informar casos donde el valor de un parámetro de iteración de declaraciones for mejoradas se sobrescribe antes de leerse: 


  for (String arg : args) {
      arg = "prueba";
      System.out.println(arg);
  }
