Informa sobre métodos que no son static, private, final o abstract y cuyo cuerpo no está vacío.

Escribir código en un estilo que evite dichos métodos protege los contratos de las clases frente a infracciones por parte de sus subclases. La ventaja de este estilo es que las subclases no pueden corromper el estado de la superclase olvidando llamar al supermétodo. El coste es que las subclases están limitadas en su flexibilidad, en particular no pueden evitar la ejecución de código en la superclase. Utiliza la corrección rápida para añadir los modificadores que faltan.

Ejemplo:


  class Foo {
    public boolean equals(Object o) { return true; }
  }

Después de aplicar la corrección rápida:


  class Foo {
    public final boolean equals(Object o) { return true; }
  }
Esta inspección está pensada para código que va a usarse en entornos seguros y probablemente no sea adecuada para entornos menos restrictivos.
