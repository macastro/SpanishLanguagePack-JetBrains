Se informa cualquier cadena de sentencias if-else cuyas condiciones sean todas expresiones instanceof o expresiones de igualdad de clases (por ejemplo, comparación con String.class). Estas construcciones usualmente indican un error en el diseño orientado a objetos que dicta que este despacho o direccionamiento basado en el tipo debe realizarse a través de llamadas polimórficas a métodos más que cadenas explícitas de pruebas de tipo.

Ejemplo:


  double getArea(Shape shape) {
    // Advertencia: error de abstracción.
    // Sería mejor declarar un método
    // abstracto getArea() en la interfaz de
    // de forma y implementarlo en cada clase
    // heredada.
    if (forma instanceof Point) {
      return 0;
    }
    if (forma instanceof Circle) {
      return Math.PI *
        Math.pow(((Circle) forma).radius(), 2);
    }
    if (forma instanceof Rectangle) {
      return ((Rectangle) forma).width() *
        ((Rectangle) forma).height();
    }
    throw new IllegalArgumentException();
  }

Usa la casilla de verificación a continuación para ignorar expresiones instanceof en clases de biblioteca.
