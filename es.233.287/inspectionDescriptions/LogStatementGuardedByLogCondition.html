Informa sobre llamadas a registro con argumentos no constantes no rodeados por una condición de protección. La evaluación de los argumentos de una llamada al registro puede resultar costosa. Rodear una llamada a registro con una cláusula de protección evita ese coste cuando el registro está deshabilitado para el nivel utilizado por la instrucción de registro. Esto es especialmente útil para el nivel menos grave (traza, depuración, excepcional) de llamadas a registro, porque a menudo son deshabilitados en un entorno de producción.

Ejemplo:


  public class Principal {
    void malo(Object object) {
      if (verdadero) {
        LOG.debug("registro registro registro " + cálculo_costoso(object));
      }
      LOG.debug("algo más de registro " + cálculo_costoso(1));
    }

    void bueno(Object) {
      if (LOG.isDebug()) {
        LOG.debug("valor: " + cálculo_costoso(object));
      }
    }
  }

Configurar la inspección:

  • Utiliza el campo Nombre de clase de registro para especificar el nombre de la clase de registro utilizado.

  • Utiliza la tabla para especificar los métodos de registro sobre los que debe advertir esta inspección, con el texto de condición de registro correspondiente.

  • Utiliza la opción Marcar todas las llamadas a registro sin protección para que la inspección marque todas las llamadas a registro sin protección, no sólo aquellas con argumentos no constantes.
