Reporta los bucles for que iteran sobre las colecciones o arreglos, y que pueden ser reemplazados automáticamente con un bucle for mejorado (sintaxis de iteración foreach).

Ejemplo:


  for (Iterator<String> iterator = list.iterator(); iterator.hasNext(); ) {
    String item = iterator.next();
    System.out.println(item);
  }

Después de aplicar la corrección rápida:


  for (String item : list) {
    System.out.println(item);
  }

Usa la opción Reportar bucles 'java.util.List' indizados para encontrar los bucles que involucran llamadas a list.get(index). Generalmente, estos bucles pueden ser reemplazados con bucles for mejorados, a menos que modifiquen una lista subyacente en el proceso, por ejemplo, llamando a list.remove(index). Si este último es el caso, el bucle for mejorado puede lanzar ConcurrentModificationException. También, en algunos casos, los bucles list.get(index) pueden funcionar un poco más rápido.

Usa la opción No reportar iteraciones sobre colecciones sin tipo para ignorar las colecciones sin parámetros de tipo. Esto evita la creación de variables de bucle for mejoradas del tipo java.lang.Object y la inserción de conversiones explícitas donde se use la variable del bucle.
