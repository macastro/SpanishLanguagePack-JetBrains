Informa sobre métodos con cláusulas de parámetro vacías que anulan métodos de acceso Java.

Se espera que los métodos que siguen el contrato de nombres JavaBean para los accesores no tengan efectos secundarios. Se recomienda utilizar un método sin parámetros siempre que no haya parámetros y el método no tenga efectos secundarios. Esta convención promueve el principio de acceso uniforme, que establece que el código del cliente no debe verse afectado por la decisión de implementar un atributo como campo o método.

El problema es que Java no implementa el principio de acceso uniforme y no permite métodos sin cláusulas de parámetros. Para cerrar esa brecha, Scala te permite anular métodos con cláusulas de parámetros vacías con métodos sin cláusulas de parámetros.

La solución rápida elimina la cláusula de parámetro vacía.

Ejemplo:


  class TestException extends Exception {
    override def getMessage(): String = ???
  }

Después de aplicar la solución rápida:


  class TestException extends Exception {
    override def getMessage: String = ???
  }
