Informa de los parámetros o variables locales declarados innecesariamente como final.

Hay estándares de codificación que desaconsejan utilizar final para declarar variables por motivos de concisión.

Ejemplo:


  class Foo {
    Foo(Object o) {}

    void bar(final Object o) {
      new Foo(o);
    }
  }

Tras aplicar la corrección rápida:


  class Foo {
    Foo(Object o) {}

    void bar(Object o) {
      new Foo(o);
    }
  }

Usa las opciones de inspección para alternar los informes de:

  • variables locales
  • parámetros (incluidos los parámetros de los bloques catch y las sentencias for mejoradas)

También puedes configurar la inspección para que solo informe de los parámetros final de los métodos abstract o de interfaz, que pueden considerarse innecesariamente adicionales ya que tales marcas no afectan a la implementación de estos métodos.
