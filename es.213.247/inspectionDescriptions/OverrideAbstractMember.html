Informa sobre las funciones que anulan a miembros abstractos pero que no cuentan con un modificador de anulación.

Scala no requiere que un desarrollador utilice override en métodos que implementan miembros abstractos. Sin embargo, es una buena idea marcar tales métodos con override para ser notificado por el compilador cuando la relación de anulación se rompe (por ejemplo, por una refactorización posterior).

Ejemplo:


  trait Base {
    def fun(): Int
  }

  class Impl extends Base {
    def fun(): Int = 42
  }

Después de aplicar la corrección rápida:


  trait Base {
    def fun(): Int
  }

  class Impl extends Base {
    override def fun(): Int = 42
  }
