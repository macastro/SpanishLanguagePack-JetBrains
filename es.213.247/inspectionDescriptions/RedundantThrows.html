Reporta excepciones que están declaradas en la signatura del método pero que nunca se lanzan por el propio método o por su implementación y métodos anulados.

Esta inspección ignora métodos relacionados con la serialización, como los métodos readObject() y writeObject().

Ejemplo:


  void method() throws InterruptedException {
    System.out.println();
  }

La corrección rápida eliminará la excepción innecesaria de la declaración y normalizará las declaraciones try-catch redundantes.


  void method() {
    System.out.println();
  }

Nota: Es posible que no se informen algunas excepciones mientras se resaltan en el editor debido a razones de rendimiento. Para ver todos los resultados, ejecuta la inspección seleccionando Código | Inspeccionar código o Código | Analizar código | Ejecutar inspección por nombre en el menú principal.

Por ejemplo, utiliza la opción Ignorar excepciones lanzadas por métodos de punto de entrada para no informar las excepciones lanzadas por métodos como main(). Los métodos de punto de entrada se pueden configurar en la configuración de la inspección Java | Redundancia de declaración | Declaraciones no utilizadas.
