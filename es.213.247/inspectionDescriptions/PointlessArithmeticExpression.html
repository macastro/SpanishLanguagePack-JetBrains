Informa expresiones aritméticas sin sentido. Estas expresiones incluyen sumar o restar cero, multiplicar por cero o uno, y dividir por uno.

Estas expresiones pueden ser el resultado de refactorizaciones automáticas y es poco probable que el desarrollador haya querido hacer esto.

La solución rápida simplifica estas expresiones.

Ejemplo: 


  void f(int a) {
    int x = a - a;
    int y = a + 0;
    int res = x / x;
  }

Después de aplicar la solución rápida:


  void f(int a) {
    int x = 0;
    int y = a;
    int res = 1;
  }

Tenga en cuenta que en casos excepcionales, el reemplazo sugerido podría no ser completamente equivalente al código original para todas las posibles entradas. Por ejemplo, la inspección sugiere reemplazar x / x con 1. Sin embargo, si x es cero, el código original lanza ArithmeticException o da como resultado NaN. Además, si x es NaN, el resultado también es NaN. Es muy poco probable que este comportamiento sea intencional.
