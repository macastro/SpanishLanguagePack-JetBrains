Informa de setters de propiedades redundantes.

Un setter se considera redundante cuando se da alguna de las siguientes condiciones:

  1. El setter no tiene cuerpo. La visibilidad del accesor no se modifica, la declaración no es external y no tiene anotaciones. 
    
  var myPropWithRedundantSetter: Int = 0
      set // redundante

  var myPropA: Int = 0
      private set // OK - la visibilidad de la propiedad se cambia a privada

  var myPropB: Int = 0
      external set // OK - no implementado en Kotlin (externo)

  var myPropC: Int = 0
      @Inject set // OK - el accesor está anotado
  2. El cuerpo del setter es un bloque con una sola sentencia que asigna el parámetro al campo de respaldo. 
    
  var prop: Int = 0
      set(value) { // redundante
          field = value
      }
