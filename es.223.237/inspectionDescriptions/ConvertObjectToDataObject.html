Informa los objetos que pueden convertirse a objetos de datos

Los objetos de datos autogeneran toString, equals y hashCode

La inspección sugiere convertir objetos a objetos de datos en 2 casos:

  • Cuando toString personalizado devuelve el nombre de la clase
  • Cuando el objeto hereda la clase/interfaz sealed

Ejemplo:


    object Foo {
        override fun toString(): String = "Foo"
    }

Después de aplicar la solución rápida:


    data object Foo

Esta inspección solo informa si el nivel de lenguaje Kotlin del proyecto o módulo es 1.9 o superior
