Reporta métodos con una cláusula de parámetro vacía que sobrescribe métodos sin cláusulas de parámetro.

La convención recomendada es usar un método sin cláusulas de parámetro siempre que no haya parámetros y el método no tenga efectos colaterales.

Esta convención promueve el principio de acceso uniforme, que establece que el código del cliente no debe verse afectado por una decisión de implementar un atributo como un campo o un método.

De acuerdo con el principio de sustitución de Liskov, siempre que un método sobreescrito no tenga una cláusula de parámetro, el método que lo sobrescribe también debe declararse como un método sin efectos colaterales.

La solución rápida elimina la cláusula de parámetro vacía.

Ejemplo:


  trait Base { def isOk: Boolean }

  class Impl extends Base {
    override def isOk(): Boolean = ???
  }

Después de aplicar la solución rápida:


  trait Base { def isOk: Boolean }

  class Impl extends Base {
    override def isOk: Boolean = ???
  }
