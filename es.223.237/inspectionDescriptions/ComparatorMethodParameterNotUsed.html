Informa de problemas en las implementaciones de Comparator.compare() y Comparable.compareTo().

Se informa de los siguientes casos:

  • No se utiliza un parámetro. Lo más probable es que sea un error tipográfico y el otro parámetro se compare consigo mismo, o que el método no se haya implementado correctamente.
  • Es evidente que el método no devuelve 0 para los mismos elementos. Este método de comparación viola el contrato y puede producir resultados impredecibles cuando se encuentran elementos iguales. En particular, la ordenación puede fallar con una excepción en algunos datos.
  • El método de comparación nunca devuelve un valor positivo o negativo. Para cumplir el contrato, si el método de comparación devuelve valores positivos, también debe devolver valores negativos si los argumentos se suministran en orden inverso.
  • El método de comparación devuelve Integer.MIN_VALUE. Aunque el contrato lo permite, puede ser propenso a errores, ya que algunos sitios de llamada pueden intentar invertir incorrectamente el valor devuelto del método de comparación utilizando el operador unario de menos. El valor negado de Integer.MIN_VALUE es Integer.MIN_VALUE.

Ejemplo:


  Comparator<String> lambda =
    (a, b) -> a.length() > b.length()
              ? 0
              : Math.random() > 0.5 ? -1 : 1;
