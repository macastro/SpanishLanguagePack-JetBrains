**Versión en español** Informes de casos para los lenguajes Java y Kotlin cuando se pasa una cadena no segura a un método como una consulta SQL. Puede ser causa de inyecciones SQL. La lista de métodos se toma de Configuración - Inyecciones de lenguaje para SQL, JPA QL, Hibernate QL y PostgreSQL

Un objeto seguro es:

  • un literal de cadena, instancia de interfaz u objeto de enumeración, int y su envoltura, booleano y su envoltura, objeto de clase
  • un resultado de una llamada de un método, cuyo receptor y argumentos son seguros
  • un campo privado en el mismo archivo, que solo se asigna con un literal de cadena y tiene un inicializador seguro
  • un campo final en el mismo archivo, que tiene un inicializador seguro
  • una variable local que se asigna a partir de objetos seguros
    • Este campo, variable local o parámetro no debe pasarse como argumento a los métodos ni usarse como calificador o debe ser un objeto primitivo, su envoltura o inmutable.
Los campos finales estáticos se consideran seguros.

El análisis solo se realiza dentro de un archivo. Ejemplo: 


    public void save(String sql) {
        JdbcTemplate jdbcTemplate = new JdbcTemplate();
        jdbcTemplate.queryForList(sql);
    }
