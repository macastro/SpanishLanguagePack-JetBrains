Reporta los usos de una API marcada como inestable con una de las anotaciones. Esta API puede ser cambiada o eliminada en versiones futuras, rompiendo así el código que la utiliza.

De forma predeterminada, la inspección ignora los usos de APIs inestables si sus declaraciones están ubicadas en fuentes del mismo proyecto. En estos casos, será posible actualizar los usos cuando cambies las APIs. Sin embargo, podría resultar inconveniente si el proyecto es grande, de forma que se puede desactivar la opción Ignorar API declarada en este proyecto para reportar los usos de APIs inestables declaradas tanto en las fuentes del proyecto como en las bibliotecas.
