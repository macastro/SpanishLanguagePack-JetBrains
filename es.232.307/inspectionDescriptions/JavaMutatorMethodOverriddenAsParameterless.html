Informa los métodos sin cláusulas de parámetros que anulan los métodos de Java con nombres similares a mutadores.

Scala permite anular los métodos de Java sin parámetros sin cláusulas de parámetros. Esto se debe principalmente a que permite que el código de Scala anule los métodos accesores de Java de la misma manera que los métodos accesores de Scala. Es decir, sin cláusula de parámetros y de ese modo adherirse al principio de acceso uniforme, que indica que el código del cliente no debería verse afectado por la decisión de implementar un atributo como un campo o método.

Sin embargo, para los métodos con efectos secundarios, es una convención tener una cláusula de parámetro vacía. Esto estaría de acuerdo con el principio de sustitución de Liskov, mediante el cual, cuando un método anulado tiene efectos secundarios, el método que anula también debe declararse como un método con efectos secundarios.

La solución rápida agrega una cláusula de parámetro vacía.

Ejemplo:


  import java.util

  class Test extends util.ArrayList[String] {
    override def clear: Unit = ???
  }

Después de aplicar la solución rápida:


  import java.util

  class Test extends util.ArrayList[String] {
    override def clear(): Unit = ???
  }
