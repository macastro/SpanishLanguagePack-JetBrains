Informa sobre la discrepancia entre el nivel de un servicio ligero y sus constructores.

Se informan los siguientes problemas:

  • Una clase de servicio ligero que tiene un constructor con dos parámetros de tipo com.intellij.openapi.project.Project y kotlinx.coroutines.CoroutineScope, o un constructor con un parámetro de tipo Project que no está anotado como servicio de nivel de proyecto.
  • Un servicio ligero anotado como servicio de nivel de aplicación que no tiene un constructor sin argumentos ni un constructor con un parámetro de tipo CoroutineScope.

Ejemplo (Kotlin):


@Service                              // Se sugiere especificar el parámetro 'Service.Level.PROJECT' en la anotación '@Service'
class MyService(val project: Project) // Se sugiere eliminar el parámetro del constructor 'MyService'

Después de aplicar la corrección rápida que sugiere especificar el parámetro 'Service.Level.PROJECT' en la anotación '@Service':


@Service(Service.Level.PROJECT)
class MyService(val project: Project)

Después de aplicar la corrección rápida que sugiere eliminar el parámetro del constructor 'MyService':


@Service
class MyService()
