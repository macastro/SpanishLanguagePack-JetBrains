Informa de expresiones y condiciones que siempre producen el mismo resultado, como true, false, null o cero. Dichas expresiones podrían reemplazarse por el valor constante correspondiente. Sin embargo, muy a menudo indican un error en el código.

Ejemplos:

  // siempre true
  // causa raíz: se utiliza || en lugar de &&
  if (x > 0 || x < 10) {}

  System.out.println(str.trim());
  // siempre false
  // causa raíz: se desreferenció la variable antes de la comprobación de nulidad
  if (str == null) {}

El comportamiento de la inspección puede controlarse mediante varias anotaciones, como las anotaciones de nulabilidad, la anotación @Contract, la anotación @Range, etc.

Configurar la inspección:

  • Usa la opción No informes de aserciones con una condición que se ha demostrado estáticamente que siempre es verdadera para evitar informar de aserciones que se ha demostrado estáticamente que siempre son verdaderas. Esto también incluye condiciones como if (alwaysFalseCondition) throw new IllegalArgumentException();.
  • Usa la opción Ignorar declaraciones assert para controlar cómo trata la inspección las declaraciones assert. Por defecto, la opción está deshabilitada, lo que significa que se asume que las aserciones se ejecutan (modo -ea). Si la opción está habilitada, las aserciones se ignorarán por completo (modo -da).
  • Usa la opción Advertencia cuando una constante se almacena en una variable para mostrar advertencias cuando se usa una variable cuyo valor se sabe que es una constante.

Antes de IntelliJ IDEA 2022.3, esta inspección formaba parte de la inspección "Condiciones constantes y excepciones". Ahora, se divide en dos inspecciones: "Valores constantes" y "Problemas de flujo de datos y nulidad".
