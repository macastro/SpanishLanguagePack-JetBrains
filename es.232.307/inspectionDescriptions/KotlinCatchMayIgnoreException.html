Informa sobre bloques catch vacíos o que pueden omitir una excepción.

Aunque en ocasiones se utilizan de forma intencionada, los bloques catch vacíos pueden complicar la depuración. Además, omitir un parámetro catch puede ser incorrecto.

La inspección no informará sobre ningún parámetro catch denominado ignore, ignored o _.

Puedes usar la corrección rápida para cambiar el nombre de la excepción a _.

Ejemplo:


  try {
    throwingMethod()
  } catch (ex: IOException) {

  }

Después de aplicar la corrección rápida:


  try {
    throwingMethod()
  } catch (_: IOException) {

  }
Utiliza la opción No avisar cuando el bloque 'catch' contenga un comentario para ignorar los bloques catch con comentarios.
