Archivo Go es una plantilla integrada que se utiliza cada vez que creas un nuevo archivo Go. Para utilizar esta plantilla, ve a Archivo | Nuevo, selecciona Archivo Go en la categoría Go y después haz clic en Archivo vacío.

La plantilla es editable. En la plantilla, puedes utilizar las variables predefinidas que se enumeran a continuación. Estas variables se expandirán posteriormente al valor correspondiente en el editor.

También puedes especificar variables personalizadas. Las variables personalizadas utilizan el siguiente formato: ${VARIABLE_NAME}, donde VARIABLE_NAME es el nombre de la variable (por ejemplo, ${MY_CUSTOM_FUNCTION_NAME}). Antes de que el IDE cree un nuevo archivo con variables personalizadas, se mostrará un cuadro de diálogo en el que puedes definir los valores de las variables personalizadas de la plantilla.

Mediante el uso de la directiva #parse, puedes incluir plantillas que se encuentran en la pestaña Incluir. Para incluir una plantilla, especifica el nombre completo de la plantilla entre comillas como parámetro (por ejemplo, #parse("File Header")).
Lista de variables predefinidas
${GO_PACKAGE_NAME_WITH_TEST_SUFFIX} Nombre del paquete en el que se crea el archivo de prueba. El archivo debe tener el sufijo _test. El nombre tiene el siguiente formato: package_test.
${GO_PACKAGE_NAME} Nombre del paquete en el que se crea el archivo nuevo.
${DATE} Fecha actual del sistema.
${DAY} Día del mes actual.
${DAY_NAME_SHORT} Primeras 3 letras del nombre del día actual (por ejemplo, Mon, Tue, etc.).
${DAY_NAME_FULL} Nombre completo del día actual (por ejemplo, Monday, Tuesday, etc.).
${DIR_PATH} Ruta al directorio donde se coloca el archivo nuevo. Esta ruta es relativa a la raíz del proyecto.
${DS} Signo de dólar ($). Esta variable se utiliza para hacer escape del carácter de dólar para que no se trate como prefijo de una variable de plantilla.
${FILE_NAME} Nombre del archivo nuevo.
${HOUR} Hora actual.
${MINUTE} Minuto actual.
${SECOND} Segundo actual.
${MONTH} Mes actual.
${MONTH_NAME_SHORT} Primeras 3 letras del nombre del mes actual (por ejemplo, Jan, Feb, etc.).
${MONTH_NAME_FULL} Nombre completo del mes actual (por ejemplo, January, February, etc.).
${NAME} Nombre de la entidad nueva: archivo, tipo, interfaz, etc.
${PRODUCT_NAME}
