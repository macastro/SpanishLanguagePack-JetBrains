Borrador de Go es una plantilla integrada que se utiliza cada vez que se crea un nuevo archivo temporal de Go. Los archivos temporales son archivos completamente funcionales, ejecutables y depurables, que puede utilizar para esbozar sus ideas de código. Para crear un archivo temporal, haga clic en Archivo | Nuevo | Archivo Temporal. En el diálogo Nuevo Archivo Temporal, busque y seleccione Go.

La plantilla es editable. En una plantilla, puede utilizar variables predefinidas que se listan a continuación. Estas variables se expanden a los valores correspondientes más adelante en el editor.

También es posible especificar variables personalizadas. Las variables personalizadas utilizan el siguiente formato: ${VARIABLE_NAME}, donde VARIABLE_NAME es un nombre para su variable (por ejemplo, ${MY_CUSTOM_FUNCTION_NAME}). Antes de que el IDE cree un nuevo archivo con variables personalizadas, verá un diálogo donde puede definir valores para las variables personalizadas en la plantilla.

Mediante la directiva #parse, puede incluir plantillas de la pestaña Incluir. Para incluir una plantilla, especifique el nombre completo de la plantilla como parámetro entre comillas (por ejemplo, #parse("Encabezado de Archivo")).
Lista de variables predefinidas
${GO_PACKAGE_NAME_WITH_TEST_SUFFIX}   Nombre del paquete en el que se crea un nuevo archivo de prueba. El archivo debe tener el sufijo _test. El nombre tiene el siguiente formato: package_test.
${GO_PACKAGE_NAME}   Nombre del paquete en el que se crea un nuevo archivo.
${DATE}   Fecha actual del sistema.
${DAY}   Día actual del mes.
${DAY_NAME_SHORT}   Primeras tres letras del nombre del día actual (por ejemplo, Lun, Mar, etc.).
${DAY_NAME_FULL}   Nombre completo del día actual (por ejemplo, Lunes, Martes, etc.).
${DIR_PATH}   Ruta al directorio del nuevo archivo. Esta ruta es relativa a la raíz del proyecto.
${DS}   Signo de dólar ($). Esta variable se utiliza para escapar el carácter de dólar, para que no se trate como prefijo de una variable de plantilla.
${FILE_NAME}   Nombre del nuevo archivo.
${HOUR}   Hora actual.
${MINUTE}   Minuto actual.
${SECOND}   Segundo actual.
${MONTH}   Mes actual.
${MONTH_NAME_SHORT}   Las primeras 3 letras del nombre del mes actual (por ejemplo, Ene, Feb, etc.).
${MONTH_NAME_FULL}   Nombre completo del mes actual (por ejemplo, Enero, Febrero, etc.).
${NAME}   Nombre de una nueva entidad: un archivo, un tipo, una interfaz, etc.
${PRODUCT_NAME}   Nombre del IDE.
${PROJECT_NAME}   Nombre del proyecto actual.
${TIME}   Hora actual del sistema.
${USER}   Nombre de inicio de sesión del sistema del usuario actual.
${YEAR}   Año actual.
