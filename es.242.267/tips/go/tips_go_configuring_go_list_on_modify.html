Configuración del comportamiento de 'go list'

Si no desea ejecutar go list con cada cambio realizado en go.mod, pulse &shortcut:ShowSettings; para abrir la configuración y vaya a "Build, Execution, Deployment | Build Tools". Desactive la casilla de selección "Reload project after build script changes".

De manera predeterminada, GoLand recarga el proyecto después de un cambio en el archivo go.mod. Si tiene seleccionada la opción "External changes", no se ejecutará go list automáticamente cuando edite archivos en el IDE. En este caso, se mostrará el icono "Cargar cambios del módulo Go". Una vez haya terminado de editar, haga clic en el icono para cargar los cambios.
