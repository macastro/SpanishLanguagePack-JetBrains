Informa el tipo de retorno incorrecto de los métodos con la anotación @Deployment.

Según la documentación de Arquillian, los métodos que tienen la anotación @Deployment, deben tener un tipo de retorno que herede de ShrinkWrap Archive.

Ejemplo:


  // Este caso de prueba no podrá ser iniciado por Arquillian
  @RunWith(Arquillian.class)
  public class SomeTest {
    @Deployment
    public static ZipFile createDeployment() {
      throw new UnsupportedOperationException("Implement me");
    }

    @Test
    public void testSomething() {
      Assert.fail("To be implemented");
    }
  }

Luego de aplicar el arreglo rápido:


  import org.jboss.shrinkwrap.api.Archive;

  @RunWith(Arquillian.class)
  public class SomeTest {
    @Deployment
    public static Archive createDeployment() {
      throw new UnsupportedOperationException("Implement me");
    }

    @Test
    public void testSomething() {
      Assert.fail("To be implemented");
    }
  }
