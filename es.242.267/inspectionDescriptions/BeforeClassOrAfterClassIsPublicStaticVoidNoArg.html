Métodos JUnit 4.0 @BeforeClass o @AfterClass que no se declaran como public y static, no devuelven void o aceptan argumentos reales. Este tipo de métodos se pueden crear fácilmente por error, pero el ejecutor de pruebas JUnit no los ejecutará.

Ejemplo:


  public class MainTest {
    @BeforeClass
    String beforeClass(int i) {
    }
  }

Después de aplicar la corrección rápida, el método se convierte en:


  class MainTest {
    @BeforeClass
    public static void beforeClass() {
    }
  }
