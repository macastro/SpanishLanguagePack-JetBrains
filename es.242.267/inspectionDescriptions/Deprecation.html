Informa de los usos de clases, campos y métodos obsoletos. Hay disponible una solución rápida para convertir automáticamente el uso obsoleto, cuando la información necesaria se puede extraer de la documentación Javadoc del miembro obsoleto.

Ejemplo:


  class Interessante {

    /**
     * @deprecated Use {@link #newHotness()} en su lugar
     */
    @Deprecated
    public void oldAndBusted() {}

    public void newHotness() {}
  }
  class EnOtraParte {
    void x(Interesante i) {
      i.oldAndBusted(); // advertencia de obsoleto aquí
    }
  }

Después de aplicar la solución rápida:


  class Interessante {

    /**
     * @deprecated Use {@link #newHotness()} en su lugar
     */
    @Deprecated
    public void oldAndBusted() {}

    public void newHotness() {}
  }
  class EnOtraParte {
    void x(Interesante i) {
      i.newHotness();
    }
  }

De forma predeterminada, la inspección no produce una advertencia si es imposible o difícil evitarla. Por ejemplo, el siguiente código no será informado:


  abstract class A { //código de biblioteca
     @Deprecated
     abstract void m();
  }
  class B extends A { //código de proyecto
     @Override
     void m() {
        //doSmth;
     }
  }

Configurar la inspección:

Utiliza las opciones para desactivar esta inspección dentro de miembros obsoletos, anulaciones de métodos obsoletos abstractos, sentencias de importación no estáticas, métodos de clases obsoletas o mismas clases de nivel superior.
