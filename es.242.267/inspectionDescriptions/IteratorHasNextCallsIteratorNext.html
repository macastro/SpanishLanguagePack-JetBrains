Informa sobre implementaciones de Iterator.hasNext() o ListIterator.hasPrevious() que llaman a Iterator.next() o ListIterator.previous() en la instancia del iterador. Esas llamadas son casi con toda seguridad un error, ya que los métodos como hasNext() no deberían modificar el estado de los iteradores, mientras que next() sí debería hacerlo.

Ejemplo:


  class MyIterator implements Iterator<Integer> {
    public boolean hasNext() {
      return next() != null;
    }
  }
