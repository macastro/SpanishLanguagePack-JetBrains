Informa sobre métodos con nombres similares a los accesorios que tienen una cláusula de parámetro vacía.

Se espera que los métodos que siguen el contrato de nombres JavaBean para los accesorios no tengan efectos secundarios. La convención recomendada para estos métodos es utilizar un método sin parámetros siempre que no haya parámetros.

Esta convención promueve el principio de acceso uniforme, que indica que el código del cliente no debe verse afectado por una decisión de implementar un atributo como campo o método.

La solución rápida elimina la cláusula de parámetro vacía.

Ejemplo:


  trait Test {
    def hasProperty(): Boolean
  }

Después de aplicar la solución rápida:


  trait Test {
    def hasProperty: Boolean
  }

Esta inspección no informará sobre los métodos que anulan otros miembros.
