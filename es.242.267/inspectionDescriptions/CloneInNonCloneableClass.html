Se informa de clases que sobreescriben el método clone() pero que no implementan la interfaz Cloneable. Esto, a menudo, representa un error de programación.

Se deben usar las opciones Advertencia sólo para métodos clone 'públicos' para ignorar métodos que no sean public.

En el caso de clases que están diseñadas para ser heredadas, usted puede optar por no implementar la interfaz Cloneable, sino que sobreescribir clone() y declararla como protected, y decidir implementa Cloneable o no en subclases.
