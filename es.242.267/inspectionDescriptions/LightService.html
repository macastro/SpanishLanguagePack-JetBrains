Se informan los siguientes problemas de servicios ligeros:
  • Las clases anotadas con @com.intellij.openapi.components.Service no son finales.
  • Los servicios que tienen constructores que aceptan el parámetro de tipo com.intellij.openapi.project.Project no están anotados como servicios de nivel de proyecto.
  • Los servicios que están anotados como servicios de nivel de aplicación no tienen constructores sin parámetros o constructores que acepten kotlinx.coroutines.CoroutineScope.
  • Hay una falta de coincidencia al recuperar el servicio: se obtiene un servicio de nivel de proyecto como servicio de nivel de aplicación o viceversa.

Para obtener más información, consulta Servicios ligeros.
