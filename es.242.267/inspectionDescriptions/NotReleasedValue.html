Inspecciona si hay objetos propios (retained objects) almacenados en variables locales o parámetros que más adelante no son liberados o son sobreescritos por otro valor antes de ser liberados.
- Si creas un objeto utilizando un método cuyo nombre empieza por alloc o new, que contiene copy o si le envías un mensaje retain, obtendrás la propiedad de ese objeto.
- Puedes renunciar a la propiedad de un objeto usando release o autorelease.
Las reglas de gestión de memoria de Objective-C requieren que liberes o liberes automáticamente cada objeto propio (retained object). Consulta Memory Management Programming Guide para obtener más información.
Esta inspección estará deshabilitada si está habilitada ARC (conteo automático de referencias).
