Informa de los constructores cuyo número de parámetros excede el máximo especificado. Estos objetos son difíciles de instanciar, especialmente si algunos parámetros son opcionales. Los constructores con demasiados parámetros pueden indicar que es necesaria una refactorización. Por ejemplo, considera aplicar el patrón de constructor.

Ejemplo:


  public BankAccount(long accountNumber,
                     String owner,
                     double balance,
                     double interestRate) {
    // inicialización de campos
  }

Configurar la inspección:

  • Utiliza el campo Límite de parámetros para especificar el número máximo de parámetros permitidos en un constructor.
  • Utiliza la lista Ignorar constructores con visibilidad para especificar si la inspección debe ignorar constructores con visibilidad específica.
