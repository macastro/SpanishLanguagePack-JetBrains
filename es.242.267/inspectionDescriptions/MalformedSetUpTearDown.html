Reporta métodos JUnit 3 `setUp()` o `tearDown()` que no están declarados como `public`, no retornan `void` o aceptan parámetros formales. Tales métodos ` setUp()` o `tearDown()` probablemente fueron creados por error y el ejecutor de pruebas de JUnit no los ejecutará.

Ejemplo:


  public class WeightyTest extends TestCase {
    private File file;
    private int setUp(String prefix) throws IOException {
      file = File.createTempFile(prefix, ".tmp");
      return 0;
    }

    // ... Las pruebas van aquí
  }
