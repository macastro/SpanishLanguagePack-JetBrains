Informa las variables locales que tienen el mismo nombre exacto que los campos de la clase externa. Tales nombres pueden hacer que se use por error la variable cuando se pretende usar el campo con el mismo nombre.

Se sugiere un arreglo rápido para cambiar el nombre de la variable.

Ejemplo:


  public class Foo {
    public Object foo;

    void bar() {
      Object o = new Object() {
        void baz() {
          Object foo; // La variable local 'foo' oculta el campo 'Foo' de la clase
        }
      };
    }
  }

Puede configurar las siguientes opciones para esta inspección:

  1. Ignorar campos inaccesibles - Ignorar variables locales con el mismo nombre exacto de un campo de superclase que no sea visible (por ejemplo, porque el campo es privado).
  2. Ignorar variables locales en contextos estáticos que ocultan campos no estáticos - Ignorar, por ejemplo, cuando la variable local está dentro de un método estático o dentro de un método de una clase interna estática.
