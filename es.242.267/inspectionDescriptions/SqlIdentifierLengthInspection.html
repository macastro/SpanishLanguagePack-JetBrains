El informe considera a los identificadores muy largos. La longitud máxima admitida varía según la base de datos.

Ejemplo (MySQL):

CREATE TABLE ooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo
(
   ooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo int
);

En MySQL, los identificadores no pueden tener más de 64 caracteres.

Considera las siguientes restricciones para diferentes bases de datos:

  • Sybase ASE:

    • Nombres entre comillas: 253 caracteres
    • Nombres sin comillas: 255 caracteres

  • Db2 LUW:

    • BUFFER POOL, INDEX EXTENSION, TABLESPACE, TRANSFORM, TYPE MAPPING: 18 caracteres
    • Otros objetos: 128 caracteres
  • Db2 iSeries: 128 caracteres

  • Db2 z/OS:

    • DATABASE y TABLESPACE: 8 caracteres
    • Columnas y cursores con WITH RETURN: 30 caracteres
    • Otros objetos: 128 caracteres
  • Derby: 128 caracteres
  • Exasol: 128 caracteres
  • HSQLDB: 128 caracteres

  • MySQL:

    • Alias en consultas: 256 caracteres
    • USER y ROLE: 32 caracteres
    • Otros objetos: 64 caracteres

  • MariaDB:

    • USER: 80 caracteres
    • ROLE: 128 caracteres
    • Otros objetos: igual que MySQL
  • Microsoft SQL Server: 128 caracteres
  • Vertica: 128 caracteres
  • SQL 92: 128 caracteres
  • Snowflake: 255 caracteres
  • Redshift: 127 caracteres
  • PostgreSQL/Greenplum/Cockroach: 63 caracteres

  • Oracle:

    • Nombres largos habilitados (database.oracle.names.long especificado, y Oracle es anterior a 12.2 o se desconoce):

      • DISKGROUP, ROLLBACK SEGMENT, TABLESPACE, PLUGGABLE DATABASE: 30 caracteres
      • DATABASE: 8 caracteres
      • Otros objetos: 128 caracteres
    • Nombres largos deshabilitados:
      • DATABASE: 8 caracteres
      • DATABASE LINK: 128 caracteres
      • Otros objetos: 30 caracteres
