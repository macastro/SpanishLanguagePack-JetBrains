Inspecciona los bucles for-in sin filtrar.

El uso de este método da como resultado que se procesen no solo las propiedades del propio objeto sino también las propiedades de su prototipo. Puede ser un comportamiento inesperado en algunos casos concretos, por ejemplo en métodos utilitarios que copian o modifican todas las propiedades, o cuando el prototipo de Object pueda ser modificado incorrectamente. Por ejemplo, el siguiente código imprimirá 42 y myMethod: 


Object.prototype.myMethod = function myMethod() {};
let a = { foo: 42 };
for (let i in a) {
  console.log(a[i]);
}

Sugiere reemplazar el bucle entero con el método Object.keys() o añadir una comprobación de hasOwnProperty(). Después de aplicar el arreglo rápido, el código se ve así:


for (let i in a) {
  if (a.hasOwnProperty(i)) {
    console.log(a[i]);
  }
}
