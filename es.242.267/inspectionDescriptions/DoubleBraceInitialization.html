Informa Inicialización de llaves dobles.

La inicialización de llaves dobles puede ocasionar una fuga de memoria si se utiliza en un contexto estático, ya que crea una clase anónima que hará referencia al objeto envolvente.

La inicialización de llaves dobles presenta un peor rendimiento en comparación con la inicialización normal, ya que requiere cargar una clase adicional.

Además, si el método equals() no acepta una subclase como parámetro, es posible que falle la comparación equals().

Asimismo, en las versiones anteriores a Java 9, la inicialización de llaves dobles no podía combinarse con el operador de diamante, ya que era incompatible con las clases anónimas.

Ejemplo:


  List<Integer> list = new ArrayList<>() {{
    add(1);
    add(2);
  }};

Después de aplicar la corrección rápida:


  List<Integer> list = new ArrayList<>();
  list.add(1);
  list.add(2);
