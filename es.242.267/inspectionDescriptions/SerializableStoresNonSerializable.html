Serializable informa sobre cualquier referencia a variables locales no Serializable fuera de lambdas, clases locales y clases anónimas.

Cuando una variable local es referenciada desde una clase anónima, su valor se almacena en un campo implícito de la clase. Un comportamiento similar ocurre con las clases locales y las lambdas. Si el tipo de la variable no es Serializable, la serialización fallará.

Ejemplo:


  interface A extends Serializable {
      abstract void foo();
  }
  class B {}
  class C {
      void foo() {
          B b = new B();
          A a = new A() {
              @Override
              public void foo() {
                  System.out.println(b); // advertencia
              }
          };
      }
  }
