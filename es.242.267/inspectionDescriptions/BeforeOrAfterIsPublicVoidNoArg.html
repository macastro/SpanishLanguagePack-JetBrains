Se informa de métodos JUnit 4 @Before o @After que no se declaran como public, devuelven algo distinto de void o aceptan argumentos. Estos métodos se pueden crear fácilmente por descuido, pero el ejecutor de pruebas JUnit no los ejecutará.

Esta inspección proporciona una corrección para la signatura del método.

Ejemplo:


  @Before private int foo(int arg) { ... }

Después de aplicar la corrección rápida:


  @Before public void foo() { ... }
