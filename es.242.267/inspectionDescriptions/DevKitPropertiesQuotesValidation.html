El número de comillas alrededor del argumento en el valor de propiedad es incorrecto.

Si hay argumentos en el valor de propiedad, entonces el valor debe pasarse como un patrón a java.text.MessageFormat. Esto significa que se usan comillas simples para el escape. Por lo tanto, para que las comillas sean visibles, se deben usar comillas repetidas:

  • '{0}' → {0}
  • ''{0}'' → '$value$'

Cuando se utiliza el formato de choice, el formato anidado se evalúa como un formato en sí mismo, por lo que se debe agregar una segunda comilla doble. Cuando se pasa 1:

  • {0, choice, 0#no|#1''{0}'' file} → {0} files
  • {0, choice, 0#no|#1''''{0}'''' file} → '$number_of_files$' files
