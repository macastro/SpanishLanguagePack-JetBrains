Esta es una plantilla utilizada para crear un método de prueba WebSocket para Ktor 1.x.x en una clase de prueba JUnit 4.
Las variables predefinidas tomarán los siguientes valores:
${NAME}   Nombre del método creado
${ROUTE}   Ruta, relativa al servicio actual, que necesita ser probada
${CONFIGURE_SERVER}   Un método de extensión de ruta que configura un árbol de enrutamiento para la parte del servidor que necesita ser probada
${BODY}   Cuerpo del método generado
