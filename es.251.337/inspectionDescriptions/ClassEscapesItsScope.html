Informa sobre el uso de clases cuya visibilidad es menor que la visibilidad del miembro que las utiliza en firmas de campos o métodos. Aunque estos miembros son Java válido, no pueden utilizarse fuera del ámbito de visibilidad de los tipos de clase a los que hacen referencia.

Ejemplo:


  public class Parent {
    public Child getChild() {
      return new Child();
    }
    private class Child {}
  }

Además, en Java 9 y versiones posteriores, los módulos pueden ocultar algunas de sus clases a otros módulos al no exportar sus paquetes. Sin embargo, si los miembros que forman parte de la API exportada hacen referencia a clases no exportadas en sus firmas, dichos miembros no pueden utilizarse fuera del módulo.

Configuración de la inspección:

  • Utilice la opción Informar clases no exportadas expuestas en API de módulo (Java 9+) para informar sobre miembros de API de módulo que exponen clases no exportadas.
    Tenga en cuenta que para esta opción, el nivel de lenguaje del proyecto o módulo debe ser 9 o superior.
  • Utilice la opción Informar clases inaccesibles expuestas en API pública para informar sobre miembros públicos que exponen clases con menor visibilidad.
  • Utilice la opción Informar clases privadas expuestas en API package-local para informar sobre miembros package-local que exponen clases private.
