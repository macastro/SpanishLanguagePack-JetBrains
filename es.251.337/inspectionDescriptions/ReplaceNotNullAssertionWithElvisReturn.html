El informe puede reemplazarse con el operador elvis y return (?: return) en lugar de la aserción non-null (!!).

La aserción non-null puede producir NPE (NullPointerException) inesperados. Se recomienda evitar el uso de !!.

Esta corrección rápida reemplazará la aserción non-null con return o return null.

Ejemplo:


  fun test(number: Int?) {
      val x = number!!
  }

Después de aplicar la corrección rápida:


  fun test(number: Int?) {
      val x = number ?: return
  }
