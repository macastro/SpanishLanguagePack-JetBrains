Se detectan archivos YAML/JSON que son susceptibles de ser interpretados como parte de una especificación OpenAPI/Swagger.

Este tipo de archivos no contiene características de especificación explícitas, pero se refieren desde el archivo de especificación principal cercano.

El siguiente ejemplo contiene dos archivos ubicados en el mismo directorio. El primero es un archivo de especificación principal genérico. El segundo es referenciado por el primero, por lo que se propone considerarlo una especificación.

Archivo de especificación principal `openapi.yaml`:


openapi: 3.1.0
components:
  schemas:
    CustomSchema:
      description: Custom schema object
      properties:
        foo:
          $ref: 'common.components.yaml#/components/schemas/CommonSchema'

Candidato a archivo de especificación `common.components.yaml`:


components: # 'Mark file as OpenAPI specification' highlighting
  schemas:
    CommonSchema:
      description: Common schema object reused in several specifications
