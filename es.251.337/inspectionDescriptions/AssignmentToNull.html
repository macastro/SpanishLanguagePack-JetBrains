Informa sobre variables que se asignan a null fuera de las declaraciones.

El uso principal de null en Java es representar variables de referencia no inicializadas. En casos muy raros, la asignación explícita de una variable a null puede ayudar con la recolección de basura. Sin embargo, usar null para representar valores ausentes, no especificados, inválidos o elementos no encontrados se considera una mala práctica y puede hacer que su código sea más propenso a NullPointerExceptions. En su lugar, considere definir objetos centinela con la semántica esperada o utilizar tipos de biblioteca como Optional para representar la ausencia de un valor.

Ejemplo:


  Integer convert(String s) {
    Integer value;
    try {
      value = Integer.parseInt(s);
    } catch (NumberFormatException e) {
      // Advertencia: null se usa para representar "valor inválido"
      value = null;
    }
    return value;
  }

Utilice la opción ignorar asignaciones a campos para ignorar las asignaciones a campos.
