Informa sobre declaraciones if, while, do o for sin llaves. Algunos estilos de código, como la Guía de Estilo de Google para Java, requieren que todas las declaraciones de control incluyan llaves.

Al agregar más declaraciones a una estructura de control sin llaves, es importante no olvidar añadirlas. También se debe tener especial cuidado al comentar una línea de código cuando no se usan llaves, para evitar que la siguiente declaración se convierta inadvertidamente en parte de la estructura de control. Usar siempre llaves permite insertar o comentar líneas de código de manera más segura.

Si siempre se sigue un estilo de código que utiliza llaves, es menos probable que ocurran vulnerabilidades como el fallo goto. Las estructuras de control con llaves hacen que el flujo de control sea más fácil de visualizar, sin depender de una indentación que podría ser incorrecta.

Ejemplo:


  class Strange {
    void x(boolean one, boolean two) {
      if(one)
        if(two)
          foo();
      else
        bar();
    }
    void foo() {}
    void bar() {}
  }

Corrección rápida envolviendo el cuerpo de las declaraciones con llaves:


  class Strange {
    void x(boolean one, boolean two) {
      if(one) {
        if(two) {
          foo();
        } else {
          bar();
        }
      }
    }
    void foo() {}
    void bar() {}
  }
