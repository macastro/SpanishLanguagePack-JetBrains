Informa sobre campos no serializables en clases que implementan java.io.Serializable. Estos campos causarán una excepción en tiempo de ejecución si el objeto es serializado.

No se informan los campos declarados como transient o static, ni los campos de clases que definen un método writeObject.

Esta inspección asume que los campos de tipo java.util.Collection y java.util.Map son Serializable, a menos que sus tipos declarados sean no Serializable.

Ejemplo:


  class NonSerializableClass {}
  public class SerializableClass implements Serializable {
      NonSerializableClass clazz; // //advertencia: campo no serializable 'clazz' en clase serializable
      static NonSerializableClass staticClazz; // sin advertencia
  }

Configure la inspección con las siguientes opciones:

  • Lista de clases cuyos herederos no deberían ser reportados por esta inspección. Esto es aplicable a clases que heredan Serializable de una superclase pero no se utilizan para serialización.
  • Lista de anotaciones que harán que la inspección ignore los campos anotados.
  • Si se deben ignorar los campos inicializados con clases anónimas.
