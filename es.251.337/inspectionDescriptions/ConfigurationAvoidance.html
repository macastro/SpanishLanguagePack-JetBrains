(Gradle 4.9+) Detecta el uso de APIs para la interacción anticipada con tareas.

La interacción anticipada con tareas implica varios problemas:

  • Los usuarios deben configurar manualmente todas las dependencias entre tareas;
  • Durante la fase de configuración, todas las tareas accedidas mediante APIs anticipadas son configuradas, incluso si no se ejecutan posteriormente. Esto resulta en una degradación del rendimiento.

Finalmente, las APIs anticipadas serán deprecadas en favor de las APIs diferidas.

Para consultar la guía de migración, consulte la documentación de Gradle.

Ejemplo:

task foo { // reporta 'task', sugiere reemplazarlo por 'task.register'
  // ...
}
