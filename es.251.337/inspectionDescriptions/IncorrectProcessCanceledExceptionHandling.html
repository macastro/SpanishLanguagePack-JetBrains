Informa cuando `ProcessCanceledException` es manejada incorrectamente.

`ProcessCanceledException` y sus herederas no deben ser cacheadas, leídas, registradas o procesadas de ninguna manera. En cambio, deben ser lanzadas nuevamente para que la infraestructura las maneje correctamente.

La inspección reporta tanto el cacheo explícito de `ProcessCanceledException` o sus herederas, como el cacheo de `RuntimeException`, `Exception` y `Throwable` que cubren `ProcessCanceledException`.

Ejemplos:


// mal:
try {
  // ...
} catch (ProcessCanceledException e) { // La excepción no debe ser leída
}

// mal:
try {
  // ...
} catch (ProcessCanceledException e) {
  LOG.error("Error ocurrido", e); // La excepción no debe ser registrada
  throw e;
}


// bien:
try {
  // ...
} catch (ProcessCanceledException e) {
  // Otra acción
  throw e;
}
