Inspecciona las asignaciones de objetos o conjuntos dentro de los bucles. Si bien no es necesariamente un problema, una asignación de objetos dentro de un bucle es un excelente lugar para buscar fugas de memoria y problemas de rendimiento.

La inspección informa las siguientes construcciones:

  • Asignaciones explícitas mediante el operador new
  • Métodos que se sabe que devuelven un nuevo objeto
  • Referencias de método vinculadas a la instancia
  • Lambdas que capturan variables o this

Ejemplo:


  // Asignación explícita
  for (Status status : Status.values()) {
    declarationsMap.put(status, new ArrayList<>());
  }

  // Lambda captura una variable
  String message = "Motor en marcha.";
  for (Engine engine : engines) {
      if (!isRunning(engine)) {
          logger.warn(() -> {
              return String.format(message);
          });
      }
  }

  // Referencia de método vinculada a la instancia
  for(Node node : nodes) {
    descriptor = node.getDescription();
    descriptor.ifPresent(dynamicTestExecutor::execute);
  }
