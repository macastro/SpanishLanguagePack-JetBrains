Reporta los bloques catch con parámetros que son más genéricos que la excepción emitida por el bloque try correspondiente.

Ejemplo:


  try  {
    File file = new File(pathToFile);
    return file.getAbsolutePath();
  } catch (Exception ex) { // advertencia: 'catch' de 'Exception' es demasiado amplia, enmascara las excepciones 'RuntimeException'
    return defaultFilePath;
  }

Después de aplicar la corrección rápida:


  try  {
    File file = new File(pathToFile);
    return file.getAbsolutePath();
  } catch (RuntimeException ex) {
    return defaultFilePath;
  }

Configurar la inspección:

  • Use la opción Advertir solo sobre RuntimeException, Exception, Error o Throwable para que esta inspección advierta solo sobre las excepciones más genéricas.
  • Use la opción Ignorar excepciones que ocultan a otras pero que son lanzadas por sí mismas para ignorar cualquier excepción que oculte a otras excepciones pero que aún pueda ser lanzada y, por lo tanto, técnicamente no es demasiado amplia.
