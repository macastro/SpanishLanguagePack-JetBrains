Reporta excepciones lanzadas desde dentro de un bloque catch que no "envuelven" la excepción capturada.

Cuando se lanza una excepción en respuesta a otra excepción, envolver la excepción inicial evita la pérdida de información contextual valiosa, como los marcos de pila y números de línea.

Ejemplo:


  ...
  catch (IOException e) {
    closeAllConnections();
    throw new ConnectException("Connection problem."); // Advertencia: 'throw' en bloque 'catch' ignora la excepción capturada 'e'
  }

Configuración de la inspección:

  • Use la opción Ignorar si se usa el resultado de un método de excepción para indicar si la inspección debe ignorar las excepciones cuyos argumentos son el resultado de llamadas a métodos de la excepción original, como getMessage().
  • Use la opción Ignorar si la excepción lanzada no puede envolver la excepción para ignorar las declaraciones throw que lanzan excepciones sin un constructor que acepte una cláusula Throwable.
