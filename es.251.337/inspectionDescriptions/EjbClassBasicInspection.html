Se informa de las siguientes infracciones de la especificación EJB:
  • La clase no implementa la interfaz requerida (p. ej., EJBLocalObject, EJBObject)
  • La clase no define un constructor público sin argumentos
  • Se usa una clase en lugar de una interfaz (o viceversa)
  • El método EJB es estático
  • La clase del bean de sesión sin estado define varios métodos `ejbCreate`
  • Tipo de retorno de método de clase de bean de sesión incorrecto
  • EJB define el método `finalize`

Ejemplo:


public class SessionEJB implements java.io.Serializable { // Error: La clase debe extender `SessionBean`
  private SessionEJB() {} // Error: La clase debe definir un constructor público sin argumentos
}
