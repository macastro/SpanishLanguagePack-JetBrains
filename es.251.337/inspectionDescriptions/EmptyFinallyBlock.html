Informa sobre bloques finally vacíos.

Los bloques finally vacíos generalmente indican un error de codificación. Pueden permanecer después de una refactorización del código y se pueden eliminar de forma segura.

Esta inspección no informa sobre bloques finally vacíos encontrados en archivos JSP.

Ejemplo:


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  } finally {
  }

Después de aplicar la corrección rápida: 


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  }
