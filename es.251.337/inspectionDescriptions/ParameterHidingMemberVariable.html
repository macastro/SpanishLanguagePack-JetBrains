Informa de parámetros de método con nombre idéntico al de un campo de una clase envolvente. Como resultado de esta denominación, puedes usar accidentalmente el parámetro cuando se pretende usar el campo con nombre idéntico.

Se sugiere una corrección rápida para cambiar el nombre del parámetro.

Ejemplo:


  class Main {
    private String valor;

    public Main(String valor) {
      valor = valor.toUpperCase();
    }
  }

Puedes configurar las siguientes opciones para esta inspección:

  1. Ignorar para definidores de propiedades: ignorar parámetros de definidores simples.
  2. Ignorar campos de superclase no visibles desde subclase: ignorar campos private en una superclase que no son visibles desde el método.
  3. Ignorar para constructores: ignorar parámetros de constructores.
  4. Ignorar para métodos abstractos: ignorar parámetros de métodos abstractos.
  5. Ignorar para parámetros de método estático que ocultan campos de instancia: ignorar parámetros de métodos static que ocultan un campo de instancia e ignorar parámetros de métodos de instancia en clases internas estáticas que ocultan un campo de instancia de una clase externa. Aunque no lo ocultan estrictamente, estos parámetros pueden resultar confusos.
