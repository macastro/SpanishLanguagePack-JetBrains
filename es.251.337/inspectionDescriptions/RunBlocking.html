Informe sobre los constructores de runBlocking alcanzables desde corrutinas.

Las corrutinas comparten el hilo utilizado para su ejecución. Cuando se llama a un constructor de runBlocking desde una corrutina, esta bloquea el hilo subyacente, impidiendo que otras corrutinas utilicen este recurso. Esto puede provocar problemas de rendimiento y, en el peor de los casos, inanición de hilos.

Proporciona una pila de llamadas potencial desde una primitiva de corrutina (función o constructor de suspensión) hasta un constructor de runBlocking.

Ejemplo: 


  suspend fun main() {
      foo() 
   }

  fun foo() {
      runBlocking { suspendFunction() }
  }

Posible solución: 


  suspend fun main() {
      foo() 
   }

  suspend fun foo() {
      suspendFunction()
  }

Use la opción Inspeccionar funciones con anulaciones para configurar el análisis de funciones virtuales:

  • Utilice la opción No si desea omitir el análisis de las funciones con anulaciones.
  • Utilice la opción Sí, excluir anulaciones para inspeccionar funciones solamente en los tipos de llamadores definidos.
  • Utilice la opción Sí, incluir anulaciones para inspeccionar las funciones y todas las anulaciones.
