Informa sobre cualquier uso de java.util.Optional<T>, java.util.OptionalDouble, java.util.OptionalInt, java.util.OptionalLong o com.google.common.base.Optional como tipos de campos o parámetros.

Optional está diseñado como un mecanismo limitado para tipos de retorno en métodos de biblioteca, cuando se necesita una forma explícita de representar "sin resultado".

El uso de campos de tipo java.util.Optional también es problemático si la clase necesita ser Serializable, ya que java.util.Optional no es serializable.

Ejemplo:


  class MyClass {
    Optional<String> name; // Campo Optional
    // Parámetro Optional
    void setName(Optional<String> name) {
      this.name = name;
    }
  }
