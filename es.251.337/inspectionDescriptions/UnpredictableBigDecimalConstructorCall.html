Informa de llamadas a constructores de BigDecimal que aceptan un valor double. Estos constructores producen BigDecimal que es exactamente igual que el valor double suministrado. Sin embargo, como los dobles se codifican mediante el formato de punto flotante binario de doble precisión de 64 bits IEEE 754, el valor exacto puede ser inesperado.

Por ejemplo, new BigDecimal(0.1) genera un objeto BigDecimal. Su valor es 0.1000000000000000055511151231257827021181583404541015625 que es el número más cercano a 0,1 que puede representarse como un doble. Para obtener BigDecimal que almacena el mismo valor que está escrito en el código fuente, use new BigDecimal("0.1") o BigDecimal.valueOf(0.1).

Ejemplo:


  class Constructor {
    void foo() {
      new BigDecimal(0.1);
    }
  }

Después de aplicar la corrección rápida:


  class Constructor {
    void foo() {
      new BigDecimal("0.1");
    }
  }
