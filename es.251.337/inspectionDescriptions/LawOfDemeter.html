Informa sobre violaciones de la Ley de Demeter.

La Ley de Demeter no es una ley real, sino que especifica una directriz de estilo: nunca invocar métodos de objetos obtenidos de otra llamada. El código que sigue esta directriz es más fácil de mantener, adaptar y refactorizar, tiene menos acoplamiento entre métodos, menos duplicación y mejor encapsulamiento de información. Por otro lado, es posible que necesites escribir varios métodos wrapper para cumplir con esta directriz.

Ejemplo:


  boolean pay(Customer c, Invoice invoice) {
    int dollars = c.getWallet().contents; // violación
    if (dollars >= invoice.getAmount()) {
      Wallet w = c.getWallet();
      w.subtract(invoice.getAmount()); // violación
      return true;
    }
    return false;
  }
El ejemplo anterior se implementaría mejor como un método payInvoice(Invoice invoice) en Customer.

Utiliza la opción Ignorar llamadas a métodos de biblioteca y accesos a campos de biblioteca para ignorar las violaciones de la Ley de Demeter que no se pueden corregir sin modificar las bibliotecas.
