Reporta errores de configuración en definiciones de <flow> basadas en XML de Spring Webflow.

  • Referencias de Bean no resueltas
  • Etiquetas o atributos requeridos faltantes
  • Tipos de atributos incorrectos
  • Inconsistencias en atributos enum
  • Tipos incorrectos de referencias a Bean

Ejemplo:


  <flow
    start-state="unknown-state" <!-- Cannot resolve symbol 'unknown-state' ->
    parent="unknownParentFlow,another-flow" <!-- No se puede encontrar el flujo 'unknownParentFlow' -->>
    <input name="" // Value must not be empty
         type="foo.com.UnknownType" <!--Cannot resolve class 'UnknownType'-->
         value="42"/>
    <exception-handler bean="dummyBean"; <!--El Bean debe ser del tipo 'org.springframework.webflow.engine.FlowExecutionExceptionHandler' -->
    <view-state id="validFlowButInvalidState_forParent"
           parent="another-flow#unknownParentState"/> <!-- No se puede resolver el estado padre 'unknownParentState' -->
    <bean-import resource="foo.xml" /> <!-- No se puede resolver el archivo 'foo.xml' -->
    <bean-import resource="highlighting-20-wrong-flow.xml"/> <!-- No es un archivo spring.xml válido -->
  </flow>
