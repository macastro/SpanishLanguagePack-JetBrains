Informa sobre casos donde se utilizan clases concretas en lugar de interfaces. Tales declaraciones pueden representar una falla en la abstracción y pueden dificultar las pruebas.

Esta inspección no informa sobre declaraciones cuyas clases provienen de bibliotecas del sistema o de terceros. No se informan las conversiones, instanceof y variables locales en implementaciones del método equals(). Además, no se informan las conversiones en implementaciones del método clone().

Ejemplo:


  interface Entity {}
  class EntityImpl implements Entity {}
  void processObject(Object obj) {
    // Advertencia: instanceof de clase concreta
    if (obj instanceof EntityImpl) {
      // Advertencia: conversión a clase concreta
      // en lugar de interfaz
      processEntity((EntityImpl)obj);
    }
  }
  // Advertencia: parámetro de clase concreta
  void processEntity(EntityImpl obj) {
  }

Utilice la opción Ignorar tipos de clases abstractas para ignorar las conversiones a clases abstractas.
