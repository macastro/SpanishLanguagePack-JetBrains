Informa sobre bloques catch vacíos o que posiblemente ignoren excepciones.

Aunque a veces es intencional, los bloques catch vacíos pueden dificultar la depuración. Además, ignorar el parámetro catch podría ser un error.

Esta inspección no informa sobre parámetros catch nombrados como ignore, ignored o _.

Puede utilizar la corrección rápida para cambiar el nombre de la excepción a _.

Ejemplo:


  try {
    throwingMethod()
  } catch (ex: IOException) {
  }

Después de aplicar la corrección rápida:


  try {
    throwingMethod()
  } catch (_: IOException) {
  }
Use la opción No avisar cuando el bloque 'catch' contenga comentarios para ignorar los bloques catch que contengan comentarios.
