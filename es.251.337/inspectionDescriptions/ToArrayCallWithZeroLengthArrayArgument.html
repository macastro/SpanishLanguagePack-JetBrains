Informa sobre llamadas no preferidas a Collection.toArray() y sugiere aplicar el estilo preferido.

Hay dos estilos para convertir una colección en un array:

  • Array con tamaño predeterminado, por ejemplo c.toArray(new String[c.size()])
  • Array vacío, por ejemplo c.toArray(new String[0])

En versiones antiguas de Java, se recomendaba usar arrays con tamaño predeterminado, ya que la llamada de reflexión necesaria para crear un array del tamaño apropiado era muy lenta.

Sin embargo, desde las últimas actualizaciones de OpenJDK 6, esta llamada ha sido optimizada, haciendo que la versión con array vacío tenga un rendimiento igual o incluso superior a la versión con tamaño predeterminado. Además, pasar un array con tamaño predeterminado es peligroso para colecciones concurrentes o sincronizadas, ya que puede ocurrir una condición de carrera entre las llamadas a size y toArray. Si la colección se reduce de manera concurrente durante la operación, puede resultar en valores null adicionales al final del array.

El estilo preferido puede seleccionarse usando las opciones de inspección.
