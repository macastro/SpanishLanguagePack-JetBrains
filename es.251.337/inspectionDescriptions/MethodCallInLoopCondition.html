Informa sobre llamadas a métodos en la parte condicional de las sentencias de bucle. Tales llamadas pueden afectar negativamente al rendimiento en entornos con recursos muy limitados.

La aplicación indiscriminada de esta inspección puede tener un impacto negativo en la claridad del código y el diseño. Esta inspección es relevante para Java ME y otros entornos con recursos muy limitados.

Ejemplo:


  String s = "example";
  for (int i = 0; i < s.length(); i++) {
    System.out.println(s.charAt(i));
  }

Después de aplicar la corrección rápida:


  String s = "example";
  int length = s.length();
  for (int i = 0; i < length; i++) {
    System.out.println(s.charAt(i));
  }

Esta opción permite ignorar las llamadas a métodos comunes de iteración de Java (como Iterator.hasNext()) y métodos conocidos con efectos secundarios (como Atomic*.compareAndSet).
