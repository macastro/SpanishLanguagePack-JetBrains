Reporta llamadas a wait() sin condiciones dentro de un contexto sincronizado.

Normalmente, wait() se usa para bloquear un hilo hasta que una condición sea verdadera. Si wait() se llama incondicionalmente, a menudo indica que la condición se verificó antes de adquirir un bloqueo. En ese caso, puede ocurrir una carrera de datos, donde la condición se vuelve verdadera entre el momento en que se verificó y el momento en que se adquirió el bloqueo.

Si bien las construcciones encontradas por esta inspección no son necesariamente incorrectas, ciertamente vale la pena examinarlas.

Ejemplo:


  class Bar {
      void foo() throws InterruptedException {
          synchronized (this) {
              wait(); // aviso
          }
      }
  }
