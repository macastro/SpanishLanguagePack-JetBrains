Informa sobre campos no final que se acceden tanto en contextos synchronized como no synchronized. Los campos volatile y los accesos en constructores e inicializadores son ignorados por esta inspección.

Estos accesos "parcialmente sincronizados" suelen ser descuidos de programación y pueden provocar inconsistencias inesperadas en las estructuras de datos.

Ejemplo:


  public class Program {
    Console console; // Advertencia: el campo 'console' se accede tanto en contexto sincronizado como no sincronizado
    public synchronized void execute() {
      console.print("running");
    }
    public void check() {
      console.check();
    }
  }

Utilice esta opción para especificar si los getters y setters simples también deben considerarse como accesos.
