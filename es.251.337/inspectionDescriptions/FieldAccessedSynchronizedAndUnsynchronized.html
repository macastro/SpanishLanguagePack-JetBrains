Reporta los campos no finales que son accedidos tanto en contextos synchronized como no synchronized. La inspección ignora los campos volatile así como los accesos en constructores e inicializadores.

Tales accesos "parcialmente sincronizados" a menudo son el resultado de un fallo en la codificación y pueden llevar a estructuras de datos inesperadamente inconsistentes.

Ejemplo:


  public class Program {
    Console console; // warning: El campo 'console' es accedido tanto en contextos sincronizados como no sincronizados

    public synchronized void execute() {
      console.print("running");
    }

    public void check() {
      console.check();
    }
  }

Utiliza la opción para especificar si también se cuenta como accesos a simples getters y setters.
