Informa sobre llamadas a equals() en instancias de StringBuilder, StringBuffer y el paquete java.util.concurrent.atomic.

El método equals() no está sobrescrito en estas clases, por lo que puede devolver false incluso cuando dos objetos tienen el mismo contenido. Si pretende usar igualdad por referencia, recomendamos usar == para evitar confusiones.

StringBuilder, StringBuffer, AtomicBoolean, AtomicInteger, AtomicBoolean y AtomicLong tienen disponible una corrección rápida para convertir a comparación de contenido. La corrección rápida podría cambiar la semántica cuando una de las instancias es null.

Ejemplo:


  public void test(StringBuilder sb1, StringBuilder sb2) {
    boolean result = sb1.equals(sb2); // sospechoso
  }

Después de aplicar la corrección rápida:


  public void test(StringBuilder sb1, StringBuilder sb2) {
      boolean result = sb1.toString().equals(sb2.toString());
  }
