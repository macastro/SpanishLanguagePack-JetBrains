Informa sobre el acceso a campos private o protected de otro objeto. Java permite el acceso a estos campos de objetos que pertenecen a la misma clase que el objeto actual, pero algunos estilos de codificación desaconsejan este uso. Además, este acceso directo a campos private puede fallar en arquitecturas orientadas a componentes (como Spring o Hibernate), que esperan que todo acceso a otros objetos se realice mediante llamadas a métodos, permitiendo que el framework medie el acceso mediante proxies.

Ejemplo:


  public class Base {
    protected int bar;
    void increment(Base base) {
      bar++;
      base.bar++; // warning: direct access to another object's non-public field
    }
  }
El campo puede ser encapsulado mediante una corrección rápida.

Configuración de la inspección:

  • Use la opción Ignorar accesos desde la misma clase para ignorar los accesos desde la misma clase y solo informar sobre accesos desde clases internas o externas.

    Para ignorar accesos desde clases internas, use la opción anidada Ignorar accesos desde clases internas.

  • Use Ignorar accesos desde el método 'equals()' para ignorar los accesos desde el método equals().
