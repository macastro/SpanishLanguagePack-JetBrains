Reporte de proveedores duplicados.

Ejemplo del problema:


provider "kubernetes" {
  config_path = "~/.kube/config"
}
provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
}

Una forma de resolver este problema es añadiendo un alias y especificándolo explícitamente al crear recursos en el proveedor deseado.

Ejemplo corregido:


  provider "kubernetes" {
  config_path = "~/.kube/config"
}
provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
  alias = "cloud"
}
Soporte técnico: Plugin de Terraform y HCL
