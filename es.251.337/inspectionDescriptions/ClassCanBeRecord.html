La clase puede convertirse en una clase record.

Las clases record se centran en modelar datos inmutables en lugar de comportamientos extensibles. La implementación implícita automática de los métodos impulsados por datos (como equals() y los accesores) ayuda a reducir el código repetitivo.

Tenga en cuenta que no todas las clases pueden ser clases record. Aquí hay algunas limitaciones:

  • La clase no puede tener subclases.
  • Todos los campos no static en la clase deben ser final.
  • No pueden existir inicializadores, constructores genéricos ni métodos nativos.

Para una descripción completa de las clases record, consulte la Especificación del Lenguaje Java.

Ejemplo:


  class Point {
    private final double x;
    private final double y;
    Point(double x, double y) {
      this.x = x;
      this.y = y;
    }
    double getX() {
      return x;
    }
    double getY() {
      return y;
    }
  }

Después de aplicar la corrección rápida:


  record Point(int x, int y) {
  }

Habilitar la opción Sugerir renombrar métodos accesores renombrará automáticamente los accesores getX()/isX() a x().

Use la opción Si los miembros se vuelven más accesibles para especificar qué hacer cuando la conversión haga que los miembros sean más accesibles:

  • Seleccione No sugerir conversión para no convertir cuando los miembros se vuelvan más accesibles.
  • Seleccione Mostrar vista de conflictos para mostrar los miembros afectados y solicitar continuar. En modo por lotes, no se sugerirá la conversión.
  • Seleccione Convertir silenciosamente para aumentar la accesibilidad silenciosamente cuando sea necesario.

Use la lista Prohibir la conversión de clases con las siguientes anotaciones para excluir clases cuyas anotaciones coincidan con los patrones especificados de las clases a convertir.
