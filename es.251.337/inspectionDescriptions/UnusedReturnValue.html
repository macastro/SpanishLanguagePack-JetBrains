Informa sobre métodos cuyo valor de retorno nunca se utiliza cuando son invocados. El tipo de retorno de estos métodos puede establecerse como void.

No se informarán los métodos anotados con @CanIgnoreReturnValue de Error Prone o AssertJ. La corrección rápida actualizará la firma del método y eliminará las declaraciones return del método.

Ejemplo:


  // Se informa cuando la visibilidad es Protected o Public
  protected String myToUpperCase(String s) {
    return s.toUpperCase();
  }
  // Simple setter, se informa según la configuración
  public String setStr(String str) {
    myStr = str;
    return myStr;
  }
  void test() {
    setStr("value");          // Valor de retorno no utilizado
    myToUpperCase("result");  // Valor de retorno no utilizado
  }

Después de aplicar la corrección rápida a ambos métodos:


    protected void myToUpperCase(String s) {
      // 'return' completamente eliminado
      // ya que 's.toUpperCase()' no tiene efectos secundarios
    }
    public void setStr(String str) {
      myStr = str;
      // 'return' eliminado
    }
  ...

Nota: Por razones de rendimiento, algunos métodos pueden no ser informados durante el resaltado en el editor. Para ver todos los resultados, ejecute la inspección a través de Código | Inspeccionar código o Código | Analizar código | Ejecutar inspección por nombre

Use la opción Ignorar métodos encadenables para ignorar los valores de retorno no utilizados en llamadas encadenables.

Use la opción Informar visibilidad máxima del método para controlar la visibilidad máxima de los métodos a informar.
