Informa de los métodos cuyos valores devueltos nunca se usan cuando son llamados. El tipo de retorno de dichos métodos puede configurarse como void.

No se informará de los métodos anotados con la anotación @CanIgnoreReturnValue de Error Prone o AssertJ. La corrección rápida actualiza la firma del método y elimina sentencias return del interior del método.

Ejemplo:


  // informado si la configuración de visibilidad es Protected o Public
  protected String myToUpperCase(String s) {
    return s.toUpperCase();
  }

  // simple setter, reporting depends on setting
  public String setStr(String str) {
    myStr = str;
    return myStr;
  }

  void test() {
    setStr("value");          // el valor devuelto no se usa
    myToUpperCase("result");  // el valor devuelto no se usa
  }

Tras aplicar la corrección rápida a ambos métodos:


    protected void myToUpperCase(String s) {
      // 'return' eliminado por completo
      // ya que 's.toUpperCase()' no tiene efectos secundarios
    }

    public void setStr(String str) {
      myStr = str;
      // 'return' eliminado
    }
  ...

NOTA: Puede que algunos métodos no se informen durante el resaltado en el editor por motivos de rendimiento. Para ver todos los resultados, ejecutar la inspección usando Código | Inspeccionar código o Código | Analizar código | Ejecutar inspección por nombre>

Utiliza la opción Ignorar métodos encadenables para ignorar los valores devueltos no usados de llamadas encadenables.

Utiliza la opción Visibilidad máxima de método informada para controlar la visibilidad máxima de los métodos a informar.
