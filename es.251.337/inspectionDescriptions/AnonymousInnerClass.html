Informar sobre clases anónimas.

Ocasionalmente, reemplazar las clases anónimas por clases internas puede mejorar la legibilidad y mantenibilidad del código. Algunos estándares de código no recomiendan el uso de clases anónimas.

Ejemplo:


  class Example {
    public static void main(String[] args) {
      new Thread() {
        public void run() {
          work()
        }
        private void work() {}
      }.start();
    }
  }

Después de aplicar la corrección rápida:


  class Example {
    public static void main(String[] args) {
      new MyThread().start();
    }
    private static class MyThread extends Thread {
      public void run() {
        work();
      }
      private void work() {}
    }
  }
