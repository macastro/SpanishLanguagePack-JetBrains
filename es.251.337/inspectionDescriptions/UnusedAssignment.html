Informa sobre asignaciones que no se utilizan después de realizarse. Es mejor eliminar estas asignaciones si no se utilizan para acortar el código y evitar asignaciones redundantes.

Se informará sobre los siguientes casos:

  • Variables que nunca se leen después de una asignación
  • Variables que siempre se sobrescriben con nuevos valores antes de ser leídas
  • Inicializadores de variables redundantes (por cualquiera de las dos razones anteriores)

Configuración de la inspección:

Utiliza la opción Informar inicializadores redundantes para detectar inicializadores redundantes: 


  int getI() {
    int i = 0; // Inicialización redundante
    i = 2;
    return i;
  }

Utiliza la opción Informar expresiones de prefijo que pueden ser reemplazadas por expresiones binarias para detectar casos donde una expresión ++i puede ser reemplazada por i + 1: 


  int preInc(int value) {
    int res = value;
    return ++res;
  }

Utiliza la opción Informar expresiones de sufijo donde el valor modificado no se utiliza para detectar casos de i++ donde el valor de i no se utiliza posteriormente: 


  int postInc(int value) {
    int res = value;
    return res++;
  }

Utiliza la opción Informar variables de patrón cuyo valor nunca se utiliza para detectar casos donde el valor de una variable de patrón se sobrescribe antes de ser leído: 


  if (object instanceof String s) {
      s = "hello";
      System.out.println(s);
  }

Utiliza la opción Informar parámetros de iteración cuyo valor nunca se utiliza para detectar casos donde el valor del parámetro de iteración en una sentencia for mejorada se sobrescribe antes de ser leído: 


  for (String arg : args) {
      arg = "test";
      System.out.println(arg);
  }
