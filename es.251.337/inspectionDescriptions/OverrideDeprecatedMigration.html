Reporta declaraciones propagadas por la anotación @Deprecated que causarán errores de compilación a partir de la versión 1.9.

Tipos de motivación:

  • Necesidad de cambios en la implementación por razones de diseño/arquitectura
  • Inconsistencias en el diseño (diferentes manejos en diferentes contextos)

Más información: KT-47902: No propagar la depreciación a través de métodos sobrescritos

La corrección rápida copia la anotación @Deprecated de la declaración padre.

Ejemplo:


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }
  class Derived : Base() {
      override fun foo() {}
  }

Después de aplicar la corrección rápida:


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }
  class Derived : Base() {
      @Deprecated("Don't use")
      override fun foo() {}
  }

Esta inspección solo reporta cuando el nivel del lenguaje Kotlin del proyecto o módulo es 1.6 o superior.
