Reportar atributos en conflicto en bloques HCL.

Ejemplo del problema:


provider "kubernetes" {
  version = "~> 1.25"
  config_path = "~/.kube/config"
  version = "~> 1.24"
}

En este ejemplo, el atributo version se declara dos veces en el bloque del proveedor Kubernetes. Esto es un error.

Ejemplo corregido:


provider "kubernetes" {
  version = "~> 1.25"
  config_path = "~/.kube/config"
}
