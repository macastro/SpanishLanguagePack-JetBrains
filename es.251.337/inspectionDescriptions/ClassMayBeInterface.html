Informa sobre clases abstract que pueden convertirse en interfaces.

Es preferible usar interfaces en lugar de clases, ya que Java no admite herencia múltiple de clases, pero una clase puede implementar múltiples interfaces.

Una clase puede convertirse en interfaz si no tiene superclase (excepto Object), solo tiene campos public static final, métodos public abstract y clases internas public.

Ejemplo:


abstract class Example {
    public static final int MY_CONST = 42;
    public abstract void foo();
}
class Inheritor extends Example {
    @Override
    public void foo() {
        System.out.println(MY_CONST);
    }
}

Después de aplicar la corrección rápida:


interface Example {
    int MY_CONST = 42;
    void foo();
}
class Inheritor implements Example {
    @Override
    public void foo() {
        System.out.println(MY_CONST);
    }
}

Configuración de la inspección:

La opción Informar clases con métodos no abstractos cuando se usa Java 8 está disponible para informar solo sobre clases cuyos métodos static y no abstractos pueden convertirse en métodos default (solo aplicable para nivel de lenguaje 8 o superior).
