Reportar clases vacías y archivos Java vacíos.

Una clase está vacía si no contiene campos, métodos, constructores o inicializadores. Las clases vacías a veces permanecen después de cambios importantes o refactorizaciones.

Ejemplo:


  class Example {
    List<String> getList() {
       return new ArrayList<>() {
       };
    }
  }

Después de aplicar la corrección rápida:


  class Example {
    List<String> getList() {
       return new ArrayList<>();
    }
  }

Configuración de la inspección:

  • Utilice la opción Ignorar cuando hay anotaciones para especificar anotaciones especiales. La inspección ignorará las clases marcadas con estas anotaciones.

  • Utilice la opción Ignorar si la clase es un parámetro del tipo padre para ignorar clases con superclases parametrizadas. Por ejemplo: 

    class MyList extends ArrayList<String> {}
  • Utilice Ignorar subclases de java.lang.Throwable para ignorar clases que extienden de java.lang.Throwable.
  • Utilice la opción Considerar comentarios como contenido para ignorar clases que contienen comentarios.
