Reporta métodos públicos que aceptan parámetros de tipo boolean.

Si un método público (parte de una API) no es un setter, casi siempre es una mala práctica añadir parámetros de tipo boolean. Al leer el código que utiliza estos métodos, es difícil entender el significado del boolean sin consultar el código fuente o la documentación.

Este problema también se conoce como trampa booleana. Los parámetros boolean generalmente pueden ser reemplazados por enum.

Ejemplo:


  // Advertencia: al ver las llamadas
  // a este método, es difícil entender
  // el significado de los parámetros booleanos
  public boolean setPermission(File f,
                               int access,
                               boolean enable,
                               boolean ownerOnly) {
    // ...
  }

Utilice la opción reportar solo métodos con múltiples parámetros booleanos para emitir advertencias únicamente cuando un método contenga varios parámetros booleanos.
