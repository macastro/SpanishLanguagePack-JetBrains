Los reportes pueden tener parámetros de tipo con varianza in o out.

El uso de varianza in y out permite una inferencia de tipos más precisa y una semántica de código más clara en Kotlin.

Ejemplo:


  class Box<T>(val obj: T)
  fun consumeString(box: Box<String>) {}
  fun consumeCharSequence(box: Box<CharSequence>) {}
  fun usage(box: Box<String>) {
      consumeString(box)
      consumeCharSequence(box) // Error de compilación
  }

La corrección rápida añadirá el modificador de varianza correspondiente:


  class Box<out T>(val obj: T)
  fun consumeString(box: Box<String>) {}
  fun consumeCharSequence(box: Box<CharSequence>) {}
  fun usage(box: Box<String>) ++{
      consumeString(box)
      consumeCharSequence(box) // Permitido
  }
