En Java 21 y versiones anteriores, se reportan los métodos nombrados como "main" que no tienen la firma public static void main(String[]). A partir de la versión preliminar de Java 21, la inspección no resalta los métodos main que son package-private, protected o de instancia (también sin parámetros). Además, se reportan los métodos main que se encuentran en clases anónimas o locales. Las clases anónimas y locales no tienen un nombre completamente calificado, por lo que no se pueden ejecutar.

Estos métodos pueden causar confusión, ya que un método llamado "main" debería ser el punto de entrada de la aplicación.

Ejemplo:


  class Main {
    void main(String[] args) {} //Advertencia aquí porque faltan los modificadores "public static"
  }

La corrección rápida para renombrar estos métodos solo está disponible en el editor.
