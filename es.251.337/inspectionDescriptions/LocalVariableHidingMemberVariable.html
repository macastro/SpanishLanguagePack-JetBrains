Informa sobre variables locales que tienen el mismo nombre que los campos de las clases circundantes. Este tipo de nomenclatura puede llevar a un uso incorrecto de la variable cuando se pretende usar el campo del mismo nombre.

Se recomienda renombrar la variable usando la corrección rápida.

Ejemplo:


  public class Foo {
    public Object foo;
    void bar() {
      Object o = new Object() {
        void baz() {
          Object foo; // La variable local 'foo' oculta el campo en la clase 'Foo'
        }
      };
    }
  }

Puede configurar las siguientes opciones para esta inspección:

  1. Ignorar campos inaccesibles - Ignora las variables locales que tienen el mismo nombre que los campos de superclase no visibles (por ejemplo, porque son campos private).
  2. Ignorar variables locales en contexto static que ocultan campos no static - Por ejemplo, cuando la variable local está dentro de un método static o dentro de un método en una clase interna static.
