Informa sobre llamadas a Math.random() que se convierten inmediatamente a int.

La conversión de un double entre 0.0 (inclusive) y 1.0 (exclusive) a int siempre se redondeará a cero. El valor debería multiplicarse por un factor antes de convertirlo a int para obtener un valor entre cero (inclusive) y el factor de multiplicación (exclusive). Otra posible solución es usar el método nextInt() de java.util.Random.

Ejemplo:

  int r = (int)Math.random() * 10;

Después de aplicar la corrección rápida:

  int r = (int)(Math.random() * 10);
