Informa sobre métodos y campos en clases Serializable y Externalizable que no son apropiados para ser anotados con la anotación java.io.Serial.

Ejemplo:


class Test implements Serializable {
  @Serial // El campo anotado no es parte del mecanismo de serialización porque no es final
  private static long serialVersionUID = 7874493593505141603L;
  @Serial // El método anotado no es parte del mecanismo de serialización porque no es private
  void writeObject(ObjectOutputStream out) throws IOException {
  }
}


class Test implements Externalizable {
  @Serial // El método anotado no es parte del mecanismo de serialización porque está dentro de una clase Externalizable
  private void writeObject(ObjectOutputStream out) throws IOException {
  }
}

Para información sobre todos los casos válidos, consulte el Javadoc de la clase java.io.Serial.
