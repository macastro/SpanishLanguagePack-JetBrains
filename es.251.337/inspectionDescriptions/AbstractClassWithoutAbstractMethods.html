Inspecciona clases abstract que no tienen métodos abstract. En la mayoría de los casos no tiene sentido tener una clase abstract sin ningún método abstract, y el modificador abstract puede ser removido de la clase. Si la clase fue declarada abstract para prevenir la instanciación, a menudo es una mejor opción usar un constructor private para prevenir la instanciación en su lugar.

Ejemplo:


  clase abstract Ejemplo {
    public String getNombre() {
      return "IntelliJ IDEA";
    }
  }

Use la opción para ignorar las clases utilitarias.
