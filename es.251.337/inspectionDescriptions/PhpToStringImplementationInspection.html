Reporta los intentos de convertir a string objetos que no tienen implementado el método __toString.

Antes de PHP 7.4, esto provocaba un error fatal de nivel E_RECOVERABLE_ERROR. En PHP 7.4, se lanza un error fatal irrecuperable ERROR. A partir de PHP 8.0, el valor devuelto sigue la semántica estándar de tipos de PHP y se convierte a cadena si es posible y el modo estricto de tipos está desactivado.

Si la opción Verificar la existencia de __toString para cada tipo de expresión está activada, la comprobación examinará todos los tipos posibles de la expresión y reportará si al menos uno de los tipos no incluye la implementación del método __toString.

Consulte __toString (php.net) para más detalles.
