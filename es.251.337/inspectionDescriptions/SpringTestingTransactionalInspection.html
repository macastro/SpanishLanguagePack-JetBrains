Reportar declaraciones de métodos de ciclo de vida transaccional inválidos con anotaciones @BeforeTransaction y @AfterTransaction en clases de prueba anotadas como @Transactional.

Los métodos anotados no deben tener argumentos ni tipo de retorno.

Ejemplo:


  @ContextConfiguration
  @Transactional
  public class AbstractShowcaseTest {
    @BeforeTransaction // El tipo de retorno de método esperado es 'void'
    public boolean setupData() {...}

    @AfterTransaction // Número de argumentos incorrecto
    public void disposeData(boolean a) throws Exception {...}
  }
