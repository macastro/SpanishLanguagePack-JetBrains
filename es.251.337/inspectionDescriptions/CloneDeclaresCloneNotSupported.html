Informa sobre métodos clone() que no declaran throws CloneNotSupportedException.

Sin la declaración throws CloneNotSupportedException, las subclases no podrán prohibir la clonación de manera estándar. Esta inspección no informa sobre métodos clone() declarados como final ni sobre métodos clone() en clases final.

Configuración de la inspección:

Use la opción Advertir solo para métodos clone 'protected' para que esta inspección emita advertencias únicamente para métodos protected clone(). Effective Java (tanto en su segunda como tercera edición) recomienda omitir la declaración CloneNotSupportedException en métodos public, ya que los métodos que no lanzan excepciones verificadas son más fáciles de usar.

Ejemplo:


  public class Example implements Cloneable {
    // El método no declara 'throws CloneNotSupportedException'
    protected Object clone() {
        try {
            return super.clone();
        } catch (CloneNotSupportedException e) {
            return null;
        }
    }
  }
