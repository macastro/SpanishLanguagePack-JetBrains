Reporta llamadas a java.lang.System.loadLibrary(), java.lang.System.load(), java.lang.Runtime.loadLibrary() y java.lang.Runtime.load() que utilizan cadenas construidas dinámicamente como nombres de biblioteca.

Los nombres de biblioteca construidos dinámicamente son una fuente común de vulnerabilidades de seguridad. Por defecto, esta inspección ignora las constantes en tiempo de compilación.

Ejemplo:


  void test(int i) {
    System.loadLibrary("foo" + i);
  }

La configuración de la inspección permite tratar cualquier campo static final como una constante. Tenga cuidado, ya que cuando esta opción está habilitada, se ignorarán cadenas como esta: 


  private static final String LIBRARY = getUserInput();
