A partir de Kotlin 1.4, se reportan las etiquetas redundantes que causan errores de compilación.

Desde Kotlin 1.0, es posible etiquetar cualquier declaración:


  fun foo() {
    L1@ val x = L2@bar()
  }
Sin embargo, estas etiquetas solo se pueden referenciar de algunas formas limitadas:
  • break/continue en bucles
  • retornos no locales desde lambdas inline o funciones anónimas inline

A partir de Kotlin 1.4, el uso de tales etiquetas está prohibido.

Esta comprobación solo se reporta cuando el nivel del lenguaje Kotlin del proyecto o módulo es 1.4 o superior.
