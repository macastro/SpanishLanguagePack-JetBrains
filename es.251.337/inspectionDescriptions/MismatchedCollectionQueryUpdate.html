Informa sobre colecciones cuyo contenido ha sido consultado pero no actualizado, o actualizado pero no consultado.

Estas inconsistencias entre consultas y actualizaciones carecen de sentido y pueden indicar código muerto o errores tipográficos.

Los ajustes de la inspección permiten especificar patrones de nombres correspondientes a métodos de actualización/consulta. Los métodos de consulta que devuelven elementos se detectan automáticamente, y solo es necesario especificar los métodos de consulta que escriben datos en parámetros de salida (por ejemplo, OutputStream).

Ejemplo:

Supongamos que tiene una clase personalizada FixedStack con un método store(): 


  public class FixedStack<T> extends Collection<T> {
    public T store(T t) {
      // implementación
    }
  }

Puede agregar store a la tabla de métodos de actualización para informar sobre consultas no coincidentes, por ejemplo: 


  void test(int i) {
    FixedStack<Integer> stack = new FixedStack<>();
    stack.store(i);
  }
