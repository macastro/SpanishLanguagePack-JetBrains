Reporta el uso de Optional.ofNullable() donde un argumento siempre nulo o siempre no nulo es pasado. No tiene sentido utilizar Optional.ofNullable() en este caso: Optional.empty() o Optional.of() debe ser usado para indicar explícitamente la intención de crear un Optional siempre vacío o siempre no vacío respectivamente. También es posible que haya un error en el argumento Optional.ofNullable(), por lo que se debe examinar.

Ejemplo: 


Optional<String> empty = Optional.ofNullable(null); // debería ser Optional.empty();
Optional<String> present = Optional.ofNullable("value"); // debería ser Optional.of("value");
