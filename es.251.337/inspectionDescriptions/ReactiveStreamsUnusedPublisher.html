Reporta instancias no utilizadas de Publisher.

Para usar operadores que generan nuevas instancias de Publisher (métodos que devuelven objetos Mono/Flux/Flowable de Mono/Flux/Flowable), debe suscribirse al Publisher creado mediante subscribe().

Usar una fábrica (por ejemplo, Mono.just()) sin suscribirse al Publisher devuelto crea un objeto que nunca se utiliza y se considera una asignación de memoria innecesaria.

Por ejemplo, Mono.just(1, 2, 3).map(i -> i + 3) no se ejecutará a menos que se suscriba a este Publisher, o a menos que genere un nuevo Publisher aplicando operadores y se suscriba a él.

Ejemplo:

Instancia de Flux no utilizada:


Flux.just(1, 2, 3);

Instancia de Flux utilizada por el consumidor:


Flux.just(1, 2, 3).subscribe(System.out::println);

No se reportarán las llamadas a métodos anotados con la anotación @CanIgnoreReturnValue cuando se use Error Prone o AssertJ.

Novedades de la versión 2019.3
