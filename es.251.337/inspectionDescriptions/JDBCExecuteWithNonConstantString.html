Informa sobre llamadas a java.sql.Statement.execute() o cualquiera de sus variantes que utilicen cadenas construidas dinámicamente como consultas a ejecutar.

Las sentencias SQL construidas dinámicamente son una fuente común de vulnerabilidades de seguridad. Por defecto, esta inspección ignora las constantes en tiempo de compilación.

Ejemplo:


  ResultSet execute(Statement statement, String name) throws SQLException {
    return statement.executeQuery("select * from " + name); // Genera advertencia
  }

Las opciones de inspección permiten tratar cualquier campo static final como una constante. Tenga cuidado, ya que cuando esta opción está habilitada, se ignorarán cadenas como esta: 


  private static final String SQL = "SELECT * FROM user WHERE name='" + getUserInput() + "'";
