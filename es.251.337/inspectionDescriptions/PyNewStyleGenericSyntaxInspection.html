Reporta uso inválido de la sintaxis de parámetros de tipo PEP 695

Encuentra los siguientes problemas en las definiciones de funciones y clases y en las declaraciones de alias de tipo de nuevo estilo:

  • Extender typing.Generic en clases genéricas de nuevo estilo
  • Extender typing.Protocol parametrizado en clases genéricas de nuevo estilo
  • Utilizar límites superiores genéricos y restricciones con parámetros de tipo para ParamSpec y TypeVarTuple
  • Mezclar variables de tipo tradicionales y de nuevo estilo
  • Utilizar variables de tipo tradicionales en alias de tipo de nuevo estilo

Ejemplos: 


  from typing import Generic

  class Ejemplo[T](Generic[T]): ... # Las clases con lista de parámetros de tipo no deben extender 'Generic'


  class Ejemplo[T: (list[S], str)]: ... # Los tipos genéricos no están permitidos dentro de restricciones y límites de parámetros de tipo


  from typing import TypeVar

  K = TypeVar("K")

  class ClassC[V]:
      def method2[M](self, a: M, b: K) -> M | K: ... # No está permitido mezclar TypeVars tradicionales y de nuevo estilo
