Informa sobre declaraciones synchronized con cuerpo vacío.

Las declaraciones synchronized vacías a veces se utilizan para esperar a que otros hilos liberen un recurso específico. Sin embargo, no hay garantía de que el mismo recurso no se vuelva a adquirir después de que se complete la declaración synchronized vacía. Para una sincronización correcta, el recurso debe utilizarse dentro del bloque synchronized.

Además, los bloques synchronized vacíos pueden aparecer después de una refactorización cuando se elimina código redundante. En este caso, el propio bloque synchronized es redundante y también debe eliminarse.

Ejemplo:


  synchronized(lock) {}

Se recomienda eliminar las declaraciones synchronized vacías mediante la corrección rápida.

Esta inspección está desactivada para archivos JSP.
