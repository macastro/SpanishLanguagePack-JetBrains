Reporta calificadores this innecesarios.

El uso de this para desambiguar referencias en el código no es recomendado en muchos estilos de codificación y puede volverse innecesario fácilmente mediante refactorización automática.

Ejemplo:


  class Foo {
    int x;
    void foo() {
      this.x = 2;
    }
  }

Después de aplicar la corrección rápida:


  class Foo {
    int x;
    void foo() {
      x = 2;
    }
  }

Las asignaciones a campos pueden ignorarse usando la configuración de la inspección. Por ejemplo, this.x = 2; no será reportado, pero int y = this.x; sí lo será.
