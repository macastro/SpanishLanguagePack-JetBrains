Reporta métodos que sobreescriben métodos de la superclase pero no están anotados con @java.lang.Override.

Anotar métodos con @java.lang.Override mejora la legibilidad del código ya que muestra la intención. Además, el compilador emite un error cuando la firma del método sobreescrito no coincide con el método de la superclase.

Ejemplo:


    class X {
      public String toString() {
        return "hello world";
      }
    }

Después de aplicar la corrección rápida:


    class X {
      @Override
      public String toString() {
        return "hello world";
      }
    }

Configurar la inspección:

  • Utilice la opción Ignorar 'equals()', 'hashCode()' y 'toString()' para ignorar estos métodos de java.lang.Object: equals(), hashCode() y toString(). El riesgo de que estos métodos desaparezcan y su código ya no se compile debido a la anotación @Override es relativamente pequeño.
  • Utilice la opción Ignorar métodos en clases anónimas para ignorar métodos en clases anónimas.
  • Deshabilite la opción Resaltar método cuando sus métodos de sobreescritura no tienen la anotación '@Override' para advertir solo sobre los métodos que faltan una anotación @Override, y no en los métodos sobreescritos donde a uno o más descendientes les falta una anotación @Override.
