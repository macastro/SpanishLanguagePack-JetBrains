Informa sobre métodos que sobrescriben métodos de la superclase pero no utilizan la anotación @java.lang.Override.

El uso de la anotación @java.lang.Override mejora la legibilidad del código al indicar la intención. Además, el compilador mostrará un error si la firma del método sobreescrito no coincide con el método de la superclase.

Ejemplo:


    class X {
      public String toString() {
        return "hello world";
      }
    }

Después de aplicar la corrección rápida:


    class X {
      @Override
      public String toString() {
        return "hello world";
      }
    }

Configuración de la inspección:

  • La opción Ignorar 'equals()', 'hashCode()' y 'toString()' permite ignorar estos métodos de java.lang.Object: equals(), hashCode() y toString(). El riesgo de que estos métodos desaparezcan y el código deje de compilar debido a la anotación @Override es relativamente bajo.
  • La opción Ignorar métodos en clases anónimas permite ignorar los métodos en clases anónimas.
  • Deshabilitar la opción Resaltar método cuando no todos sus métodos de sobreescritura tienen la anotación '@Override' generará advertencias solo para los métodos que carecen de la anotación @Override, en lugar de advertir sobre métodos sobreescritos que carecen de la anotación @Override en uno o más de sus descendientes.
