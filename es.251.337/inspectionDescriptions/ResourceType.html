Esta inspección examina las llamadas a la API de Android que utilizan varias anotaciones de soporte (como RequiresPermission o UiThread) y marca cualquier llamada que no use la API correctamente según lo especificado en las anotaciones.

Ejemplos de errores marcados por esta inspección:
  • Pasar un entero de recurso del tipo incorrecto (como R.string) a una API que requiere un tipo diferente (como R.dimen).
  • Olvidar llamar a un método sobrescrito (mediante super) en métodos que lo requieren
  • Llamar a métodos que requieren permisos que no están declarados en el manifiesto
  • Pasar una referencia de color de recurso a un método que requiere un valor entero RGB.
...y más.

Para más información, consulte la documentación en http://developer.android.com/tools/debugging/annotations.html
