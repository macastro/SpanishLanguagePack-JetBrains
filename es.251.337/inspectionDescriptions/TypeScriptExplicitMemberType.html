Informa sobre anotaciones de tipo que no coinciden con el estilo de código actual para tipos explícitos.

Las declaraciones de tipo no son necesarias cuando el tipo inferido del contexto coincide exactamente con la anotación de tipo, por ejemplo: 


    var pi: number = 3.14

En algunos casos, es preferible mantener siempre tipos explícitos, ya que esto previene cambios accidentales de tipo y hace que el código sea más claro.
