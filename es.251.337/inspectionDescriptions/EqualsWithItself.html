Informa sobre llamadas a equals(), compareTo() o métodos similares que comparan un objeto consigo mismo. Las convenciones de estos métodos especifican que tales llamadas siempre devolverán true para equals() o 0 para compareTo(). La inspección también verifica llamadas a Objects.equals(), Objects.deepEquals(), Arrays.equals(), Comparator.compare(), métodos assertEquals() de frameworks de prueba (JUnit, TestNG, AssertJ), Integer.compare(), Integer.compareUnsigned() y métodos similares.

Tenga en cuenta que, en casos muy raros, la inspección podría informar sobre llamadas a equals() que devuelven false, ya que aunque las expresiones en ambos lados son idénticas, generan objetos separados y la comparación se realiza por referencia en lugar de por contenido. El ejemplo más simple es new Object().equals(new Object()). En cualquier caso, tales llamadas son sospechosas y podrían tener otras intenciones.

Ejemplo:


  class Foo {
    boolean foo(Object o) {
        return o.equals(o); // advertencia
    }
    boolean bar(String[] ss) {
        return Arrays.equals(ss, ss); // advertencia
    }
}
