Informa sobre clases que implementan java.lang.Comparator pero no implementan java.io.Serializable.

Si se utiliza un comparador no serializable para construir colecciones ordenadas como java.util.TreeMap o java.util.TreeSet, la colección tampoco será serializable. Esto puede provocar errores inesperados y difíciles de diagnosticar.

Las subclases de java.lang.Comparator generalmente no tienen estado, por lo que estos problemas se pueden evitar fácilmente simplemente marcándolas como serializables.

Ejemplo:


  class Foo implements Comparator { // advertencia
      @Override
      public int compare(Object o1, Object o2) {
          /* ... */
      }
  }

Después de aplicar la corrección rápida:


  class Foo implements Comparator, Serializable { // sin advertencia
      @Override
      public int compare(Object o1, Object o2) {
          /* ... */
      }
  }
