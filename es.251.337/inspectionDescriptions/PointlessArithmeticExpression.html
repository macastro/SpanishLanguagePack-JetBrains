Informa sobre expresiones aritméticas sin sentido. Estas expresiones incluyen sumar 0, restar 0, multiplicar por 0 o 1, y dividir por 1.

Estas expresiones pueden ser el resultado de refactorizaciones automáticas y es poco probable que sean la intención del desarrollador.

La corrección rápida simplifica estas expresiones.

Ejemplo: 


  void f(int a) {
    int x = a - a;
    int y = a + 0;
    int res = x / x;
  }

Después de aplicar la corrección rápida:


  void f(int a) {
    int x = 0;
    int y = a;
    int res = 1;
  }

Tenga en cuenta que, en casos muy raros, la sustitución sugerida puede no ser completamente equivalente al código original para todas las entradas posibles. Por ejemplo, la inspección sugiere reemplazar x / x por 1. Sin embargo, si x es cero, el código original lanzará una ArithmeticException o resultará en NaN. Además, si x es NaN, el resultado también será NaN. Es poco probable que estos comportamientos sean intencionales.
