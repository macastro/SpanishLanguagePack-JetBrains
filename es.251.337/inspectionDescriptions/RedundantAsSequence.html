Informa sobre llamadas redundantes a asSequence() que nunca producirán un impacto positivo en el rendimiento.

asSequence() acelera el procesamiento de colecciones con múltiples operaciones, ya que ejecuta las operaciones de forma diferida y no crea colecciones intermedias.

Sin embargo, si se utiliza una operación terminal (como toList()) inmediatamente después de asSequence(), no se obtiene ningún beneficio de rendimiento.

Ejemplo:


  fun test(list: List<String>) {
      list.asSequence().last()
  }

Después de aplicar la corrección rápida:


  fun test(list: List<String>) {
      list.last()
  }
