Un protector de informe es una anotación @GuardedBy en un campo de instancia o método que utiliza un campo static como protector. Usar un protector static para elementos no static puede resultar en una contención excesiva de bloqueos, ya que el acceso a cualquier campo bloqueado en cualquier instancia de objeto impedirá el acceso simultáneo a los campos en todas las instancias de objetos.

Ejemplo:


    private static ReadWriteLock lock = new ReentrantReadWriteLock(); //campo protegido static
    private Object state;
    @GuardedBy("lock")
    public void bar() {
        state = new Object();
    }

Las anotaciones @GuardedBy soportadas son:

  • net.jcip.annotations.GuardedBy
  • javax.annotation.concurrent.GuardedBy
  • org.apache.http.annotation.GuardedBy
  • com.android.annotations.concurrency.GuardedBy
  • androidx.annotation.GuardedBy
  • com.google.errorprone.annotations.concurrent.GuardedBy
