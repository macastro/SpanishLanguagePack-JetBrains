Informa sobre campos en clases derivadas que tienen el mismo nombre que los campos en la superclase. Los campos de Java no se pueden sobrescribir en clases derivadas, por lo que los campos en la clase derivada ocultan los campos de la superclase.

Debido a este tipo de nomenclatura, puede usar incorrectamente el campo de la clase derivada en situaciones donde se pretendía usar el campo de la clase base con el mismo nombre.

Se recomienda renombrar el campo en la clase derivada mediante la corrección rápida.

Ejemplo:

class Parent {
  Parent parent;
}
class Child extends Parent {
  Child parent;
}

Puede configurar las siguientes opciones para esta inspección:

  1. Ignorar campos inaccesibles - Indica si la inspección debe informar sobre todos los conflictos de nombres o solo sobre conflictos con campos visibles para la subclase.
  2. Ignorar campos static que ocultan campos static - Ignora los campos static que ocultan campos static en la clase base.
