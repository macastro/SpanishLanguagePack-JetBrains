Informa sobre instancias de Logger inicializadas con literales de clase en lugar de usar la clase residente del Logger.

Esto puede ocurrir cuando se copia y pega código desde otra clase. Puede resultar en eventos registrados bajo categorías incorrectas y filtrado erróneo.

Las opciones de inspección permiten especificar las clases y métodos de fábrica del logger que esta inspección debe identificar.

Ejemplo:


class AnotherService
class MyService {
    private val logger = LoggerFactory.getLogger(AnotherService::class.java)
}

Después de aplicar la corrección rápida:


class MyService {
    private val logger = LoggerFactory.getLogger(MyService::class.java)
}
