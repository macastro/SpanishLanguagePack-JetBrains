A partir de Kotlin 1.4, se reportan los objetivos de anotación sin sentido en las superclases.

Los objetivos de anotación como @get: no tienen sentido en las superclases y están prohibidos.

Ejemplo:


  interface Foo
  annotation class Ann
  class E : @field:Ann @get:Ann @set:Ann @setparam:Ann Foo

Después de aplicar la corrección rápida:


  interface Foo
  annotation class Ann
  class E : Foo

Esta inspección solo reporta cuando el nivel del lenguaje Kotlin del proyecto o módulo es 1.4 o superior.
