Informe sobre extensiones que son instanciadas por la plataforma IntelliJ pero están declaradas como objetos Kotlin.

El ciclo de vida de las extensiones es gestionado por la plataforma IntelliJ. El uso de objetos Kotlin para el registro de extensiones puede resultar en la creación de instancias innecesarias de extensiones y hacer que el plugin no se pueda desinstalar.

Ejemplo

Registro de extensión:


<annotator language="myLang" implementationClass="com.example.MyAnnotator"/>

Implementación de extensión:


// Incorrecto:
object MyAnnotator : Annotator {
  ...
}
// Correcto:
class MyAnnotator : Annotator {
  ...
}
