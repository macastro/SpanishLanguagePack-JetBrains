Informa sobre casos donde se pasan cadenas no seguras como consultas SQL a métodos en Java y Kotlin. Esto podría ser una causa de inyección SQL. La lista de métodos se toma de Configuración - Inyección de lenguaje para SQL, JPA QL, Hibernate QL y PostgreSQL

Los objetos seguros son:

  • Literales de cadena, instancias de interfaz u objetos enum, int y sus wrappers, boolean y sus wrappers, objetos de clase
  • Resultados de llamadas a métodos donde el receptor y los argumentos son seguros
  • Campos private en el mismo archivo, asignados solo con literales de cadena y con inicializadores seguros
  • Campos final en el mismo archivo con inicializadores seguros
  • Variables locales asignadas desde objetos seguros
    • Este campo, variable local o parámetro no debe pasarse como argumento a métodos ni usarse como calificador, o debe ser un tipo primitivo, su wrapper u objeto inmutable.
Los campos static final se consideran seguros.

Este análisis se realiza solo dentro de un archivo. Ejemplo: 


    public void save(String sql) {
        JdbcTemplate jdbcTemplate = new JdbcTemplate();
        jdbcTemplate.queryForList(sql);
    }
