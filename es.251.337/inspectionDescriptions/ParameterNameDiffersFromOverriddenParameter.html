Reporta parámetros que tienen nombres diferentes en los métodos sobrescritos o sobrecargados correspondientes. Aunque es legal en Java, estos nombres inconsistentes pueden resultar confusos y reducir los beneficios de la documentación con buenas prácticas de nomenclatura.

Ejemplo:


  class Person {
    Person(String fullName) {}
  }
  class Child extends Person {
    Child(String name) { super(name); }
  }

Después de aplicar la corrección rápida:


  class Person {
    Person(String fullName) {}
  }
  class Child extends Person {
    Child(String fullName) { super(fullName); }
  }

Las opciones permiten especificar si se deben ignorar los nombres de parámetros que tienen solo un carácter de longitud o que provienen de métodos de biblioteca. Ambos son útiles si no desea estar limitado por convenciones de nomenclatura cuestionables utilizadas en las bibliotecas.
