Informa sobre problemas causados por el uso incorrecto de valores nil.

El IDE analizará el flujo de datos para determinar si las variables pueden tener valores nil o no nil. El IDE informará sobre problemas potenciales en el código basándose en estos resultados. Considere las siguientes situaciones que pueden llevar a consecuencias inesperadas:

  • Las llamadas a métodos con un receptor nil pueden causar 'desreferencia de puntero nil'.
  • El índice de un slice nil puede causar un panic.
  • Las comparaciones como v == nil pueden ser sin sentido si se sabe que v es siempre nil o nunca nil.
  • No se verifica si una variable con su error correspondiente es nil. Un error corresponde a una variable cuando ambos se definen o asignan juntos en una declaración como v, err := foo().
