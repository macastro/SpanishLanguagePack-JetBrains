Reporta casos de prueba que tienen lógica de inicialización en sus constructores. Si el constructor falla, no se llamarán los métodos con la anotación @After ni el método tearDown(). Esto puede resultar en un entorno de pruebas parcialmente inicializado, afectando negativamente a otras pruebas. En su lugar, la inicialización del caso de prueba debe realizarse en el método setUp() o en métodos con la anotación @Before.

Ejemplo incorrecto:


  public class ImportantTest {
    private File file;
    public ImportantTest() throws IOException {
      file = File.createTempFile("xyz", ".tmp");
    }
    // ... tests go here
  }
