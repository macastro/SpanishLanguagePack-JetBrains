Informa llamadas de registro SLF4J, Log4j2 en una clase, tales como logger.info("message: {}", key) con mensajes de registro similares. Estas llamadas pueden no ser distinguibles entre sí, y esto introduce dificultades para entender de dónde proviene un determinado mensaje de registro.

Ejemplo (para Java):


  private static void request1(String text) {
      log.info("Message: {}", text); //llamada similar
      doSomething1();
  }

  private static void request2(int i) {
      log.info("Message: {}", i); //llamada similar
      doSomething2();
  }
  • Utiliza la opción Longitud mínima de una secuencia similar para establecer la longitud mínima de secuencias similares después de la cual se informarán las llamadas
  • Utiliza la opción No informar llamadas con el nivel de registro 'error' para ignorar mensajes con nivel de registro `error` y cuando hay una excepción. Puede ser útil ocultar las advertencias, ya que las ubicaciones de llamada aún pueden localizarse mediante seguimientos de pila
