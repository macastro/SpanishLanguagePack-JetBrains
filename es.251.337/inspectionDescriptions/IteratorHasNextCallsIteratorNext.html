Informa sobre implementaciones de Iterator.hasNext() o ListIterator.hasPrevious() que llaman a Iterator.next() o ListIterator.previous() en la instancia del iterador. Tales llamadas son casi seguramente erróneas, ya que métodos como hasNext() no deberían modificar el estado del iterador, mientras que next() sí debería hacerlo.

Ejemplo:


  class MyIterator implements Iterator<Integer> {
    public boolean hasNext() {
      return next() != null;
    }
  }
