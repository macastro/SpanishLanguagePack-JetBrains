Informa sobre calificadores super innecesarios en llamadas a métodos y referencias a campos.

El calificador super no es necesario cuando el campo o método de la superclase no está oculto/sobrescrito en la clase que realiza la llamada.

Ejemplo:


  class Foo {
    void foo() {}
  }
  class Bar extends Foo {
    void bar() {
      super.foo();
    }
  }

Después de aplicar la corrección rápida:


  class Foo {
    void foo() {}
  }
  class Bar extends Foo {
    void bar() {
      foo();
    }
  }

La configuración de la inspección permite ignorar los calificadores que ayudan a distinguir el acceso a miembros de la superclase de miembros con el mismo nombre en la clase externa.

  • Java | Visibilidad | El acceso a campos heredados es similar al acceso a elementos en el código circundante
  • Java | Visibilidad | Las llamadas a métodos heredados son similares a las llamadas a métodos locales
