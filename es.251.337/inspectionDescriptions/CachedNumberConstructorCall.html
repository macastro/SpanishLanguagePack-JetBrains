Informa sobre la instanciación de nuevos objetos Long, Integer, Short o Byte con argumentos primitivos long, integer, short o byte.

Se recomienda usar el método estático valueOf() introducido en Java 5, que por defecto almacena en caché objetos con valores entre -128 y 127 (inclusive).

Ejemplo:


  Integer i = new Integer(1);
  Long l = new Long(1L);

Después de aplicar la corrección rápida, el código se convierte en:


  Integer i = Integer.valueOf(1);
  Long l = Long.valueOf(1L);

Esta inspección solo informa cuando el nivel de lenguaje del proyecto o módulo es 5 o superior.

Use la opción Ignorar nuevas expresiones numéricas con argumentos String para ignorar las llamadas a constructores numéricos que contengan argumentos String.

Use la opción Informar solo cuando el constructor está @Deprecated para informar únicamente sobre llamadas a constructores obsoletos. Los constructores de Long, Integer, Short y Byte están obsoletos desde JDK 9.
