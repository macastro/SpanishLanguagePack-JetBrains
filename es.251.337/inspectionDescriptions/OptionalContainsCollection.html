Reporta el uso de tipos java.util.Optional o com.google.common.base.Optional que contienen parámetros de tipo array o colección.

En estos casos, es más claro usar un array o colección vacía para representar la ausencia de resultados.

Ejemplo:


  Optional<List<Integer>> foo() {
    return Optional.empty();
  }

Este código podría escribirse como:


  List<Integer> foo() {
    return List.of();
  }
