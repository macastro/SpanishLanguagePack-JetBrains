Informa sobre puntos y coma (;) redundantes que pueden eliminarse de forma segura.

Kotlin no requiere puntos y coma al final de cada declaración o expresión. Se recomienda usar esta corrección rápida para eliminar los puntos y coma redundantes.

Ejemplo:


  val myMap = mapOf("one" to 1, "two" to 2);
  myMap.forEach { (key, value) ->  print("$key -> $value")};

Después de aplicar la corrección rápida:


  val myMap = mapOf("one" to 1, "two" to 2)
  myMap.forEach { (key, value) ->  print("$key -> $value")}

Hay dos casos donde se requiere el punto y coma:

  1. Múltiples declaraciones en una misma línea deben separarse con punto y coma: 
    
  map.forEach { val (key, value) = it; println("$key -> $value") }
  2. Las clases enum que declaran propiedades o funciones requieren un punto y coma después de la lista de constantes enum: 
    
  enum class Mode {
      SILENT, VERBOSE;
      fun isSilent(): Boolean = this == SILENT
  }
