Reporta anotaciones innecesarias de opt-in que pueden eliminarse de forma segura.

La anotación @OptIn es requerida para código que utiliza APIs experimentales que pueden cambiar en cualquier momento en el futuro. Esta anotación pasa a ser inútil y posiblemente engañosa si no se utiliza ninguna API de este tipo (por ejemplo, cuando la API experimental se vuelve estable y ya no requiere opt-in para su uso).

El quick-fix Remover anotación puede usarse para eliminar la anotación @OptIn innecesaria.

Ejemplo:


  @OptIn(ExperimentalApi::class)
  fun foo(x: Bar) {
      x.baz()
  }

Después de que se aplica el quick-fix:


  fun foo(x: Bar) {
      x.baz()
  }
