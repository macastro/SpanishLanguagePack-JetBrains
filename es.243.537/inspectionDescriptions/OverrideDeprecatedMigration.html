Informa sobre declaraciones propagadas mediante la anotación @Deprecated que provocarán un error de compilación desde la versión 1.9.

Tipos de motivación:

  • Se requieren cambios de implementación por razones de diseño o arquitectura de la implementación
  • Incoherencia en el diseño (las cosas se hacen de manera diferente en diferentes contextos)

Más detalles: KT-47902: No propagar la deprecación de métodos a través de las anulaciones

La corrección rápida copia la anotación @Deprecated de la declaración principal.

Ejemplo:


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }

  class Derived : Base() {
      override fun foo() {}
  }

Después de aplicar la corrección rápida:


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }

  class Derived : Base() {
      @Deprecated("Don't use")
      override fun foo() {}
  }

Esta inspección solo informa si el nivel de lenguaje Kotlin del proyecto o módulo es 1.6 o superior.
