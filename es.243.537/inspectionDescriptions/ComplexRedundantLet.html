Informa una llamada let redundante basada en argumento.

let es redundante cuando el parámetro lambda solo se utiliza como calificador en una expresión de llamada.

Si necesita dar un nombre a la expresión calificadora, declare una variable local.

Ejemplo:


  fun splitNumbers() {
      "1,2,3".let { it.split(',') }
  }

La solución rápida elimina la llamada let() adicional:


  fun example() {
      "1,2,3".split(',')
  }

Alternativa:


  fun splitNumbers() {
      val numbers = "1,2,3"
      numbers.split(',')
  }
