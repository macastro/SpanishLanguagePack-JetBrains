Informa de llamadas a métodos con nombres similares a mutadores sin cláusulas de argumento.

Scala permite llamar a métodos Java sin parámetros sin cláusulas de argumento. Esto se debe principalmente a que el código de Scala utilice métodos de acceso sin parámetros de Java de la misma manera que los métodos de acceso de Scala. Es decir, sin ninguna cláusula de argumento y por lo tanto se adhiere al principio de acceso uniforme , que indica que el código del cliente no debe verse afectado por la decisión de implementar un atributo como campo o método.

Sin embargo, para los métodos con efectos secundarios, es práctica habitual utilizar una cláusula de argumento vacía.

La corrección rápida añade una cláusula de argumento vacía.

Ejemplo:


  val stringBuilder = new java.util.ArrayList[String]
  stringBuilder.clear

Después de aplicar la corrección rápida:


  val stringBuilder = new java.util.ArrayList[String]
  stringBuilder.clear()
