Reporta llamadas de registro de SLF4J o Log4j 2, como logger.info("{}: {}", key), donde el número de marcadores de posición {} en el mensaje del registrador no coincide con el número de argumentos reales en la llamada de registro.

Puedes especificar qué implementación utiliza SLF4J con la opción de inspección. Si se selecciona Comprobar automáticamente, se busca org.apache.logging.slf4j.Log4jLogger en la ruta de clase. Si se encuentra este archivo, o se selecciona Sí, no se informarán los casos en los que el último argumento con un tipo de excepción tenga un marcador de posición en la API SLFJ4. Ejemplo: 


// No se informará
log.error("For id {}: {}", id, new RuntimeException());
Esta opción solo se aplica a SLF4J.
