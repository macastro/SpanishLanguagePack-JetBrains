Informa de las llamadas a los métodos accesores de Java con cláusulas de argumento vacías.

Se espera que los métodos que siguen el contrato de nombres JavaBean para accesores no tengan efectos secundarios. La convención recomendada es utilizar un método sin parámetros siempre que no haya parámetros y el método no tenga efectos secundarios. Esta convención promueve el principio de acceso uniforme, que dice que el código cliente no debe verse afectado por la decisión de implementar un atributo como campo o método.

El problema es que Java no implementa el principio de acceso uniforme. Para salvar esa brecha, Scala le permite omitir los paréntesis vacíos en una invocación de un método Java que no toma argumentos.

El arreglo rápido elimina la cláusula de argumento vacía.

Ejemplo:


  "test".getClass()

Después de aplicar el arreglo rápido:


  "test".getClass
