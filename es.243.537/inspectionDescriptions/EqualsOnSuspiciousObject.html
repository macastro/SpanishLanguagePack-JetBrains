Reporta llamadas equals() en StringBuilder, StringBuffer e instancias del paquete java.util.concurrent.atomic.

El método equals() no está sobrescrito en estas clases, por lo que puede devolver false incluso cuando el contenido de los dos objetos es el mismo. Si se pretende la igualdad de referencia, es mejor utilizar == para evitar confusión.

Una corrección rápida para StringBuilder, StringBuffer, AtomicBoolean, AtomicInteger, AtomicBoolean y AtomicLong está disponible para transformar en una comparación de contenidos. La corrección rápida puede cambiar la semántica cuando una de las instancias es nula.

Ejemplo:


  public void test(StringBuilder sb1, StringBuilder sb2) {
    boolean result = sb1.equals(sb2); // Sospechoso
  }

Una vez aplicada la corrección rápida:


  public void test(StringBuilder sb1, StringBuilder sb2) {
      boolean result = sb1.toString().equals(sb2.toString());
  }
