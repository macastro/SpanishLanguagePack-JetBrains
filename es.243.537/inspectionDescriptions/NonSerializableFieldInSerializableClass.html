Inspecciona los campos no serializables en las clases que implementan java.io.Serializable. Estos campos generarán una excepción de tiempo de ejecución si se serializa el objeto.

Los campos declarados como transient o static no se muestran. Tampoco se muestran los campos de las clases que tienen definido un método writeObject.

Esta inspección asume que los campos de los tipos java.util.Collection y java.util.Map son Serializable, a menos que los tipos en los que se declaren sean no Serializable.

Ejemplo:


  class NonSerializableClass {}

  public class SerializableClass implements Serializable {
      NonSerializableClass clazz; // advertencia: Campo no serializable 'clazz' en una clase serializable
      static NonSerializableClass staticClazz; // sin advertencias
  }

Utiliza las siguientes opciones para configurar la inspección:

  • Lista las clases cuyos herederos no se deben mostrar con esta inspección. Esto está hecho para clases que heredan Serializable de una superclase pero no están destinadas a la serialización.
  • Lista las anotaciones que harán que la inspección ignore los campos anotados.
  • Si se deben ignorar los campos inicializados con una clase anónima.
