Inspecciona los siguientes errores dentro de consultas Persistence QL:
  • Tipos de expresiones que no coinciden
  • Parámetros incorrectos
  • Condiciones vacías o constantes
  • Símbolos sin resolver

Consultas de ejemplo:


SELECT OBJECT(e) FROM JavaEntity e WHERE TRUE // Aviso: condición constante

SELECT OBJECT(e) FROM JavaEntity e WHERE e.id BETWEEN 0 AND '2' // Error: se esperaba un valor numérico pero se encontró una cadena

SELECT e from JavaEntity e where e.id=?1 and e.name=?3 // Error: solo hay 2 parámetros de consulta reales
