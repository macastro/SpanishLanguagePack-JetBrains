Informa de expresiones lógicas ambiguas en ramas when que provocan errores de compilación en Kotlin 1.8 y posteriores.


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      this in (4..7) -> true // es ambiguo
      else -> false
  }

Después de aplicar la corrección rápida:


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      (this in (4..7)) -> true // envuelto en paréntesis
      else -> false
  }

La inspección está disponible para el nivel de lenguaje Kotlin a partir de 1.7.
