Inspecciona las llamadas a métodos en EL que no coinciden con las restricciones de method-signature en los archivos TLD.

Clase Java de ejemplo:



public class ExampleAction {
  public void goodAction() { ... }
  public void badAction(String str) { ... }
}

Archivo JSP de ejemplo:



<jsp:useBean id="example" scope="request" type="org.example.ExampleAction"/>
<f:view>
  <h:commandButton action="#{example.goodAction}"/> // Bueno: el método acepta cero argumentos y devuelve `void`
  <h:commandButton action="#{example.badAction}"/> // Error: el método `void badAction(String)` devuelve el tipo `java.lang.String` en lugar de `void`
</f:view>
