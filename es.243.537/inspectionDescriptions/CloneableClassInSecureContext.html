Informa clases que pueden ser clonadas.

Una clase puede ser clonada si soporta la interface Cloneable, y su método clone() no está definido para lanzar un error inmediatamente. Las clases clonables pueden ser peligrosas en códigos diseñados para uso seguro.

Ejemplo:

class SecureBean implements Cloneable {}

Después de aplicar la corrección rápida:

class SecureBean {}

Cuando la clase extiende una clase clonable existente o implementa una interfaz clonable, entonces después de aplicar la corrección rápida, el código puede verse así: 

class SecureBean extends ParentBean {
    @Override
    protected SecureBean clone() throws CloneNotSupportedException {
        throw new CloneNotSupportedException();
    }
}
