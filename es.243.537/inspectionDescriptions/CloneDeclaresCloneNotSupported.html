Informa sobre métodos clone() que no declaran throws CloneNotSupportedException.

Si no se declara throws CloneNotSupportedException, las subclases del método no podrán prohibir la clonación de la forma estándar. Esta inspección no informa sobre los métodos clone() declarados final y los métodos clone() en clases final.

Configurar la inspección:

Utiliza la opción Solo avisar en métodos clon 'protegidos' para indicar que esta inspección solo debe avisar en métodos clone() protegidos. El libro Effective Java (segunda y tercera edición) recomienda omitir la declaración CloneNotSupportedException en métodos public, porque los métodos que no lanzan excepciones comprobadas son más fáciles de usar.

Ejemplo:


  public class Example implements Cloneable {
    // el método no declara 'throws CloneNotSupportedException'
    protected Object clone() {
        try {
            return super.clone();
        } catch (CloneNotSupportedException e) {
            return null;
        }
    }
  }
