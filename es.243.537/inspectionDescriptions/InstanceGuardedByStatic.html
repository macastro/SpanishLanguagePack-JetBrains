Informa de anotaciones `@GuardedBy` en campos o métodos de instancia en los que el protector es un campo `static`. Proteger un no estático con un static puede dar lugar a un exceso de contienda de bloqueo, ya que el acceso a cada campo bloqueado en cualquier instancia de objeto impedirá el acceso simultáneo a ese campo en cada instancia de objeto.

Ejemplo:


    private static ReadWriteLock lock = new ReentrantReadWriteLock(); //campo de protección estático
    private Object state;

    @GuardedBy("lock")
    public void bar() {
        state = new Object();
    }

Las anotaciones `@GuardedBy` compatibles son:

  • net.jcip.annotations.GuardedBy
  • javax.annotation.concurrent.GuardedBy
  • org.apache.http.annotation.GuardedBy
  • com.android.annotations.concurrency.GuardedBy
  • androidx.annotation.GuardedBy
  • com.google.errorprone.annotations.concurrent.GuardedBy
