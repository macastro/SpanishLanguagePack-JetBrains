Informa sobre el modificador synchronized en los métodos.

Hay varias razones por las que un modificador synchronized en un método puede ser una mala idea:

  1. Se debe realizar el menor trabajo posible bajo un bloqueo. Por lo tanto, suele ser mejor usar un bloque synchronized y mantener allí solo el código que funciona con el estado compartido.
  2. La sincronización se convierte en parte de la interfaz de un método. Esto dificulta la transición a un mecanismo de bloqueo diferente.
  3. Hacer un seguimiento de lo que bloquea un objeto en particular se vuelve más difícil.
  4. El ataque DoS (denegación de servicio) se vuelve factible ya sea a propósito o sin saberlo al heredar la clase del método.

Como alternativa, considere sincronizar con un objeto de bloqueo private final, cuyo acceso se puede controlar completamente.

Se proporciona una solución rápida para envolver el cuerpo del método con synchronized(this).

Ejemplo:


  class Main {
    public synchronized void fooBar() {
    }
  }

Después de aplicar la solución rápida:


  class Main {
    public void fooBar() {
      synchronized (this) {
      }
    }
  }

Puede configurar las siguientes opciones para esta inspección:

  1. Incluir métodos nativos - incluir métodos nativos en el ámbito de la inspección.
  2. Ignorar métodos que sobrescriben un método sincronizado - no informar métodos que sobrescriben un método synchronized.
