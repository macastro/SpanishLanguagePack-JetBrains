Reporta las clases que implementan java.io.Serializable donde los métodos readResolve() o writeReplace() no están declarados protected.

Declarar los métodos readResolve() y writeReplace() como privados puede forzar a las subclases a ignorarlos silenciosamente, mientras que declararlos públicos permite que sean invocados por código no confiable.

Si la clase contenedora está declarada como final, estos métodos pueden declararse como privados.

Ejemplo:


  class ClassWithSerialization implements Serializable {
    public Object writeReplace() { // advertencia: 'writeReplace()' no declarado como protected
        ...
    }
  }
