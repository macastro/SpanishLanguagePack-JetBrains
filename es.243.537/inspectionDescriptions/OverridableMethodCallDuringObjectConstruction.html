Informa de llamadas a métodos que se pueden anular de la clase actual durante la construcción del objeto.

Se llama a un método durante la construcción del objeto si se encuentra dentro de:

  • Constructor
  • Inicializador de instancias no estático
  • Inicializador de campo no estático
  • Método clone()
  • Método readObject()
  • Método readObjectNoData()

Los métodos se pueden anular si no se declaran como final, static o private. Los métodos locales de paquetes se consideran seguros, aunque se puedan anular. Estas llamadas pueden provocar errores sutiles, ya que la inicialización del objeto puede producirse antes de la llamada al método.

Ejemplo:


  class Padre {
    void someMethod() { }
  }

  class Hijo extends Padre {
    Hijo() {
      someMethod();
    }
  }

Esta inspección comparte la funcionalidad con las siguientes inspecciones:

  • Método abstracto llamado durante la construcción del objeto
  • Método anulado llamado durante la construcción del objeto

Sólo debe habilitarse una inspección a la vez para evitar la duplicación de advertencias.
