Informa cuando un método sin anotación @Pattern anula un método anotado como @Pattern. Esto no impide el resaltado de errores dentro del editor, sin embargo, el método de anulación no se comprobará en tiempo de ejecución.

Se proporciona una solución rápida para añadir una anotación @Pattern que coincida con la del método de la superclase. Esto asegura que la instrumentación de la comprobación en tiempo de ejecución funciona correctamente.

Ejemplo:


  abstract class Parent {
    abstract @Pattern("\\d\\d-\\d\\d\\d") String getId();
  }
  class Child extends Parent {
    @Override String getId() { // advertencia aquí
      return "12-345";
    }
  }
