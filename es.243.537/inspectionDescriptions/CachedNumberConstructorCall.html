Informa sobre las instanciaciones de nuevos objetos Long, Integer, Short o Byte que tienen un argumento primitivo long, integer, short o byte.

Se recomienda utilizar el método estático valueOf() introducido en Java 5. Por defecto, este método guarda en caché los objetos para valores entre -128 y 127, ambos incluidos.

Ejemplo:


  Integer i = new Integer(1);
  Long l = new Long(1L);

Después de aplicar la corrección rápida, el código cambia a:


  Integer i = Integer.valueOf(1);
  Long l = Long.valueOf(1L);

Esta inspección solo informa si el nivel de lenguaje del proyecto o módulo es 5 o superior.

Utiliza la opción Ignorar nuevas expresiones numéricas con un argumento de cadena para ignorar las llamadas a constructores numéricos con un argumento String.

Utiliza la opción Informar solo cuando el constructor esté marcado como @Deprecated para informar solo sobre las llamadas a constructores obsoletos. Los constructores Long, Integer, Short y Byte están obsoletos desde JDK 9.
