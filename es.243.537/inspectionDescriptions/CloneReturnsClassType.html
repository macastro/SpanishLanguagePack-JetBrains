Informa a los métodos clone() con tipos de retorno diferentes a la clase en la que se encuentran.

A menudo un método clone() tendrá un tipo de retorno de java.lang.Object, lo que dificulta su uso por sus clientes. Effective Java (segunda y tercera edición) recomienda hacer que el tipo de retorno de los métodos clone() sea el mismo que el tipo de clase del objeto que devuelve.

Ejemplo:


  class Foo implements Cloneable {
    public Object clone() {
      try {
        return super.clone();
      } catch (CloneNotSupportedException e) {
        throw new AssertionError();
      }
    }
  }

Después de que se aplica la corrección rápida:


  class Foo implements Cloneable {
    public Foo clone() {
      try {
        return (Foo)super.clone();
      } catch (CloneNotSupportedException e) {
        throw new AssertionError();
      }
    }
  }
