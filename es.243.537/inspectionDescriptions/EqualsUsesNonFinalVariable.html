Reporta implementaciones del equals() que acceden a variables no final. Tal acceso puede hacer que el equals() devuelva resultados diferentes en distintos momentos del ciclo de vida del objeto, lo que puede causar problemas al utilizar clases de colección estándar.

Ejemplo:


  public class Persona {
    private String apellido;

    @Override
    public boolean equals(Object obj) {
      ...
      Persona other = (Persona) obj;
      if (apellido == null) {
        if (!apellido.equals(other.apellido)) {
        return false;
      ...
      }
    }
  }
