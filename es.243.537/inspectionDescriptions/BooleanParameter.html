Informa sobre métodos públicos que aceptan un parámetro boolean.

Casi siempre es una mala práctica agregar un parámetro boolean a un método público (parte de una API) si ese método no es un setter. Al leer código que utiliza tal método, puede ser difícil descifrar qué representa el boolean sin mirar el código fuente o la documentación.

Este problema también se conoce como la trampa booleana. El parámetro boolean a menudo puede reemplazarse con una enumeración.

Ejemplo:


  // Advertencia: es difícil entender qué
  // significan los parámetros booleanos al mirar
  // una llamada a este método
  public boolean setPermission(File f,
                               int access,
                               boolean enable,
                               boolean ownerOnly) {
    // ...
  }

Utiliza la opción Solo informar métodos con múltiples parámetros booleanos para advertir solo cuando un método contiene más de un parámetro booleano.
