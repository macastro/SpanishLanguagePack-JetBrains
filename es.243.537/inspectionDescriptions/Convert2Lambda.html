Reporta clases anónimas que pueden ser reemplazadas con expresiones lambda.

Ejemplo:


  new Thread(new Runnable() {
    @Override
    public void run() {
      // corre hilo
    }
  });

Después de aplicar la corrección rápida:


  new Thread(() -> {
    // corre hilo
  });

Tenga en cuenta que si una clase anónima es convertida en una lambda sin estado, el mismo objeto lambda puede ser reutilizado por el runtime Java durante invocaciones subsecuentes. Por otro lado, cuando se emplea una clase anónima, se crean objetos separados cada vez. Por tanto, aplicar la corrección rápida puede causar un cambio en la semántica en casos raros, por ejemplo, cuando las instancias de una clase anónima se utilizan como claves en un HashMap.

Utilice la opción Reportar cuando la interfaz no esté anotada con @FunctionalInterface para ignorar los casos en los que una clase anónima implementa una interfaz sin la anotación @FunctionalInterface.
