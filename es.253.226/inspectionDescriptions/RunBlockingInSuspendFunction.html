Informa sobre las llamadas a runBlocking dentro de una función suspend.

Usar runBlocking dentro de una función suspend bloquea el hilo de la llamada, lo que va en contra del propósito de la programación asíncrona.

La corrección rápida reemplaza la llamada a runBlocking por una de las siguientes alternativas, según el contexto:

  • Una llamada a run.
  • Una llamada a withContext cuando se usa un CoroutineContext específico.
  • El código insertado directamente sin el contenedor runBlocking.

Ejemplo:


suspend fun something() {
  runBlocking {
    code() // El hilo se bloquea aquí
  }
}

Después de aplicar la corrección rápida:


suspend fun something() {
  code() // Se ejecuta de forma asíncrona
}
