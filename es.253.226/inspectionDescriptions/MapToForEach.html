Informa sobre las llamadas a map cuyo resultado se ignora y que pueden sustituirse por forEach.

El uso de map de esta forma es engañoso, ya que map está diseñado para transformar colecciones y devolver un resultado. Cuando su valor de retorno no se utiliza, la llamada crea asignaciones innecesarias y reduce la legibilidad del código. En tales casos, se debe utilizar forEach en su lugar, ya que expresa claramente una iteración con efectos secundarios.

La corrección rápida sustituirá la llamada a map por una llamada a forEach.

Ejemplo:


fun foo(list: List<String>) {
 list.map { print(it) } // Informado

 val x = list.map { it.length } // No informado: el resultado se utiliza
}

Tras aplicar la corrección rápida:


fun foo(list: List<String>) {
 list.forEach { print(it) }

 val x = list.map { it.length } // No informado: el resultado se utiliza
}
