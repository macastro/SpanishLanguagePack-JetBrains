Informa del uso de anotaciones en parámetros de valor sin un destino explícito, donde el destino predeterminado puede ser inesperado y no deseado.

Antes de Kotlin 2.1.20, las anotaciones en parámetros de valor sin un destino explícito solo se aplicaban al propio parámetro. En el caso de las anotaciones que se pueden aplicar tanto a parámetros como a campos o propiedades, este comportamiento puede no ser el deseado cuando la anotación se utiliza en un parámetro del constructor val o var.

Ejemplo: 


class Example(
    @Param val foo: String,
    @ParamField val bar: String, // el campo no está anotado, solo el parámetro
    @ParamProperty val baz: String, // la propiedad no está anotada, solo el parámetro
    @ParamProperty boo: String,
)

@Target(AnnotationTarget.VALUE_PARAMETER)
annotation class Param
@Target(AnnotationTarget.VALUE_PARAMETER, AnnotationTarget.FIELD)
annotation class ParamField
@Target(AnnotationTarget.VALUE_PARAMETER, AnnotationTarget.PROPERTY)
annotation class ParamProperty

Tras aplicar la solución rápida:


class Example(
    @Param val foo: String,
    @field:ParamField val bar: String,
    @property:ParamProperty val baz: String,
    @ParamProperty boo: String,
)

@Target(AnnotationTarget.VALUE_PARAMETER)
annotation class Param
@Target(AnnotationTarget.VALUE_PARAMETER, AnnotationTarget.FIELD)
annotation class ParamField
@Target(AnnotationTarget.VALUE_PARAMETER, AnnotationTarget.PROPERTY)
annotation class ParamProperty
