Esta inspección detecta posibles desreferenciaciones de nil y tiene en cuenta todas las llamadas a funciones. El análisis es interprocedimental y considera las llamadas en todo el proyecto, incluidas las que se producen entre paquetes diferentes.

Ejemplo:


package a

func CreateUser(name string) *User {
  if isValid(name) {
    return &User{Name: name}
  }
  return nil
}


package main

func main() {
	user := CreateUser("NilDereference")
	print(user.Name) // Posible desreferenciación de nil
}

En el ejemplo proporcionado, user es un puntero a la estructura User, pero puede ser nil porque CreateUser puede devolver nil. Intentar desreferenciar user para acceder a Name sin comprobar si user es nil provocará un error en tiempo de ejecución.
