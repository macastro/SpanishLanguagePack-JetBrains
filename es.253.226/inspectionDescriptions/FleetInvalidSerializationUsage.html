Detecta el uso inadecuado de los métodos de serialización de Kotlin en la base de código de Fleet que carecen de un parámetro explícito de estrategia de serialización/deserialización.

Esta inspección identifica las llamadas a los métodos de serialización JSON que pueden provocar una reflexión inesperada en tiempo de ejecución si no se parametrizan correctamente.

La inspección comprueba específicamente las llamadas a los siguientes métodos del paquete kotlinx.serialization.json.Json: 


decodeFromString
decodeFromJsonElement
encodeToString
encodeToJsonElement

Estos métodos deben llamarse siempre con un parámetro explícito de estrategia de serialización/deserialización para evitar la dependencia de la reflexión, lo que podría causar problemas en el entorno multiplataforma de Fleet.

Ejemplo de código problemático:


Json.encodeToString(myObject) // Falta SerializationStrategy

Uso correcto:


Json.encodeToString(MyType.serializer(), myObject)
