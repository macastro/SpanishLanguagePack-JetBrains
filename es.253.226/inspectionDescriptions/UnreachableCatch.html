Informa sobre las secciones catch que nunca se ejecutan, a pesar de estar permitidas por la Especificación del lenguaje Java.

Aunque las secciones catch inalcanzables normalmente no son permitidas por el compilador de Java y se notifican como errores de compilación, en ciertos casos el análisis requerido por la especificación del lenguaje Java es incompleto. Esta inspección proporciona un análisis mejorado y notifica las secciones catch inalcanzables que el compilador no detecta. Dichas secciones son redundantes y pueden eliminarse de forma segura.

Ejemplo:


  void method() {
    try { 
      throw new FileNotFoundException(); 
    }
    catch (FileNotFoundException e) { 
      
    }
    catch (IOException e) {
      // La especificación permite este catch
      // pero nunca se ejecuta
    }
  }

Se proporciona una corrección rápida para eliminar la sección catch redundante:


  void method() {
    try { 
      throw new FileNotFoundException(); 
    }
    catch (FileNotFoundException e) { 
      
    }
  }
