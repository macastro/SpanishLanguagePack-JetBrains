Consolas de base de datos, archivos scratch y buffers scratch

Las consolas de base de datos son archivos SQL que ya están adjuntos a una fuente de datos. Cuando creas una fuente de datos, se crea automáticamente una consola de base de datos. Para abrir la consola, haz clic derecho en la fuente de datos y selecciona Navigation | Jump to Query Console | Open Default Console. Para crear una nueva consola, haz clic derecho en la fuente de datos y selecciona New | Query Console.

Los archivos scratch son archivos SQL que no están adjuntos a una fuente de datos. Considera los archivos scratch como notas temporales o borradores para algunas ideas de código fuera del contexto del proyecto. Para crear un archivo scratch, navega a File | New | Scratch File (o presiona &shortcut:NewScratchFile;).

Los buffers scratch son archivos de texto simples sin ninguna característica de asistencia de codificación. Los buffers scratch pueden usarse para listas de tareas simples y notas para ti mismo. No hay un elemento de menú dedicado para la acción de crear un nuevo buffer scratch, pero puedes usar el popup Find Action (&shortcut:GotoAction;) y ejecutar la acción New Scratch Buffer.
