Reporta las cláusulas throws que son más genéricas que las excepciones que el método lanza en realidad.

Ejemplo:


  public void createFile() throws Exception { // Advertencia: 'throws Exception' es demasiado amplia, oculta la excepción 'IOException'
    File file = new File("pathToFile");
    file.createNewFile();
  }

Después de aplicar la corrección rápida:


  public void createFile() throws IOException {
    File file = new File("pathToFile");
    file.createNewFile();
  }

Configuración de la inspección:

  • Utilice el campo Número máximo de excepciones ocultas para advertir para ignorar las excepciones que ocultan más excepciones de las especificadas.
  • Utilice la opción Advertir solo para RuntimeException, Exception, Error o Throwable para que esta inspección solo advierta sobre las excepciones más comunes.
  • Utilice la opción Ignorar excepciones declaradas en métodos que sobrescriben métodos de biblioteca para ignorar las cláusulas throws demasiado amplias en métodos que sobrescriben métodos de biblioteca.
  • Utilice la opción Ignorar excepciones que ocultan otras pero que se lanzan a sí mismas para ignorar cualquier excepción que oculte otras excepciones pero que aún pueda ser lanzada desde el cuerpo del método y, por lo tanto, técnicamente no sea demasiado amplia.
