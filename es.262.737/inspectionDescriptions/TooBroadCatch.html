Reporta los bloques catch cuyos parámetros son más generales que las excepciones lanzadas por el bloque try correspondiente.

Ejemplo:


  try  {
    File file = new File(pathToFile);
    return file.getAbsolutePath();
  } catch (Exception ex) { // advertencia: el 'catch' de 'Exception' es demasiado amplio, ocultando la excepción 'RuntimeException'
    return defaultFilePath;
  }

Después de aplicar el arreglo rápido:


  try  {
    File file = new File(pathToFile);
    return file.getAbsolutePath();
  } catch (RuntimeException ex) {
    return defaultFilePath;
  }

Configuración de la inspección:

  • Utilice la opción Advertir solo para RuntimeException, Exception, Error o Throwable para que esta inspección solo advierta sobre las excepciones más comunes.
  • Utilice la opción Ignorar excepciones que ocultan otras excepciones pero que son lanzadas por sí mismas para ignorar cualquier excepción que oculte otras excepciones pero que aún pueda ser lanzada y, por lo tanto, técnicamente no sea demasiado amplia.
