Informa sobre campos cuyos nombres son demasiado cortos, demasiado largos o no siguen el patrón de expresión regular especificado.

Ejemplo: si la inspección está habilitada para constantes y la longitud mínima especificada para el nombre del campo es 5 (valor predeterminado), se generará una advertencia para la siguiente constante, ya que la longitud de su nombre es 3, que es menor que 5: public static final int MAX = 42;.

La corrección rápida para cambiar el nombre de tales campos solo está disponible en el editor.

Configuración de la inspección:

Utilice la lista en la sección Opciones para especificar qué campos deben inspeccionarse. Desmarque la casilla de los campos para los que desee omitir la inspección.

Para cada tipo de campo, utilice los campos de entrada proporcionados para especificar la longitud mínima, la longitud máxima y la expresión regular del nombre del campo. Especifique 0 en los campos de longitud para omitir la comprobación correspondiente.

Las expresiones regulares deben especificarse en el formato estándar de java.util.regex.
