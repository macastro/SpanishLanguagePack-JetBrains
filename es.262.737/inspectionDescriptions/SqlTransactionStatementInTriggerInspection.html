Informa sobre el uso de sentencias de gestión de transacciones (como COMMIT o ROLLBACK) en el cuerpo de un disparador (trigger).

Si se utilizan sentencias COMMIT o ROLLBACK en el cuerpo de un disparador, este no se compilará. El error se produce porque el disparador se activa en medio de una transacción. En el momento en que se inicia el disparador, la transacción actual aún no ha finalizado. Dado que COMMIT finaliza la transacción, ambas sentencias (COMMIT y ROLLBACK) provocan excepciones. Los cambios realizados dentro de un disparador deben ser confirmados (committed) o revertidos (rolled back) por la transacción propietaria que activó el disparador.

Ejemplo (Oracle):

CREATE TABLE employee_audit
(
    id          INT  NOT NULL,
    update_date DATE NOT NULL,
    old_name    VARCHAR2(100),
    new_name    VARCHAR2(100)
);

CREATE TABLE employees
(
    id   INT           NOT NULL,
    name VARCHAR2(100) NOT NULL
);

CREATE OR REPLACE TRIGGER trig_commit
    AFTER UPDATE OF name
    ON employees
    FOR EACH ROW
BEGIN
    INSERT INTO employee_audit VALUES (:old.id, SYSDATE, :old.name, :new.name);
    COMMIT;
END;

CREATE OR REPLACE TRIGGER trig_rollback
    AFTER UPDATE OF name
    ON employees
    FOR EACH ROW
BEGIN
    INSERT INTO employee_audit VALUES (:old.id, SYSDATE, :old.name, :new.name);
    ROLLBACK;
END;
