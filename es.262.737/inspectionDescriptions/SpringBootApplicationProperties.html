Informa sobre claves de configuración no resueltas y obsoletas, así como valores no válidos en los archivos de configuración .properties de aplicaciones Spring Boot, lo que podría provocar errores en tiempo de ejecución.

Ejemplo:


server.port=invalid # informa 'no se puede convertir 'invalid' a java.lang.Integer'

Si una clave de configuración obsoleta tiene una clave de reemplazo, puede aplicar la corrección rápida 'Usar clave de reemplazo'.

Ejemplo:


logging.path=${path} # informa 'propiedad de configuración obsoleta 'logging.path''

Tras aplicar la corrección rápida:


logging.file.path=${path}

Si una clave de configuración no está definida en spring-configuration-metadata.json, puede aplicar la corrección rápida 'Definir clave de configuración' para crear el archivo META-INF/spring-additional-configuration-metadata.json y definir las claves necesarias.

Ejemplo:


new.key=value # informa 'no se puede resolver la propiedad de configuración 'new.key''

Tras aplicar la corrección rápida, se añadirá lo siguiente a META-INF/spring-additional-configuration-metadata.json:


{
  "properties": [
    {
      "name": "new.key",
      "type": "java.lang.String",
      "description": "Description for new.key."
  }
] }

Esta inspección también resalta errores de notación de índice en las claves de configuración de listas y mapas.

Ejemplo:


spring.datasource.schema[]=${schema} # informa 'falta el valor del índice'

Utilice la opción Tokens de reemplazo para definir los tokens utilizados para los marcadores de posición de valores en los archivos de configuración. Estos tokens se especifican en el formato tokenInicio*tokenFin. Si no hay un *, se asume que los tokens de inicio y fin son los mismos.

Por ejemplo, el valor predeterminado para los tokens de inicio y fin es @, lo que le permite definir marcadores de posición como some.property=@another.property@.

Los valores dentro de los tokens de reemplazo (@property.key@) no se resaltarán.
