Reporta llamadas a runBlocking dentro de funciones suspend.

Usar runBlocking en una función suspend bloquea el hilo de llamada, lo cual va en contra del propósito de la programación asíncrona.

Esta corrección rápida puede reemplazar la llamada a runBlocking con una de las siguientes alternativas, dependiendo del contexto:

  • Una llamada a run.
  • Una llamada a withContext cuando se usa un CoroutineContext específico.
  • Código inline directo sin el wrapper runBlocking.

Ejemplo:


suspend fun something() {
  runBlocking {
    code() // el hilo se bloquea aquí
  }
}

Después de aplicar la corrección rápida:


suspend fun something() {
  code() // se ejecuta de forma asíncrona
}
