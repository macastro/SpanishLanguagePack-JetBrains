Esta inspección puede detectar potenciales desreferencias nil y considera todas las llamadas a funciones. El análisis es interprocedimental y considera las llamadas en todo el proyecto, incluyendo llamadas entre diferentes paquetes.

Ejemplo:


package a
func CreateUser(name string) *User {
  if isValid(name) {
    return &User{Name: name}
  }
  return nil
}


package main
func main() {
	user := CreateUser("NilDereference")
	print(user.Name) // Potencial desreferencia nil
}

En el ejemplo proporcionado, user es un puntero a la estructura User, pero puede ser nil, ya que CreateUser puede devolver nil. Intentar desreferenciar user para acceder a Name sin verificar si user es nil causará un error en tiempo de ejecución.
