Reporta el uso incorrecto de objetos companion en extensiones.

Los objetos companion de Kotlin siempre se crean cuando intentas cargar su clase contenedora, y la creación de implementaciones de puntos de extensión debería tener un costo bajo. La carga excesiva de clases en plugins es un gran problema para el arranque del IDE.

Patrón incorrecto:


  class KotlinDocumentationProvider : AbstractDocumentationProvider(), ExternalDocumentationProvider {
      companion object {
          private val LOG = Logger.getInstance(KotlinDocumentationProvider::class.java)
          private val javaDocumentProvider = JavaDocumentationProvider()
      }
  }

Aquí KotlinDocumentationProvider es una extensión registrada en plugin.xml: 


  <lang.documentationProvider language="kotlin"
                              implementationClass="org.jetbrains.kotlin.idea.KotlinDocumentationProvider"
                              order="first"/>

En este ejemplo, tan pronto como alguien llame a new KotlinDocumentationProvider(), se cargará JavaDocumentationProvider desde el disco.

Los objetos companion de Kotlin en implementaciones de puntos de extensión solo pueden contener loggers y constantes simples. Otras declaraciones pueden causar carga excesiva de clases o inicialización temprana de recursos costosos (TokenSet, Regex, etc.) al cargar la clase de extensión. Ten en cuenta que incluso las declaraciones marcadas como @JvmStatic generan una clase adicional para el objeto companion, lo que puede llevar a cálculos extremadamente costosos.

Estas declaraciones deben almacenarse en objetos o a nivel superior, no en objetos companion.

Preguntas frecuentes

¿Cómo reescribir ConfigurationType en ejecución?

Mueve la declaración al nivel superior:


  // Forma correcta, usando fun de nivel superior
  internal fun mnRunConfigurationType(): MnRunConfigurationType = runConfigurationType<MnRunConfigurationType>()
  internal class MnRunConfigurationType : ConfigurationType {
    companion object { // Forma incorrecta
      fun getInstance(): MnRunConfigurationType = runConfigurationType<MnRunConfigurationType>()
    }
    ...

¿Cómo reescribir FileType?

Antes:


  internal class SpringBootImportsFileType : LanguageFileType(SPILanguage.INSTANCE, true) {
    companion object {
      val FILE_TYPE = SpringBootImportsFileType()
      ...

Después:


  internal object SpringBootImportsFileType : LanguageFileType(SPILanguage.INSTANCE, true) {
  ...

Usa INSTANCE fieldName en plugin.xml:


  <fileType name="Spring Boot Imports"
              fieldName="INSTANCE"
              implementationClass="com.intellij.spring.boot.spi.SpringBootImportsFileType"/>

¿Cómo reescribir CounterUsagesCollector?

API interna

Antes:


  class AntActionsUsagesCollector : CounterUsagesCollector() {
    override fun getGroup(): EventLogGroup = GROUP
    companion object {
      private val GROUP = EventLogGroup("build.ant.actions", 1)
      @JvmField
      val runSelectedBuildAction = GROUP.registerEvent("RunSelectedBuild")
   }
}

Después:


 object AntActionsUsagesCollector : CounterUsagesCollector() {
  override fun getGroup(): EventLogGroup = GROUP
  private val GROUP = EventLogGroup("build.ant.actions", 1)
  @JvmField
  val runSelectedBuildAction = GROUP.registerEvent("RunSelectedBuild")
}
