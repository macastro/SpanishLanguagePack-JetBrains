Reporta clases que pueden convertirse en clases record.

Las clases record se enfocan en modelar datos inmutables en lugar de comportamiento extensible. Las implementaciones automáticas implícitas de métodos orientados a datos (como equals() y accessors) ayudan a reducir el código repetitivo.

Ten en cuenta que no todas las clases pueden ser clases record. Estas son algunas limitaciones:

  • La clase no puede tener subclases.
  • Todos los campos no static en la clase deben ser final.
  • No debe haber inicializadores, constructores genéricos ni métodos nativos.

Para una descripción completa de las clases record, consulta la Especificación del Lenguaje Java.

Ejemplo:


  class Point {
    private final double x;
    private final double y;
    Point(double x, double y) {
      this.x = x;
      this.y = y;
    }
    double getX() {
      return x;
    }
    double getY() {
      return y;
    }
  }

Después de aplicar la corrección rápida:


  record Point(int x, int y) {
  }

Habilitar la opción Sugerir renombrar métodos accessor puede renombrar automáticamente los accessors getX()/isX() a x().

Usa la opción Deshabilitar resaltado si los miembros se vuelven más accesibles para excluir clases donde la accesibilidad de los miembros se debilitaría por la conversión. La corrección rápida siempre estará disponible como una intención, y activarla mostrará los miembros afectados y solicitará confirmación. En modo batch, la conversión no será sugerida.

Usa la lista Prohibir conversión de clases con las siguientes anotaciones para excluir de la conversión las clases cuyas anotaciones coincidan con los patrones especificados.
