Excepciones verificadas en Kotlin

Esta detección solo se aplica a métodos con la anotación com.intellij.openapi.ThrowsChecked.

Esta verificación intenta encontrar casos donde se llama a funciones anotadas pero no se capturan las excepciones.

Diferencias con las excepciones verificadas de Java:

  • Esta verificación produce advertencias, pero no interrumpe la compilación.
  • Por diseño, se pueden suprimir las excepciones.
  • Esta verificación no debería intentar demostrar en cada caso que ciertas excepciones no son capturadas. Solo sirve para recordar a los usuarios de la API sobre las excepciones.
