Reporta clases, interfaces y métodos que extienden, implementan o sobrescriben elementos de API marcados con @ApiStatus.NonExtendable.

La anotación @ApiStatus.NonExtendable indica que una clase, interface o método no debe ser extendido, implementado o sobrescrito. Es común convertir tales interfaces y clases en implementaciones internas de biblioteca, por lo tanto, si el cliente proporciona una implementación diferente, obtendrá una ClassCastException. Agregar nuevos métodos abstract a tales clases e interfaces romperá la compatibilidad con las implementaciones del cliente.

Ejemplo:


  // En código de biblioteca upstream
  @ApiStatus.NonExtendable
  public interface MyService {
    public void perform();
  }
  // En código de cliente downstream
  public class MyServiceImpl implements MyService {
    @Override
    public void perform() {
      // ...
    }
  }
Esta inspección también puede detectar los siguientes problemas relacionados con la aplicación incorrecta de anotaciones:
  • Agregar anotaciones a clases/métodos que no pueden ser extendidos/sobrescritos
  • Agregar redundantemente anotaciones a métodos en clases que ya están anotadas con @ApiStatus.NonExtendable
