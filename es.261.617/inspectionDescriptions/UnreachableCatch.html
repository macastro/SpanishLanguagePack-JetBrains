Reporta aquellas partes catch que, aunque cumplen con la especificación del lenguaje Java, nunca son ejecutadas.

Aunque las partes catch inalcanzables generalmente no son permitidas por el compilador de Java y se reportan como errores de compilación, en ciertas situaciones, el análisis requerido por la especificación del lenguaje Java no es completo. Esta inspección proporciona un análisis mejorado y reportará algunas partes catch inalcanzables que el compilador no reporta. Estas partes son redundantes y pueden ser eliminadas de forma segura.

Ejemplo:


  void method() {
    try { 
      throw new FileNotFoundException(); 
    }
    catch (FileNotFoundException e) { 
    }
    catch (IOException e) {
      // La especificación permite este catch
      // pero nunca es ejecutado
    }
  }

Se proporciona una corrección rápida para eliminar las partes catch redundantes:


  void method() {
    try { 
      throw new FileNotFoundException(); 
    }
    catch (FileNotFoundException e) { 
    }
  }
