Informa sobre bloques catch vacíos o que podrían estar omitiendo excepciones.

Aunque ocasionalmente es intencionado, los bloques catch vacíos pueden complicar la depuración. Además, ignorar los parámetros de un catch puede ser un error. Por último, los analizadores de código estático informan si detectan que un bloque catch podría estar ignorando silenciosamente excepciones importantes de la VM, como NullPointerException. Ignorar tales excepciones (sin registrarlas ni volver a lanzarlas) puede ocultar errores.

Esta inspección no informará sobre ningún parámetro catch denominado ignore o ignored. Por el contrario, advertirá sobre los parámetros catch llamados ignore o ignored que se estén utilizando realmente. Además, la inspección no informará sobre los parámetros catch en fuentes de prueba denominados expected u ok.

Puede utilizar un arreglo rápido para cambiar el nombre de la excepción a ignored. Para los bloques catch vacíos, se sugiere otro arreglo rápido para generar el cuerpo del catch. Puede modificar la plantilla "Catch Statement Body" en la pestaña "Código" en Configuración | Editor | Plantillas de archivos y código.

Ejemplo:


  try {
    throwingMethod();
  } catch (IOException ex) {

  }

Después de aplicar el arreglo rápido:


  try {
    System.out.println(System.in.read());
  } catch (IOException ignored) {

  }

Configurar inspección:

  • Utilice la opción No advertir cuando el bloque 'catch' contenga un comentario para omitir los bloques catch que contienen comentarios.
  • Utilice la opción No advertir cuando el bloque 'catch' no esté vacío para omitir los bloques catch que contienen sentencias o comentarios pero no utilizan la variable en sí.
  • Utilice la opción No advertir cuando una excepción de nombre 'ignore(d)' no se ignore realmente para omitir las variables en uso denominadas ignored.
