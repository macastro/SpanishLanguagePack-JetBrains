Reporta llamadas a wait(), notify() y notifyAll() que no se realizan dentro de una sentencia synchronized o un método synchronized correspondiente.

Llamar a estos métodos en un objeto sin poseer el bloqueo (lock) provocará una excepción IllegalMonitorStateException. Esta construcción no es necesariamente un error, ya que el bloqueo necesario podría haberse adquirido antes de llamar al método que la contiene, pero aun así se recomienda revisarla con atención.

Ejemplo:


  class Sync {
    private final Object lock = new Object();

    void test() throws InterruptedException {
      synchronized (this) {
        lock.wait(); // 'lock.wait()' no está sincronizado en 'lock'
      }
    }
  }
