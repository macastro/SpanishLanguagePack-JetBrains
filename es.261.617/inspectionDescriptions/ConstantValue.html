Reporta expresiones y condiciones que siempre producen el mismo resultado, como true, false, null o cero. Tales expresiones pueden ser reemplazadas por el valor constante correspondiente, o pueden indicar la presencia de un bug en el código.

Ejemplo:

  // Siempre true
  // Razón: se usa ||, no &&
  if (x > 0 || x < 10) {}
  System.out.println(str.trim());
  // Siempre false
  // Razón: la variable se desreferencia antes de la verificación de null
  if (str == null) {}

El comportamiento de la inspección puede ser controlado por muchas anotaciones, como las anotaciones de nullabilidad, la anotación @Contract, la anotación @Range, etc.

Configurar la inspección:

  • Usa la opción No reportar aserciones que están estáticamente probadas como siempre true para evitar reportar aserciones que están estáticamente probadas como siempre true. Esto también incluye condiciones como if (alwaysFalseCondition) throw new IllegalArgumentException();.
  • Usa la opción Ignorar declaraciones assert para controlar cómo la inspección maneja las declaraciones assert. Por defecto, las aserciones se asumen como ejecutadas (modo -ea). Cuando esta opción está habilitada, las aserciones serán ignoradas (modo -da).
  • Usa la opción Advertir cuando las constantes se almacenan en variables para mostrar advertencias cuando se usan variables cuyo valor se conoce que es constante.

Antes de IntelliJ IDEA 2022.3, esta inspección era parte de la inspección Condiciones constantes y excepciones. Esta inspección se ha dividido en dos inspecciones: Valores constantes y Problemas de nullabilidad y flujo de datos.
