Reporta llamadas al constructor de BigDecimal que acepta valores double. Estos constructores producen un BigDecimal igual al valor double proporcionado. Sin embargo, dado que double está codificado en formato de punto flotante binario de doble precisión IEEE 754 de 64 bits, el valor exacto puede ser inesperado.

Por ejemplo, new BigDecimal(0.1) producirá un objeto BigDecimal con el valor 0.1000000000000000055511151231257827021181583404541015625, que es el número más cercano a 0.1 representable como double. Para obtener un BigDecimal que contenga el valor esperado 0.1, use new BigDecimal("0.1") o BigDecimal.valueOf(0.1).

Ejemplo:


  class Constructor {
    void foo() {
      new BigDecimal(0.1);
    }
  }

Después de aplicar la corrección rápida:


  class Constructor {
    void foo() {
      new BigDecimal("0.1");
    }
  }
