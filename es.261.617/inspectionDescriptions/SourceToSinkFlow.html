Informa de los casos en los que se pasa un objeto no seguro a un método con un parámetro marcado con la anotación @Untainted, se devuelve desde un método anotado o se asigna a un campo, parámetro o variable local anotados. No se admiten los métodos set y get de Kotlin para campos como puntos de entrada.

Los objetos seguros (dentro de la misma clase) son:

  • Literales de cadena, instancias de interfaz u objetos de enumeración
  • Resultados de llamadas a métodos marcados con @Untainted
  • Campos private asignados únicamente con literales de cadena y que tengan inicializadores seguros
  • Campos final con inicializadores seguros
  • Variables locales o parámetros marcados con @Untainted que no hayan sido asignados a partir de un objeto no seguro
Este campo, variable local o parámetro no debe pasarse como argumento a un método ni utilizarse como calificador, o bien debe ser un tipo primitivo, su wrapper o un objeto inmutable. Además, los campos static final también se consideran seguros.

El análisis se realiza únicamente dentro de un archivo. Para gestionar dependencias de otras clases, utilice las opciones. El análisis se extiende a métodos private o static y tiene límites de profundidad de propagación.

Ejemplo: 


  void doSmth(boolean b) {
    String s = safe();
    String s1 = "other";
    if (b) s1 = s;
    sink(s);
  }

  String sink(@Untainted String s) {}

Aquí no hay ninguna asignación de cadena no segura a s, por lo que no se genera ninguna advertencia. Por otro lado: 


  void doSmth(boolean b) {
    String s = safe();
    String s1 = "other";
    s1 = foo();
    if (b) s = s1;
    sink(s);        // Advertencia aquí
  }
  
  String foo();

  String sink(@Untainted String s) {}

Aquí hay una advertencia porque s1 tiene un estado desconocido tras la asignación del resultado de la llamada a foo.
