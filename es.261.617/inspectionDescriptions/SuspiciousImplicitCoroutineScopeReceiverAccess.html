Reporta accesos implícitos al receptor CoroutineScope que pueden ser problemáticos en contextos de suspensión.

Cuando una función de suspensión o Lambda captura un receptor CoroutineScope implícito desde un contexto externo, puede provocar comportamientos inesperados. Esto se debe a que el scope capturado puede ser cancelado o completado mientras la función de suspensión aún se está ejecutando.

Esta inspección puede detectar casos donde el código accede implícitamente al receptor CoroutineScope en funciones de suspensión o Lambdas, el acceso implícito crea una dependencia potencialmente peligrosa al scope externo.

Ejemplo:


class MyClass {
    fun CoroutineScope.launchJobs() {
        launch { // OK: uso directo en función de extensión
            doSomething()
        }
        suspendingFunction { // Advertencia: acceso sospechoso al receptor 'CoroutineScope' implícito
            launch { // la llamada launch usa el CoroutineScope implícito del contexto externo
                doSomethingElse()
            }
        }
    }
}

Opciones:

  • Detectar subtipos de CoroutineScope – Cuando está habilitado, la inspección también detectará accesos implícitos a subtipos de CoroutineScope

Para resolver este problema, puedes:

  • Especificar explícitamente el receptor usando this@receiverLabel.method()
  • Pasar el scope como parámetro a la función de suspensión
  • Usar el constructor coroutineScope { } para crear un scope hijo vinculado al ciclo de vida de la función de suspensión
