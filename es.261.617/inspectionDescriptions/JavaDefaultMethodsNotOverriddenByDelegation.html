Reporta delegación cuando el objeto delegado es una instancia de una clase que sobrescribe (en el caso de clases final y sealed) o puede sobrescribir (en el caso de clases open y abstract) métodos por defecto de Java.

Por diseño, la delegación de Kotlin no sobrescribe métodos por defecto de Java, lo que puede llevar a errores difíciles de detectar, especialmente en jerarquías de herencia profundas.

Ejemplo:


// Base.java:
public interface Base {
  default void print() {
    System.out.println("Base");
  }
}


// BaseImpl.kt:
class BaseImpl : Base {
  override fun print() {
    println("BaseImpl")
  }
}


// Main.kt:
fun main() {
  val inherited = BaseImpl()
  val delegated = object : Base by inherited {}
  inherited.print() // Outputs: BaseImpl
  delegated.print() // Outputs: Base
}

Los desarrolladores normalmente esperan que todos los métodos, incluyendo los métodos por defecto sobrescritos en la delegación, sean reenviados. En su lugar, se invoca el método por defecto de Java, lo que puede llevar a resultados inesperados.

Una solución rápida es sobrescribir estos métodos delegando la implementación del método por defecto de Java al objeto delegado. 


// Main.kt:
fun main() {
  val inherited = BaseImpl()
  val delegated = object : Base by inherited {
    override fun print() {
      inherited.print()
    }
  }
  inherited.print() // Outputs: BaseImpl
  delegated.print() // Outputs: BaseImpl
}

Otra solución rápida es sobrescribir estos métodos delegando explícitamente la implementación del método por defecto de Java a la superclase. 


// Main.kt:
fun main() {
  val inherited = BaseImpl()
  val delegated = object : Base by inherited {
    override fun print() {
      super.print()
    }
  }
  inherited.print() // Outputs: BaseImpl
  delegated.print() // Outputs: Base
}
