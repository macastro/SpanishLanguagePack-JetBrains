Reporta llamadas a métodos que usan java.nio.file.Files para leer o escribir String como bytes Estas llamadas pueden ser reemplazadas por llamadas a los métodos Files.readString() o Files.writeString() introducidos en Java 11.

Ejemplo:


  String s = "example";
  Files.write(Paths.get("out.txt"), s.getBytes(StandardCharsets.UTF_8), StandardOpenOption.WRITE);
  s = new String(Files.readAllBytes(Paths.get("in.txt")), StandardCharsets.ISO_8859_1);

Después de aplicar la corrección rápida:


  String s = "example";
  Files.writeString(Paths.get("out.txt"), s, StandardOpenOption.WRITE);
  s = Files.readString(Paths.get("in.txt"), StandardCharsets.ISO_8859_1);

Tenga en cuenta que readString() se comporta de manera diferente a new String(bytes, charset) cuando se trata de manejar caracteres inválidos (no mapeables). En tal caso, el método readString() lanzará una excepción, mientras que el método new String(bytes, charset) reemplazará silenciosamente los caracteres inválidos con caracteres de reemplazo. Si desea el reemplazo silencioso, es mejor suprimir la advertencia de inspección.
