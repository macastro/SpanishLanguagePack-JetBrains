Reporta problemas causados por el uso incorrecto de valores nil.

El IDE analizará el flujo de datos para determinar si una variable puede tener valores nil o no nil. El IDE reportará problemas potenciales en el código basándose en este resultado. Considera las siguientes situaciones que pueden llevar a consecuencias inesperadas:

  • Las llamadas a métodos con receptor nil pueden causar 'desreferencia de puntero nil'.
  • La indexación de slice nil puede causar panic.
  • Las comparaciones como v == nil pueden ser sin sentido si v se sabe que siempre es nil o no nil.
  • No se verifica si las variables con errores correspondientes no son nil. Un error corresponde a una variable cuando la variable y el error se definen o asignan juntos en una declaración como v, err := foo().
