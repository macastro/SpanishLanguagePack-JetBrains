Reporta expresiones if en cascada donde la rama else final es parte de una expresión con punto calificado o binaria.

Esto puede llevar a comportamientos inesperados, ya que solo la expresión if-else más interna se convierte en el receptor o el operando izquierdo de la expresión externa, lo que puede no coincidir con la lógica esperada.

Ejemplo:


fun printNumberSign(num: Int) {
    if (num > 0) {
        "positive"
    } else if (num < 0) {
        "negative"
    } else {
        "zero"
    }.let { print(it) }
}
fun main() {
    printNumberSign(1) // No imprime nada, porque el receptor de 'let' es solo la expresión 'if-else' interna
}

Después de aplicar la primera corrección rápida (convertir a when):


fun printNumberSign(num: Int) {
    when {
        num > 0 -> {
            "positive"
        }
        num < 0 -> {
            "negative"
        }
        else -> {
            "zero"
        }
    }.let { print(it) }
}
fun main() {
    printNumberSign(1) // Imprime "positive"
}

Alternativamente, la segunda corrección rápida añadirá llaves explícitas para mantener la semántica original, mientras hace explícita la rama else anidada:


fun printNumberSign(num: Int) {
    if (num > 0) {
        "positive"
    } else {
        if (num < 0) {
            "negative"
        } else {
            "zero"
        }.let { print(it) }
    }
}
fun main() {
    printNumberSign(1) // No imprime nada, pero el código es más claro
}
