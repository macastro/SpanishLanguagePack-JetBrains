Reporta clases que implementan java.lang.Comparator, pero que no implementan java.io.Serializable.

Si un comparador no serializable es usado para construir una colección ordenada como un java.util.TreeMap o un java.util.TreeSet, la colección también será no serializable. Esto puede resultar en errores inesperados y difíciles de diagnosticar.

Dado que las subclases de java.lang.Comparator a menudo no tienen estado, simplemente marcarlas como serializables es un pequeño costo para evitar tales problemas.

Ejemplo:


  class Foo implementa Comparator { // advertencia
      @Override
      public int compare(Object o1, Object o2) {
          /* ... */
      }
  }

Después de que el arreglo rápido es aplicado:


  class Foo implementa Comparator, Serializable { // no hay advertencia aquí
      @Override
      public int compare(Object o1, Object o2) {
          /* ... */
      }
  }
