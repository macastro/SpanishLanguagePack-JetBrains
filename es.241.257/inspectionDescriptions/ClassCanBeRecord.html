Informa las clases que se pueden convertir en clases de registro.

Las clases de registro se centran en el modelado de datos inmutables en lugar de un comportamiento extensible. La implementación implícita automática de métodos basados en datos, como equals() y accesores, ayuda a reducir el código repetitivo.

Tenga en cuenta que no todas las clases pueden ser clases de registro. Estas son algunas de las restricciones:

  • La clase debe ser de nivel superior y no debe tener subclases.
  • Todos los campos no estáticos de la clase deben ser finales.
  • No deben existir inicializadores, constructores genéricos ni métodos nativos.

Para obtener una descripción completa de las clases de registro, consulte Especificación del lenguaje Java.

Ejemplo:


  class Punto {
    private final double x;
    private final double y;

    Punto(double x, double y) {
      this.x = x;
      this.y = y;
    }

    double getX() {
      return x;
    }

    double getY() {
      return y;
    }
  }

Después de aplicar la solución rápida:


  registro Punto(int x, int y) {
  }

Habilite la opción Sugerir renombrar métodos accesores para renombrar los accesores getX()/isX() a x() automáticamente.

Utilice la opción Si los miembros se vuelven más accesibles para especificar qué hacer cuando la conversión hará que los miembros sean más accesibles:

  • Elija la opción No sugerir conversión para no convertir cuando los miembros se vuelvan más accesibles.
  • Elija la opción Mostrar vista de conflictos para mostrar los miembros afectados y solicitar que continúe. En el modo batch, no se sugerirá la conversión.
  • Elija la opción Convertir silenciosamente para aumentar la accesibilidad de forma silenciosa cuando sea necesario.

Utilice la lista Suprimir conversión si la clase está anotada por para excluir clases de la conversión cuando están anotadas por anotaciones que coinciden con los patrones especificados.
