Reporta clases vacías y archivos Java vacíos.

Una clase está vacía si no contiene algún campo, método, constructor o inicializador. Las clases vacías a veces son dejadas desatendidas después de cambios significativos o refactorizaciones.

Ejemplo:


  clase Ejemplo {
    Lista<String> obtenerLista() {
       return new ArrayList<>() {

       };
    }
  }

Después de que la reparación rápida es aplicada:


  clase Ejemplo {
    Lista<String> obtenerLista() {
       return new ArrayList<>();
    }
  }

Configurar el examen:

  • Usa la opción Ignorar si está anotada por para especificar anotaciones especiales. El examen ignorará las clases marcadas con estas anotaciones.

  • Usa la opción Ignorar clase si es una parametrización de un supertipo para ignorar las clases que parametrizan una superclase. Por ejemplo: 

    clase MiLista extiende ArrayList<String> {}
  • Usa la opción Ignorar subclases de java.lang.Throwable para ignorar las clases que extienden java.lang.Throwable.
  • Usa la opción Los comentarios cuentan como contenido para ignorar clases que contienen comentarios.
