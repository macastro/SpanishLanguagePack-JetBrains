Informa las llamadas a Collection.addAll() y Map.putAll() inmediatamente después de una instancia de una colección usando un constructor sin argumentos.

Tales construcciones pueden remplazarse con una sola llamada a un constructor parametrizado, lo que simplifica el código. Además, para algunas colecciones, el reemplazo podría ser más eficiente en cuanto al rendimiento.

Ejemplo:


  Set<String> set = new HashSet<>();
  set.addAll(Arrays.asList("alpha", "beta", "gamma"));

Después de que se aplique la solución rápida:


  Set<String> set = new HashSet<>(Arrays.asList("alpha", "beta", "gamma"));

Las clases de colección de JDK están soportadas de forma predeterminada. Además, puedes especificar otras clases usando el panel Clases para revisar.
