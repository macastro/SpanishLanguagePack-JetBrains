Informa sobre proveedores duplicados.

Ejemplo del problema:


provider "kubernetes" {
  config_path = "~/.kube/config"
}

provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
}

Una de las formas de resolver este problema es agregar un alias y especificarlo explícitamente al crear recursos en el proveedor necesario.

Ejemplo de corrección:


  provider "kubernetes" {
  config_path = "~/.kube/config"
}

provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
  alias = "cloud"
}
