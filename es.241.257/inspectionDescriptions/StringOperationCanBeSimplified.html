Reporta llamadas redundantes a constructores y métodos de String como toString() o substring() que pueden ser reemplazadas por una expresión más simple.

Por ejemplo, las llamadas a estos métodos pueden ser removidas de forma segura en código como "string".substring(0), "string".toString(), o new StringBuilder().toString().substring(1,3).

Ejemplo:


  System.out.println(new String("mensaje"));

Después de aplicar la corrección rápida:


  System.out.println("mensaje");

Tenga en cuenta que la corrección rápida elimina la llamada redundante al constructor, y esto puede afectar la igualdad referencial de String. Si necesita preservarla, aunque se considera una mala práctica, suprima la advertencia o use la configuración de inspección para ignorar las llamadas redundantes al constructor de String.

Utilice la opción No reportar llamadas al constructor de String a continuación para no reportar código como el ejemplo anterior. Esto evitará cambiar el resultado de las comparaciones de cadenas con == o != después de aplicar la corrección rápida en código que usa llamadas a new String() para garantizar una identidad de objeto diferente.
