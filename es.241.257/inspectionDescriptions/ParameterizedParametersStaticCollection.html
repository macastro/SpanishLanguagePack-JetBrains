Reporta clases de prueba parametrizada de JUnit 4 que están anotadas con @RunWith(Parameterized.class) pero que no incluyen un método proveedor de datos anotado con @Parameterized.Parameters o este método tiene una sintaxis incorrecta. Dichas clases de prueba no se pueden ejecutar. El método proveedor de datos debe ser public y static y debe tener un tipo de retorno de Iterable u Object[].

Sugiere crear un método proveedor de parámetros vacío o cambiar la sintaxis del método proveedor de datos incorrecto.

Ejemplo:


  @RunWith(Parameterized.class)
  public class ImportantTest {
    private int input;
    private int expected;

    ImportantTest(int input, int expected) {
      this.input = input;
      this.expected = expected;
    }

    // ... test cases
  }

Después de aplicar la corrección rápida:


  @RunWith(Parameterized.class)
  public class ImportantTest {
    private int input;
    private int expected;

    ImportantTest(int input, int expected) {
      this.input = input;
      this.expected = expected;
    }

    @Parameters
    public static Iterable<Object[]> parameters() {
      return null;
    }

    // ... test cases
  }
