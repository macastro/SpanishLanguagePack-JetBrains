Se informa de los casos en los que el argumento de una llamada a un método en un java.util.Collection o un java.util.Map es la propia colección o mapa. Estas situaciones pueden darse como resultado de una copia y pegado de código con tipos sin procesar.

Ejemplo:


  ArrayList list = new ArrayList<>();
  list.add(list); // aquí se genera una advertencia
  return list.hashCode(); // lanza StackOverflowError
