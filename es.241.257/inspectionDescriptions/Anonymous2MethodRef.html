Informa clases anónimas que se pueden reemplazar con referencias a métodos.

Tenga en cuenta que si una clase anónima se convierte en una referencia a un método sin vincular, el mismo objeto de referencia a un método puede ser reutilizado por el tiempo de ejecución de Java durante invocaciones posteriores. Por otro lado, cuando se utiliza una clase anónima, se crean objetos separados cada vez. Por lo tanto, la aplicación de la solución rápida puede provocar un cambio de semántica en casos excepcionales, por ejemplo, cuando las instancias de clases anónimas se utilizan como claves HashMap.

Ejemplo:


  Runnable r = new Runnable() {
    @Override
    public void run() {
      System.out.println();
    }
  };

Después de aplicar la solución rápida:


  Runnable r = System.out::println;

Utilice la opción Informar cuando la interfaz no está anotada con @FunctionalInterface para habilitar esta inspección para interfaces que no están anotadas con @FunctionalInterface.
