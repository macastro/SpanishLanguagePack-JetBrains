Informa a lambda que puede ser reemplazado por una referencia de método. Esto es generalmente una cuestión de preferencia, pero se puede decir que las referencias de método son más fáciles de entender y leer que las lambda.

Ejemplos:


  Runnable r = () -> System.out.println();

Después de aplicar la corrección rápida:


  Runnable r = System.out::println;

Esta inspección puede sugerir una referencia de método, incluso si la lambda no llama a ningún método (por ejemplo, reemplazando obj -> obj != null por Objects::nonNull). Para configurar referencias de método específicas, usa la configuración "Configuración | Editor | Estilo de código | Java | Generación de código".
