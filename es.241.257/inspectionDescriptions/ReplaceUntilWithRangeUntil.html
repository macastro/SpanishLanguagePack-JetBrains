Informa de los until que se pueden reemplazar por el operador ..<.

Ningún reemplazo de until por ..< cambia la semántica de ninguna manera.

La investigación UX muestra que los desarrolladores cometen ~20-30% menos errores al leer código que contiene ..< en comparación con until.

Ejemplo:


  fun main(args: Array<String>) {
      for (index in 0 until args.size) {
          println(index)
      }
  }

Después de aplicar la solución rápida:


  fun main(args: Array<String>) {
      for (index in 0..<args.size) {
          println(index)
      }
  }

Esta inspección solo informa si el nivel de lenguaje Kotlin del proyecto o módulo es 1,7 o superior.
