Informa sobre los campos que no se usan en el método toString() de una clase.

Ayuda a descubrir los campos añadidos después de que el método toString() se actualizó por última vez. La corrección rápida regenera el método toString().

En el cuadro de diálogo "Generar | toString()", es posible excluir campos de esta comprobación. Esta inspección también buscará problemas con los métodos getter si la opción Habilitar getters en la generación de código está activada allí.

Ejemplo:


  public class Relevant {
    private String name; // no usado en toString()
    private int index;
    private int length;

    @Override
    public String toString() {
        return "Relevant{" + "index=" + index +
          ", length=" + length + '}';
    }
  }

Después de aplicar la corrección rápida:


  public class Relevant {
    private String name;
    private int index;
    private int length;

    @Override
    public String toString() {
        return "Relevant{" + "name='" + name + '\'' +
          ", index=" + index + ", length=" + length + '}';
    }
  }
