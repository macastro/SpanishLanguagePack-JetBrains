Informa de llamadas a los métodos wait() que pueden darse mientras el hilo actual sujeta dos bloqueos.

Dado que invocar wait() sólo libera un bloqueo en su objetivo, esperar con dos bloqueos sujetos puede llevar fácilmente a un punto muerto.

Ejemplo:


  synchronized (lockA) {
    synchronized (lockB) {
      lockB.wait(); //advertencia
      //el hilo A está atascado aquí sujetando lockA
    }
  }

  synchronized (lockA) { //el hilo B no puede entrar en el bloque y liberar al hilo A
    lockB.notify();
  }
