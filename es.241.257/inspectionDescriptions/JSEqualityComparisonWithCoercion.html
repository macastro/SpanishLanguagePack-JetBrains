Reporta el uso de un operador de igualdad que puede causar coerciones de tipo inesperadas. Sugiere reemplazar == y != con operadores de igualdad seguros para el tipo, === y !==.

Según la opción seleccionada, se informará uno de los siguientes casos:

  • Todos los usos de los operadores == y !=.
  • Todos los usos excepto la comparación con null. Algunos estilos de código permiten usar x == null como reemplazo de x === null || x === undefined.
  • Solo expresiones sospechosas, como: comparaciones == o != con 0, '', null, true, false o undefined.
