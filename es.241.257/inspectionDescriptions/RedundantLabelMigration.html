A partir de Kotlin 1.4, informa etiquetas innecesarias que pueden provocar errores de compilación.

A partir de Kotlin 1.0, cualquier declaración se puede marcar con la siguiente etiqueta:


  fun foo() {
    L1@ val x = L2@bar()
  }
Sin embargo, solo se puede hacer referencia a estas etiquetas de las siguientes maneras limitadas:
  • romper o continuar en bucle
  • Retorno no local de lambda en línea o función anónima en línea

Este tipo de etiquetas están prohibidas a partir de Kotlin 1.4.

Esta verificación solo informa si el nivel de lenguaje Kotlin de su proyecto o módulo es 1.4 o superior.
